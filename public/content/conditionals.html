Conditional Statements

How to make decisions in code - execute different blocks based on conditions. Covers if, if-else, nested if-else, and switch-case with practical examples.

Concepts

Two Types of Statements

✓

Real-Life Example: Think of statements as road signs:
- Conditional = A fork in the road - "If it's raining, take the tunnel. Otherwise, take the bridge." You choose ONE path based on a condition.
- Looping = A roundabout - you keep going around until you find your exit. You repeat the same action multiple times.

Type 1
Conditional / Decision-Making
if, if-else, nested if-else, switch-case
Type 2
Looping / Iterative
while, do-while, for (next topic)
Type 1

if Statement - "Do this IF true"

✓

Real-Life Example: An if statement is like a bouncer at a club - "IF you have an ID, you can enter." No ID? Nothing happens - you just walk away. There's no "else" - the bouncer doesn't say anything to people who don't qualify.

Syntax

if (condition) {
  // runs ONLY if condition is true
  // if false - this entire block is skipped
}
1
Example - voting eligibility
if only 
let age: number = 20;

if (age >= 18) {
  console.log("You are eligible to vote!");
}
// Output: "You are eligible to vote!"

// Change age to 15 - NO output at all (condition is false)

The condition inside () must return true or false. Use relational operators (> < >= <= == !=) or logical operators (&& || !).

Type 2

if - else - "Do this OR that"

✓

Real-Life Example: if-else is like a traffic light - green means GO (if block runs), red means STOP (else block runs). One of the two ALWAYS happens. You never just stand at the intersection doing nothing.

Syntax

if (condition) {
  // runs if condition is TRUE
} else {
  // runs if condition is FALSE
}
// Only ONE block executes - never both!
2
Example - even or odd number
if-else 
let num: number = 10;

if (num % 2 === 0) {
  console.log(`${num} is an Even number`);
} else {
  console.log(`${num} is an Odd number`);
}
// Output: "10 is an Even number"

// num = 15 - Output: "15 is an Odd number"

% is the modular operator - it returns the remainder. If num % 2 is 0, the number divides evenly by 2 - it's even!

i

Template literals reminder: Use backticks ` with ${variable} to insert variable values into strings. Single/double quotes won't work for this - use + concatenation instead.

Type 3

Nested if - else if - Multiple Conditions

✓

Real-Life Example: Nested if-else is like a restaurant menu with price tiers:
- Budget = $100? - Fine dining
- Budget = $50? - Casual restaurant
- Budget = $20? - Fast food
- Otherwise - Cook at home
The waiter checks from top to bottom. The FIRST match wins - no need to check further.

Syntax

if (condition1) {
  // runs if condition1 is true
} else if (condition2) {
  // runs if condition1 is FALSE and condition2 is TRUE
} else if (condition3) {
  // runs if condition1 & 2 are FALSE and condition3 is TRUE
} else {
  // runs if ALL conditions are false (catch-all)
}
3
Example - grade calculator
Nested 
let marks: number = 85;

if (marks >= 90 && marks <= 100) {
  console.log("Grade A");
} else if (marks >= 75 && marks < 90) {
  console.log("Grade B");
} else if (marks >= 60 && marks < 75) {
  console.log("Grade C");
} else {
  console.log("Grade D");
}
// marks = 85 - "Grade B"
// marks = 95 - "Grade A"
// marks = 50 - "Grade D"

Use && (AND) to combine conditions for ranges. The first matching condition executes - the rest are skipped.

4
Example - browser selection (with strict equality)
Automation 
let browser: string = "Chrome";

if (browser === "Chrome") {
  console.log("Browser is Chrome");
} else if (browser === "Firefox") {
  console.log("Browser is Firefox");
} else if (browser === "Safari") {
  console.log("Browser is Safari");
} else {
  console.log("Some other browser");
}
// Output: "Browser is Chrome"

// - Using === (strict equality) to compare value AND type.
// This is a common pattern in test automation!
i

Top-to-bottom: Conditions are checked from first to last. The moment ONE matches, its block runs and the rest are completely skipped. Only one block ever executes.

Type 4

switch - case - Match a Value

✓

Real-Life Example: A switch statement is like a vending machine. You press a button (the expression), and the machine checks which slot matches your button: A1 - chips, A2 - soda, A3 - candy. If no slot matches ? "invalid selection" (default). One button, one product.

Syntax

switch (expression) {
  case value1:
    // runs if expression === value1
    break;
  case value2:
    // runs if expression === value2
    break;
  default:
    // runs if NO case matches (optional)
}
Term In if-else In switch-case
What goes in ()? A condition (returns true/false) An expression (returns any value)
How are values checked? Relational operators (> < ==) Exact match with case values
Best for Ranges, complex logic Many exact-value comparisons
5
Example - day of the week
switch 
let day: number = 3;

switch (day) {
  case 1: console.log("Monday");    break;
  case 2: console.log("Tuesday");   break;
  case 3: console.log("Wednesday"); break;
  case 4: console.log("Thursday");  break;
  case 5: console.log("Friday");    break;
  case 6: console.log("Saturday");  break;
  case 7: console.log("Sunday");    break;
  default: console.log("Invalid day");
}
// day = 3 - "Wednesday"
// day = 7 - "Sunday"
// day = 10 - "Invalid day"
6
Example - expression in switch (not just a variable)
Expression 
let x: number = 20, y: number = 10;

switch (x - y) {       // expression returns 10
  case 0:
    console.log("Result is zero");
    break;
  case 5:
    console.log("Result is five");
    break;
  case 10:
    console.log("Result is ten");   // - this runs!
    break;
  default:
    console.log("Result is something else");
}
// x - y = 10 - matches case 10 - "Result is ten"
Important!

Why break is Mandatory

✓

Real-Life Example: break is like an exit door in a building. If you find your room on floor 3, you go in and leave through the exit. Without the exit door, you'd keep walking through floor 4, floor 5, floor 6 - visiting rooms you never needed!

!
Without break - fall-through bug!
? Bug 
let day: number = 1;

switch (day) {
  case 1: console.log("Monday");     // - matches here
  // NO break! Falls through to case 2...
  case 2: console.log("Tuesday");    // - also runs!
  // NO break! Falls through to case 3...
  case 3: console.log("Wednesday");  // - also runs!
    break;                            // - finally stops here
}
// Output:
// Monday
// Tuesday
// Wednesday   - only wanted Monday!

Without break, execution "falls through" to the next case regardless of whether it matches. Always use break after each case!

Watch Out!

? Common Beginner Mistakes

⚠
Mistake 1: Multiple if blocks vs else if

Separate if blocks are like separate doors - the computer checks EVERY one independently. else if is like one door with multiple locks - it stops at the FIRST one that opens. 

let marks = 95;

// - WRONG - ALL matching ifs run! (3 outputs!)
if (marks >= 60) {
  console.log("Pass");     // - Prints (95 >= 60)
}
if (marks >= 80) {
  console.log("Good");     // - Prints (95 >= 80)
}
if (marks >= 90) {
  console.log("Excellent"); // - Prints (95 >= 90)
}
// Output: Pass, Good, Excellent - 3 messages!

// - CORRECT - stops at FIRST match!
if (marks >= 90) {
  console.log("Excellent"); // - Prints this ONLY
} else if (marks >= 80) {
  console.log("Good");     // Skipped
} else if (marks >= 60) {
  console.log("Pass");     // Skipped
}
// Output: Excellent - only 1 message!

Rule: Use else if when you want ONLY ONE block to run. Use separate if blocks only when MULTIPLE conditions can be true at the same time and you want ALL of them to execute.

⚠
Mistake 2: Using = instead of === in conditions

This is the #1 most common bug for beginners. A single = means "put this value in the box." It does NOT mean "check if they're equal." 

let color = "red";

// - WRONG - assigns "blue" to color, then checks if "blue" is truthy
if (color = "blue") {
  console.log("It's blue!"); // ALWAYS runs!
  // color is now "blue" - you accidentally changed it!
}

// - CORRECT - compares color to "blue"
if (color === "blue") {
  console.log("It's blue!"); // Only runs if color was actually "blue"
}

Rule: In conditions, ALWAYS use === (three equals). If you see = (one equal) inside an if, it's almost certainly a bug!

What you wrote What happens What you wanted
Separate if + if + if ALL matching blocks run if - else if - else if
if (x = 5) Assigns 5 to x (always true) if (x === 5)
switch without break Falls through ALL cases below Add break; after each case

When to Use Which?

Statement Use when Conditions Has else?
if Single positive check 1 No
if-else Two paths (yes/no) 1 Yes
nested if-else Multiple conditions/ranges Many Yes
switch-case Many exact-value matches Many default (optional)
Feature Nested if-else switch-case

Quick Reference Cheatsheet

if - single check

if (age >= 18) {
  console.log("Adult");
}

if-else - two paths

if (num % 2 === 0) {
  console.log("Even");
} else {
  console.log("Odd");
}

Nested if-else - multiple conditions

if (marks >= 90) {
  console.log("Grade A");
} else if (marks >= 75) {
  console.log("Grade B");
} else if (marks >= 60) {
  console.log("Grade C");
} else {
  console.log("Grade D");
}

switch-case - exact value matching

switch (day) {
  case 1: console.log("Monday");  break;
  case 2: console.log("Tuesday"); break;
  case 3: console.log("Wednesday"); break;
  default: console.log("Other day");
}

Key rules

// - Conditions return true/false (for if-else)
// - Expressions return any value (for switch)
// - Always use break in switch-case
// - default is optional but recommended
// - Use === (strict equality) for string comparisons
// - Only ONE block executes - the first match wins
