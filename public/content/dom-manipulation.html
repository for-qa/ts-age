Basics

What is the DOM?

✓

Real-Life Example: Your HTML page is like a building. The DOM is the blueprint with labels on every room, door, and window. JavaScript/TypeScript lets you rearrange the furniture, paint walls, add rooms, or remove doors — all through this blueprint.

!
DOM = Document Object Model
Core Concept 
// The browser reads your HTML and builds a tree of objects
// Each HTML element becomes a "node" in this tree

// HTML:
// <div id="app">
//   <h1>Hello</h1>
//   <p class="desc">World</p>
// </div>

// DOM Tree:
// document
//   └── html
//       └── body
//           └── div#app
//               ├── h1 → "Hello"
//               └── p.desc → "World"

// TypeScript can read, modify, and create any of these nodes!
Basics

Selecting Elements

1
3 ways to find elements
Selection 
// 1. By ID — returns ONE element or null
const app = document.getElementById("app");
// Type: HTMLElement | null

// 2. By CSS selector — returns FIRST match or null
const title = document.querySelector("h1");
// Type: Element | null

const desc = document.querySelector(".desc");
const btn = document.querySelector("#submit-btn");

// 3. By CSS selector — returns ALL matches (NodeList)
const items = document.querySelectorAll(".item");
// Type: NodeListOf<Element>

// Loop through all matches:
items.forEach(item => {
  console.log(item.textContent);
});

// ⚠ KEY DIFFERENCE:
// getElementById → returns HTMLElement | null
// querySelector  → returns Element | null (more generic)
// querySelectorAll → returns NodeListOf<Element> (static list)

Type Assertions — TypeScript-Specific!

✓

Why this matters: TypeScript doesn't know what HTML element you're selecting. It only sees a generic Element. You need to tell TypeScript what specific type it is (like HTMLInputElement) to access properties like .value.

2
Tell TypeScript what type the element is
as Type 
// ❌ ERROR — generic Element has no .value property
const input = document.querySelector("#email");
// console.log(input.value);  // TS Error: Property 'value' not on Element

// ✅ Type assertion — tell TS it's an input
const emailInput = document.querySelector("#email") as HTMLInputElement;
console.log(emailInput.value);  // ✅ works!

// Common HTML element types:
const btn = document.querySelector("button") as HTMLButtonElement;
const img = document.querySelector("img") as HTMLImageElement;
const link = document.querySelector("a") as HTMLAnchorElement;
const form = document.querySelector("form") as HTMLFormElement;
const div = document.querySelector("#box") as HTMLDivElement;
const select = document.querySelector("select") as HTMLSelectElement;

// ✅ Safer way — check for null first!
const el = document.querySelector("#email");
if (el) {
  const input = el as HTMLInputElement;
  console.log(input.value);
}
Modifying

Changing Content

3
textContent vs innerHTML
Content 
const heading = document.querySelector("h1") as HTMLHeadingElement;

// ✅ textContent — safe, plain text only
heading.textContent = "New Title";

// ⚠ innerHTML — parses HTML (security risk with user input!)
heading.innerHTML = "<em>Styled</em> Title";

// Reading content:
console.log(heading.textContent);  // "Styled Title" (text only)
console.log(heading.innerHTML);    // "<em>Styled</em> Title" (with HTML)

// Attributes:
const link = document.querySelector("a") as HTMLAnchorElement;
link.href = "https://example.com";     // set attribute
link.setAttribute("target", "_blank"); // alternative way
console.log(link.getAttribute("href")); // read attribute
Modifying

Changing Styles & Classes

4
Inline styles and CSS classes
Styling 
const box = document.querySelector("#box") as HTMLDivElement;

// Inline styles (camelCase, not kebab-case!)
box.style.backgroundColor = "blue";   // not "background-color"
box.style.fontSize = "20px";
box.style.display = "none";           // hide element

// ✅ Better: use CSS classes (preferred approach)
box.classList.add("active");           // add a class
box.classList.remove("hidden");        // remove a class
box.classList.toggle("dark-mode");     // toggle on/off
box.classList.contains("active");      // check: true/false

// Replace one class with another:
box.classList.replace("old-class", "new-class");

Best practice: Define styles in CSS and toggle classes with classList. Avoid inline .style when possible.

Creating & Removing Elements

5
Build and destroy DOM nodes
Create/Delete 
// CREATE a new element
const newItem = document.createElement("li");
newItem.textContent = "New task";
newItem.classList.add("todo-item");

// ADD to the page
const list = document.querySelector("#todo-list") as HTMLUListElement;
list.appendChild(newItem);            // add at END
list.prepend(newItem);                // add at START
list.insertBefore(newItem, list.firstChild);  // add before specific child

// REMOVE an element
newItem.remove();                     // modern way — remove itself

// REPLACE an element
const oldItem = list.querySelector(".old") as HTMLLIElement;
list.replaceChild(newItem, oldItem);  // replace old with new

Event Handling

✓

Real-Life: Events are like a doorbell — you set up a listener ("when the bell rings, open the door"). addEventListener sets up the listener. The event (click, keypress) is the bell ringing.

6
Listen for user interactions
Events 
const btn = document.querySelector("#btn") as HTMLButtonElement;

// ✅ Add an event listener
btn.addEventListener("click", () => {
  console.log("Button clicked!");
});

// ✅ With the event object
btn.addEventListener("click", (event) => {
  console.log(event.target);     // the element that was clicked
  console.log(event.type);       // "click"
});

// ✅ Remove a listener (must use named function)
function handleClick() {
  console.log("Clicked!");
}
btn.addEventListener("click", handleClick);
btn.removeEventListener("click", handleClick);

// Common events:
// "click"     — mouse click
// "dblclick"  — double click
// "mouseover" — hover enter
// "mouseout"  — hover leave
// "keydown"   — key pressed
// "keyup"     — key released
// "submit"    — form submitted
// "change"    — input value changed
// "input"     — input value changing (real-time)
✓

Event Types in TypeScript

7
TypeScript knows what kind of event it is
Type Safety 
// TypeScript has specific event types for type safety:

// Mouse events
btn.addEventListener("click", (e: MouseEvent) => {
  console.log(e.clientX, e.clientY);  // mouse position
});

// Keyboard events
document.addEventListener("keydown", (e: KeyboardEvent) => {
  console.log(e.key);      // "Enter", "Escape", "a", etc.
  console.log(e.code);     // "KeyA", "Enter", etc.
  if (e.key === "Enter") {
    console.log("Enter pressed!");
  }
});

// Input events
const input = document.querySelector("input") as HTMLInputElement;
input.addEventListener("input", (e: Event) => {
  const target = e.target as HTMLInputElement;
  console.log(target.value);  // current input value
});

// Focus events
input.addEventListener("focus", (e: FocusEvent) => { });
input.addEventListener("blur", (e: FocusEvent) => { });

Form Handling

8
Prevent default, read values, validate
Forms 
const form = document.querySelector("#login-form") as HTMLFormElement;

form.addEventListener("submit", (e: SubmitEvent) => {
  e.preventDefault();  // ⚠ MUST DO — stops page reload!

  const email = (document.querySelector("#email") as HTMLInputElement).value;
  const pass = (document.querySelector("#pass") as HTMLInputElement).value;

  if (!email || !pass) {
    console.log("Please fill all fields");
    return;
  }

  console.log("Logging in:", email);
  // Send to server...
});

// Reading different input types:
const checkbox = document.querySelector("#agree") as HTMLInputElement;
console.log(checkbox.checked);  // true or false (not .value!)

const dropdown = document.querySelector("select") as HTMLSelectElement;
console.log(dropdown.value);    // selected option's value
Watch Out!

⚠ Common Beginner Mistakes

⚠
Mistake 1: querySelector returns null — not checking!

If the element doesn't exist, you get null. Calling methods on null crashes your app. 

// ❌ CRASH — element might not exist
const btn = document.querySelector("#btn");
btn.addEventListener("click", () => {});
// TypeError: Cannot read properties of null

// ✅ SAFE — null check first
const btn2 = document.querySelector("#btn");
if (btn2) {
  btn2.addEventListener("click", () => {});
}

// ✅ OR use non-null assertion (only if you're SURE it exists)
const btn3 = document.querySelector("#btn")!;  // ! = "trust me, it's there"

Rule: Always check for null or use ! only when you're 100% certain the element exists.

⚠
Mistake 2: Forgetting type assertion — .value doesn't exist on Element 
// ❌ ERROR — generic Element has no .value
const input = document.querySelector("#name");
console.log(input.value);
// Property 'value' does not exist on type 'Element'

// ✅ CORRECT — assert the specific type
const input2 = document.querySelector("#name") as HTMLInputElement;
console.log(input2.value);  // works!

Rule: Use as HTMLInputElement (or the correct type) when you need element-specific properties.

⚠
Mistake 3: innerHTML with user input — XSS vulnerability! 
// ❌ DANGEROUS — user can inject scripts!
const userInput = '<img src=x onerror="alert(1)">';
div.innerHTML = userInput;  // 🚨 XSS attack!

// ✅ SAFE — textContent escapes HTML
div.textContent = userInput;  // Displays as plain text

Rule: NEVER use innerHTML with user-provided data. Use textContent instead.

⚠
Mistake 4: Forgetting e.preventDefault() on forms 
// ❌ Page reloads, losing all your data!
form.addEventListener("submit", () => {
  console.log("This logs then page reloads...");
});

// ✅ Prevent the default browser behavior
form.addEventListener("submit", (e) => {
  e.preventDefault();  // stops page reload
  console.log("Now you can handle the form!");
});

Rule: Always call e.preventDefault() first in form submit handlers.

⚠
Mistake 5: getElementsByClassName returns LIVE collection 
// querySelectorAll = STATIC snapshot (safe)
const items1 = document.querySelectorAll(".item");
// Adding/removing .item elements won't change items1

// getElementsByClassName = LIVE collection (changes!)
const items2 = document.getElementsByClassName("item");
// Adding/removing .item elements WILL update items2 automatically
// This can cause unexpected bugs in loops!

Rule: Prefer querySelectorAll — it returns a static snapshot that won't surprise you.

Mistake What happens Fix
No null check TypeError crash if (el) or el!
No type assertion TS error on .value as HTMLInputElement
innerHTML + user data XSS attack Use textContent
No preventDefault() Page reloads e.preventDefault()
Live vs static collection Unexpected mutations Use querySelectorAll

Quick Reference Cheatsheet

Select

document.getElementById("id")          // by ID
document.querySelector(".class")       // first match

Type Assert

const el = document.querySelector("#x") as HTMLInputElement;
const el = document.querySelector("#x")!;  // non-null assert

Modify

el.textContent = "text";               // safe text
el.innerHTML = "<b>html</b>";           // parse HTML
el.style.color = "red";               // inline style

const li = document.createElement("li");
parent.appendChild(li);                // add child
el.remove();                           // remove self

el.addEventListener("click", (e: MouseEvent) => { });
