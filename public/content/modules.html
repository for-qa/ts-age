Core Concept

What is a Module?

✓

Real-Life Example: A module is like a toolbox in a workshop. Instead of carrying a hammer, screwdriver, and wrench everywhere, you keep them in ONE toolbox. Any worker (file) who needs a tool just opens the toolbox (imports) and uses what they need.

A module is a file that contains reusable components -- variables, functions, classes -- that can be shared across multiple files using export and import.

What can a module contain? Example
Variables export let appName = "Calculator";
Functions export function add(a, b) { return a + b; }
Classes export class Formatter { ... }
Interfaces export interface Person { ... }
i

Module ≠ Class. Don't compare them. A module is a FILE containing reusable code. A class is a blueprint for objects. A module can CONTAIN classes, functions, variables -- anything.

Why Modules?

The Problem Without Modules

✓

Real-Life Example: Imagine 3 classrooms all need the same math formulas. Without modules, each teacher writes the formulas on their own board (duplication!). With modules, there's ONE reference sheet that all classrooms share. 

// ❌ WITHOUT modules  --  duplication everywhere!

// test1.ts
function add(a: number, b: number) { return a + b; }  // duplicated!
console.log(add(10, 20));

// test2.ts
function add(a: number, b: number) { return a + b; }  // duplicated!
console.log(add(5, 15));

// test3.ts
function add(a: number, b: number) { return a + b; }  // duplicated!
console.log(add(100, 200));

Problems: No reusability, code duplication, increased file length, harder to maintain. 

// ❌ Without export, other files CANNOT access your code!

// module.ts
let appName = "Calculator";

// main.ts
console.log(appName);  // ERROR: appName is not defined!
Step 1

export -- Make Code Available to Other Files

Add the export keyword in front of any variable, function, or class you want to share.

1
module.ts -- the reusable file
export 
// Export a variable
export let appName: string = "Calculator";

// Export a function
export function add(a: number, b: number): number {
  return a + b;
}

// Export a class
export class Formatter {
  static toUpper(str: string): string {
    return str.toUpperCase();
  }
}

Each export keyword makes that item available for other files to import. Without export, the item stays private to this file.

⚠

Every item needs export. You must write export in front of EACH variable, function, or class you want to share. There's no single "export all" shortcut at the top.

Step 2

import -- Use Code from Another File

In the file where you want to USE the module's code, write an import statement at the top. 

// Import individual items
import { appName } from "./module";
import { add } from "./module";
import { Formatter } from "./module";

// OR  --  import everything in ONE line!
import { appName, add, Formatter } from "./module";
//                                       → no .ts extension!
Import Rule Detail
Syntax import { name } from "./path"
Path Relative path: ./ = same folder
Extension Do NOT include .ts -- just the filename
Multiple items Comma-separated inside { }
Position Must be at the TOP of the file
Complete Example

Full Module Workflow

A
module.ts -- the reusable file
export 
export let appName: string = "Calculator";

export function add(a: number, b: number): number {
  return a + b;
}

export class Formatter {
  static toUpper(str: string): string {
    return str.toUpperCase();
  }
}
B
main.ts -- uses the module
import 
import { appName, add, Formatter } from "./module";

console.log(appName);               // "Calculator"
console.log(add(10, 20));            // 30
console.log(Formatter.toUpper("welcome"));  // "WELCOME"

Run main.ts, NOT module.ts. The module is a utility file -- it's not meant to be executed directly.

i

Execution rule: You execute the file that IMPORTS (main.ts), not the file that exports (module.ts). The module is just a library of reusable code.

Rules

Key Rules for Modules

# Rule
1 Add export in front of every item you want to share
2 Use import { ... } from "./filename" to bring items in
3 Do NOT include the .ts extension in the import path
4 Without export, code stays private to its file
5 Without import, you cannot use exported code
6 Execute the importing file, not the module file
7 A module can contain variables, functions, classes, interfaces -- anything
How-To

Default Export -- One Main Thing Per File

✓

Real-Life: A named export is like a store with many labeled shelves. A default export is like a store with ONE featured product in the window -- you don't need to know its name to grab it.

D
export default -- one per file
Default 
// math.ts  --  default export
export default function add(a: number, b: number): number {
  return a + b;
}

// You can ALSO have named exports alongside:
export const PI = 3.14;

// app.ts  --  import default (no curly braces!)
import add from "./math";       // name it whatever you want!
import myAdd from "./math";    // same thing, different name
import add, { PI } from "./math";  // default + named together

console.log(add(2, 3));  // 5
console.log(PI);         // 3.14

Rule: Only ONE export default per file. Named exports can have many. Import default = no { }. Import named = needs { }.

How-To

Type-Only Imports (TS 3.8+)

i

When you import a type or interface that only exists at compile time, use import type. This tells TypeScript "this import is only for type checking -- don't include it in the compiled JavaScript output."

T
import type -- types only
Type-Only 
// types.ts
export interface User {
  name: string;
  age: number;
}
export type Role = "admin" | "user" | "guest";

// app.ts  --  type-only import
import type { User, Role } from "./types";

function greet(user: User): string {
  return `Hello ${user.name}`;
}

// ✅ The import is erased from compiled JS  --  zero runtime cost
// ✅ Prevents accidentally using a type as a value
Reference

Quick Reference Cheatsheet

Export (in module file)

export let x = 10;
export function doStuff() { ... }
export class Helper { ... }
export interface Config { ... }
export default function main() { ... }  // one per file
export type Role = "admin" | "user";     // type export

Import (in consuming file)

// Named imports
import { x, doStuff, Helper } from "./module";

// Default import (no curly braces)
import main from "./module";

// Default + named together
import main, { x, doStuff } from "./module";

// Type-only import (erased at compile time)
import type { Config } from "./module";

Use case -- Automation utility

// utils.ts (module)
export function waitForElement(selector: string) { ... }
export function takeScreenshot(name: string) { ... }
export function login(user: string, pass: string) { ... }

// test-login.ts (test file)
import { login, takeScreenshot } from "./utils";
login("admin", "pass123");
takeScreenshot("dashboard");
