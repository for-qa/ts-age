Why Do We Need Loops?

✓

Real-Life Example: Imagine you need to stamp 100 envelopes. Without a loop, you'd write 100 separate instructions: "stamp envelope 1," "stamp envelope 2," ... all the way to 100. With a loop, you write ONE instruction: "stamp the next envelope" and repeat it 100 times. Same work, way less writing!

!
Without loops -- repetitive and painful
❌ Bad 
// Printing 1 to 5 WITHOUT a loop  --  5 statements!
console.log(1);
console.log(2);
console.log(3);
console.log(4);
console.log(5);
// What if you need 1 to 1000? Write 1000 lines? ✓
✓
With a loop -- write once, repeat many times
✅ Smart 
// Printing 1 to 5 WITH a loop  --  just 3 lines!
for (let i = 1; i <= 5; i++) {
  console.log(i);
}
// Change 5 to 1000? Just change ONE number!
Concepts

Every Loop Needs 3 Things

✓

Real-Life Example: A loop is like climbing stairs:
✓ Start -- Which step do you begin on? (step 1)
✓ Stop -- Which step do you stop at? (step 10)
✓ Step size -- How many steps at a time? (1 at a time, or 2 at a time)

①
Initialization
Where to start (let i = 1)
②
Condition
When to stop (i <= 10)
③
Increment / Decrement
Step size (i++ or i += 2)
Loop Type 1

while Loop -- Check FIRST, Then Run

✓

Real-Life Example: A while loop is like a security guard at a gate -- "BEFORE you enter, show me your pass." If your pass is valid (condition true), you enter. If not, you don't get in at all. The guard checks EVERY time before letting anyone through.

Syntax

let i: number = 1;        // ① Initialization (OUTSIDE)

while (i <= 5) {           // ② Condition
  console.log(i);          // Statements (repeat)
  i++;                     // ③ Increment (INSIDE block)
}
1
Print 1 to 5
Ascending 
let i: number = 1;
while (i <= 5) {
  console.log(i);  // 1, 2, 3, 4, 5
  i++;
}
// i starts at 1 → prints → increments → checks condition → repeats
// When i becomes 6 → 6 <= 5 is FALSE → loop exits
2
Print only even numbers (1 -- 10)
Filter 
// Method 1: Start from 2, increment by 2
let i: number = 2;
while (i <= 10) {
  console.log(i);  // 2, 4, 6, 8, 10
  i += 2;           // jump by 2 each time
}

// Method 2: Check with % (modular operator)
let j: number = 1;
while (j <= 10) {
  if (j % 2 === 0) {
    console.log(j);  // 2, 4, 6, 8, 10
  }
  j++;
}
3
Descending order (10 to 1)
Descending 
let i: number = 10;
while (i >= 1) {
  console.log(i);  // 10, 9, 8, ... 2, 1
  i--;              // decrease by 1
}
⚠

Infinite Loop Danger! If the condition NEVER becomes false, the loop runs forever:
while (true) { console.log("forever!"); }
Always make sure your condition will eventually become false. Press Ctrl+C to force-stop an infinite loop.

Loop Type 2

do ... while Loop -- Run FIRST, Then Check

✓

Real-Life Example: A do-while loop is like a food tasting station -- you get ONE free sample BEFORE they ask "Do you want more?" You always taste at least once, even if you decide not to buy. The action happens first, the question comes after.

Syntax

let i: number = 1;         // ① Initialization

do {
  console.log(i);           // Statements (run AT LEAST ONCE)
  i++;                      // ③ Increment
} while (i <= 5);           // ② Condition (checked AFTER)
!
Key difference -- runs even when condition is false!
At least once 
// while loop  --  condition checked FIRST
let i: number = 10;
while (i <= 5) {
  console.log(i);  // ❌ NEVER runs! 10 <= 5 is false
  i++;
}

// do-while  --  runs ONCE before checking
let j: number = 10;
do {
  console.log(j);  // ✅ Prints 10! (runs before checking)
  j++;
} while (j <= 5);
// After printing 10, checks 11 <= 5 → false → exits
Feature while do-while
Checks condition BEFORE running AFTER running
Minimum executions 0 (might never run) 1 (always runs once)
Use when You might not need to run at all You must run at least once
Loop Type 3

for Loop -- All-in-One (Most Popular!)

✓

Real-Life Example: A for loop is like programming a washing machine -- you set everything upfront in ONE panel: start temperature, number of cycles, rinse speed. Press start and it handles everything. No separate buttons. The for loop puts all 3 parts (start, stop, step) in ONE line.

Syntax

for (initialization; condition; increment) {
  // statements
}

// Example:
for (let i = 1; i <= 10; i++) {
  console.log(i);  // 1, 2, 3, ... 10
}

Execution workflow

①
Initialize
Runs ONCE only
②
Check condition
true → continue
③
Run statements
Execute block
④
Increment
Then back to ②
⚠
Common Mistake: j + 1 does NOT increment!

A very common beginner mistake is writing j + 1 in the increment section of a for loop. This looks like it adds 1 to j, but it does NOT -- it just calculates a result and immediately throws it away. The variable j is never updated, so the condition never becomes false and the loop runs forever! 

// ❌ WRONG  --  infinite loop! j stays 1 forever
for (let j = 1; j <= 10; j + 1) {
  console.log(j);  // prints 1... 1... 1... forever
}

// ✅ CORRECT  --  j is actually saved each time
for (let j = 1; j <= 10; j++) {
  console.log(j);  // 1, 2, 3, ... 10
}

Rule: Always use an assignment (something with =) so the new value is saved back into the variable:

Expression Saves to j? Result
j + 1 ❌ No Infinite loop
j++ ✅ Yes Works correctly
j += 1 ✅ Yes Works correctly
j = j + 1 ✅ Yes Works correctly
1
Even numbers with for loop
i += 2 
// Method 1: Start at 2, step by 2
for (let i = 2; i <= 10; i += 2) {
  console.log(i);  // 2, 4, 6, 8, 10
}

// Method 2: Check with %
for (let i = 1; i <= 10; i++) {
  if (i % 2 === 0) {
    console.log(i);  // 2, 4, 6, 8, 10
  }
}
2
Descending order
i-- 
for (let i = 10; i >= 1; i--) {
  console.log(i);  // 10, 9, 8, ... 2, 1
}
Important!

Variable Scope in for Loops

✓

Real-Life Example: A variable declared with let inside the for() is like a wristband at a concert -- it only works inside the venue. Once you leave (loop ends), the wristband is invalid. A variable declared outside is like a government ID -- it works everywhere.

1
let inside for = LOCAL (can't access outside)
Local scope 
for (let i = 1; i <= 5; i++) {
  console.log(i);  // 1, 2, 3, 4, 5 ✅
}
console.log(i);    // ❌ ERROR! 'i' doesn't exist here
// 'i' was born inside the loop and dies when the loop ends
2
Variable declared outside = GLOBAL (accessible everywhere)
Global scope 
let i: number;                // declared OUTSIDE (global)
for (i = 1; i <= 5; i++) {   // no 'let' here  --  reusing global
  console.log(i);             // 1, 2, 3, 4, 5
}
console.log(i);               // 6 ← accessible! (last value after loop exits)

When the loop exits, i is 6 (the value that made the condition false). Since i is global, it's still accessible.

Jumping Statements

break -- "Emergency Exit!"

✓

Real-Life Example: break is like pulling the fire alarm in a building -- everyone stops what they're doing and EXITS immediately. The loop doesn't finish naturally -- it's forced to stop.

1
Exit loop early when i reaches 5
Force exit 
for (let i = 1; i <= 10; i++) {
  if (i === 5) {
    break;  // ✓ EXIT the entire loop NOW!
  }
  console.log(i);
}
// Output: 1, 2, 3, 4
// 5 is NOT printed  --  break fires BEFORE console.log

// Move console.log BEFORE the if to print 5 too:
// Output would be: 1, 2, 3, 4, 5
Jumping Statements

continue -- "Skip This One!"

✓

Real-Life Example: continue is like skipping a song on a playlist -- you don't stop listening to music (the loop continues), you just skip that one track and move to the next one.

1
Skip specific values
Skip & go 
for (let i = 1; i <= 10; i++) {
  if (i === 5) {
    continue;  // ⊏ Skip 5, jump to next iteration
  }
  console.log(i);
}
// Output: 1, 2, 3, 4, 6, 7, 8, 9, 10
// 5 is MISSING  --  it was skipped!

// Skip multiple values (3, 5, 7):
for (let i = 1; i <= 10; i++) {
  if (i === 3 || i === 5 || i === 7) {  // use || (OR), NOT &&
    continue;
  }
  console.log(i);
}
// Output: 1, 2, 4, 6, 8, 9, 10

Use || (OR) to skip multiple values -- because i can only be ONE number at a time. && (AND) would never be true!

Statement What it does Analogy
break EXIT the entire loop immediately 💥 Fire alarm -- everyone out!
continue SKIP current iteration, go to next ⊏ Skip song on playlist

Quick Reference Cheatsheet

while

let i = 1;
while (i <= 5) {
  console.log(i);
  i++;
}

do-while

let i = 1;
do {
  console.log(i);
  i++;
} while (i <= 5);

for (most popular)

for (let i = 1; i <= 5; i++) {
  console.log(i);
}

Descending

for (let i = 10; i >= 1; i--) {
  console.log(i);
}

Even numbers

for (let i = 2; i <= 10; i += 2) {
  console.log(i);  // 2, 4, 6, 8, 10
}

break & continue

// break  --  exit entire loop
if (i === 5) break;

// continue  --  skip this iteration
if (i === 5) continue;
