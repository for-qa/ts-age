Higher-Order Array Methods

Six powerful methods that accept a function as a parameter and apply it to every element in an array — forEach, map, filter, reduce, some, every.

Common Behavior

One Rule They ALL Share

✓

Real-Life Example: All six methods work like a factory assembly line. Items (array elements) move down a conveyor belt one by one, and a worker (the function you pass) does something to each item. The difference is what happens to the output.

Every method takes a function as a parameter (callback). That function runs once per element. You can pass any function type — named, anonymous, or arrow.

Standard callback parameters

array.method(function(element, index, array) {
  // element → current value being processed (MANDATORY)
  // index   → position in the array (optional)
  // array   → the original array itself (optional)
});
Method Returns Key behavior
forEach Nothing (void) Just runs the function — no output
map New array (same size) Transforms each element
filter New array (same or fewer) Keeps only elements that pass a test
reduce Single value Accumulates all elements into one result
some boolean At least ONE element passes?
every boolean ALL elements pass?
✓

🧠 Memory Trick — Remember The Big Six:

"Functions Make Filtering Records Super Easy"
→ forEach, map, filter, reduce, some, every

💡 Three words literally match the methods: For, Some, Every!

Method 1

forEach — "Do something for each item"

✓

Real-Life Example: forEach is like a teacher taking attendance. She goes through each student name, calls it out loud, and moves to the next. She doesn't write a new list — she just performs an action for each name.

Definition: Executes a function once for each array element. It does NOT return anything.

1
Print fruits with index
forEach 
let fruits: string[] = ["apple", "banana", "orange", "mango", "grape"];

// Using anonymous function
fruits.forEach(function(element, index) {
  console.log(`${index}: ${element}`);
});

// Using arrow function (same result, shorter!)
fruits.forEach((element, index) => {
  console.log(`${index}: ${element}`);
});

// Output:
// 0: apple
// 1: banana
// 2: orange
// 3: mango
// 4: grape
2
Convert each fruit to uppercase
forEach 
fruits.forEach((element) => {
  console.log(element.toUpperCase());
});

// Output: APPLE, BANANA, ORANGE, MANGO, GRAPE
// forEach just DOES something — it doesn't create a new array

If index isn't needed, just pass the element parameter alone.

i

When to use forEach: When you want to perform a side effect (print, log, update DOM) for each element — and you do NOT need a new array back.

Method 2

map — "Transform and collect"

✓

Real-Life Example: map is like a photo filter app. You give it 10 photos, it applies a filter to EACH one, and gives back 10 new filtered photos. Same count in, same count out — but every item is transformed.

Definition: Creates a new array with the result of calling a function on every element. Always returns the same number of elements as the original.

1
Square each number
map 
let numbers: number[] = [1, 2, 3, 4, 5];

let squaredNumbers = numbers.map(function(element) {
  return element * element;
});

console.log(squaredNumbers);  // [1, 4, 9, 16, 25]
console.log(numbers);         // [1, 2, 3, 4, 5] ← original unchanged!

map never modifies the original array. It always returns a brand new one.

2
Double each number (arrow function)
map + implicit return 
// Full arrow function
let doubled = numbers.map((element) => {
  return element * 2;
});

// Simplified — single return = no { } and no 'return' keyword!
let doubled = numbers.map(element => element * 2);

console.log(doubled);  // [2, 4, 6, 8, 10]
i

Arrow function shortcut: If your callback has only one return statement, you can remove { } and the return keyword entirely. This is called implicit return.

Method 3

filter — "Keep only what passes the test"

✓

Real-Life Example: filter is like a bouncer at a club. Everyone in the line (array) is checked against a condition (age ≥ 21). Only those who pass get into the new array. The rest are turned away.

Definition: Creates a new array with only the elements that satisfy the condition. Returns the same or fewer elements.

1
Get only even numbers
filter 
let numbers: number[] = [1, 2, 3, 4, 5];

// Full version
let evenNumbers = numbers.filter(function(number) {
  return number % 2 === 0;
});

// Simplified arrow version
let evenNumbers = numbers.filter(num => num % 2 === 0);

console.log(evenNumbers);  // [2, 4]
// Only 2 and 4 passed the "is even?" test
2
Numbers greater than 3
filter 
let filtered = numbers.filter(num => num > 3);

console.log(filtered);  // [4, 5]
⚠

map vs filter: map transforms every element and always returns the same count. filter tests every element and may return fewer items. map = transform, filter = select.

Method 4

reduce — "Boil everything down to one value"

✓

Real-Life Example: reduce is like a snowball rolling downhill. It starts small (initial value = 0), picks up snow (adds each element) as it rolls, and at the bottom you have ONE big snowball (single result). The snowball IS the accumulator.

Definition: Applies a function on every element and returns a single value. Uses an accumulator — a running total that builds up with each iteration.

Syntax (slightly different!)

array.reduce((accumulator, element, index, array) => {
  return accumulator + element;
}, initialValue);  // ← initial value of accumulator (default: 0)
1
Sum all numbers
reduce 
let numbers: number[] = [1, 2, 3, 4, 5];

// Full version
let total = numbers.reduce(function(total, element) {
  return total + element;
}, 0);

// Simplified arrow version
let total = numbers.reduce((total, element) => total + element, 0);

console.log(total);  // 15

How the accumulator works step-by-step

Round Accumulator (before) Element Accumulator (after)
1 0 1 0 + 1 = 1
2 1 2 1 + 2 = 3
3 3 3 3 + 3 = 6
4 6 4 6 + 4 = 10
5 10 5 10 + 5 = 15 ✅
i

Initial value: The , 0 after the function sets the accumulator's starting value. If omitted, it defaults to 0. You can set it to 1, 100, or any value your logic needs.

Method 5

some — "Is at least ONE element passing?"

✓

Real-Life Example: some is like asking a classroom: "Has ANYONE finished the homework?" If even ONE student raises their hand → true. If nobody raises their hand → false.

Definition: Returns true if at least one element satisfies the condition. Otherwise false.

1
Does array contain any negatives?
some 
let numbers: number[] = [1, 2, 3, 4, 5];

let hasNegative = numbers.some(element => element < 0);
console.log(hasNegative);  // false — no negatives!

let hasPositive = numbers.some(element => element > 0);
console.log(hasPositive);  // true — at least one positive!
Method 6

every — "Do ALL elements pass?"

✓

Real-Life Example: every is like a fire safety inspector checking every room. If ALL rooms pass inspection → true. If even ONE room fails → the whole building gets false.

Definition: Returns true only if ALL elements satisfy the condition. Even one failure → false.

1
Are all numbers even?
every 
let numbers: number[] = [1, 2, 3, 4, 5];

let allEven = numbers.every(element => element % 2 === 0);
console.log(allEven);  // false — 1, 3, 5 are odd!

let allPositive = numbers.every(element => element > 0);
console.log(allPositive);  // true — all are positive!

let allGTE1 = numbers.every(element => element >= 1);
console.log(allGTE1);  // true — all are ≥ 1
⚠

some vs every:
some = "At least ONE?" (like logical OR) → returns true if ANY element matches.
every = "ALL of them?" (like logical AND) → returns true only if EVERY element matches.

i

Interview favorites: "What's the difference between map and filter?" → map transforms, filter selects. "map vs forEach?" → map returns new array, forEach returns nothing. "some vs every?" → some = at least one, every = all.

Quick Reference Cheatsheet

forEach — do something, return nothing

fruits.forEach(item => console.log(item));

map — transform each element, get new array

let doubled = nums.map(n => n * 2);        // [2,4,6,8,10]

filter — keep elements that pass the test

let evens = nums.filter(n => n % 2 === 0);  // [2,4]

reduce — accumulate into single value

let sum = nums.reduce((acc, n) => acc + n, 0);  // 15

some — at least one passes?

let hasNeg = nums.some(n => n < 0);        // false

every — do ALL pass?

let allPos = nums.every(n => n > 0);       // true

Arrow function shortcut

// If only ONE return statement inside arrow function:
// ❌ Verbose
nums.map((n) => { return n * 2; });
// ✅ Concise — remove { } and 'return'
nums.map(n => n * 2);
