What is Destructuring?

✓

Real-Life Example: Imagine you order a combo meal -- it comes in one box with a burger, fries, and a drink. Destructuring is like opening the box and putting each item on the table separately. Instead of reaching into the box every time you want something, you lay everything out at once.

Without destructuring -- you access each value one at a time:

!
Old way -- repetitive and verbose
❌ Verbose 
const person = { name: "Alice", age: 25, city: "Mumbai" };

// Old way  --  access one by one
const name = person.name;
const age = person.age;
const city = person.city;
// Three lines for three values!
✓
New way -- destructuring in ONE line
✅ Clean 
const person = { name: "Alice", age: 25, city: "Mumbai" };

// Destructuring  --  extract all at once!
const { name, age, city } = person;
console.log(name);  // "Alice"
console.log(age);   // 25
console.log(city);  // "Mumbai"

Array Destructuring

Array Destructuring -- Extract by Position

✓

Real-Life Example: A train has seats in order -- seat 1, seat 2, seat 3. Array destructuring is like saying: "Person A gets seat 1, Person B gets seat 2." The position matters, not the name.

1
Basic array destructuring
Position-based 
const colors = ["red", "green", "blue"];

// Old way
const first = colors[0];  // "red"
const second = colors[1]; // "green"

// ✅ Destructuring way
const [a, b, c] = colors;
console.log(a);  // "red"
console.log(b);  // "green"
console.log(c);  // "blue"

// You can name variables anything:
const [primary, secondary] = colors;
console.log(primary);   // "red"
console.log(secondary); // "green"

Array destructuring uses square brackets [ ] and extracts by position (index 0, 1, 2...).

Array Destructuring

Skipping Elements

2
Skip elements with empty commas
Skip trick 
const scores = [90, 85, 78, 92, 88];

// Skip first two, get only the third
const [, , third] = scores;
console.log(third);  // 78

// Get first and third, skip second
const [gold, , bronze] = scores;
console.log(gold);   // 90
console.log(bronze); // 78

// Swap two variables WITHOUT a temp variable!
let x = 10, y = 20;
[x, y] = [y, x];
console.log(x);  // 20
console.log(y);  // 10 ← Swapped!

Each comma , skips one element. The swap trick is a classic interview question!

Array Destructuring

Default Values

3
Defaults when array is too short
Fallback 
const pair = [10];

// Without default  --  b is undefined
const [a, b] = pair;
console.log(a);  // 10
console.log(b);  // undefined 😬

// ✅ With default  --  b gets fallback value
const [x, y = 0] = pair;
console.log(x);  // 10
console.log(y);  // 0 ← default used!

// Default is ONLY used when value is undefined
const [m, n = 99] = [5, 20];
console.log(n);  // 20 (NOT 99  --  actual value exists)

Default values kick in ONLY when the position is undefined. If a value exists (even null or 0), the default is ignored.

Object Destructuring

Object Destructuring -- Extract by Name

✓

Real-Life Example: A hotel check-in form has labeled fields -- Name, Phone, Email. Object destructuring is like saying: "Give me just the Name and Email fields." You pick by label name, not by position.

1
Basic object destructuring
Name-based 
const user = { name: "Bob", age: 30, role: "admin" };

// ✅ Destructuring  --  variable names MUST match property names
const { name, age, role } = user;
console.log(name);  // "Bob"
console.log(age);   // 30
console.log(role);  // "admin"

// You can pick only what you need:
const { name: userName } = user;  // rename (see next section)
const { role: r } = user;

// With defaults (when property might not exist):
const { email = "none" } = user;
console.log(email);  // "none" (user has no email property)

Object destructuring uses curly braces { } and extracts by property name. Order doesn't matter!

Object Destructuring

Renaming While Destructuring

2
Give a different name with :
Alias 
const product = { name: "Laptop", price: 50000 };

// Problem: 'name' might conflict with another variable
// Solution: rename with colon
const { name: productName, price: cost } = product;
console.log(productName);  // "Laptop"
console.log(cost);         // 50000

// ⚠ After renaming, the ORIGINAL name doesn't work:
// console.log(name);  // ❌ Error  --  'name' was not created
// console.log(price); // ❌ Error  --  'price' was not created

// Rename + Default together:
const { discount: disc = 0 } = product;
console.log(disc);  // 0 (product has no 'discount')

Syntax: { originalName: newName }. The colon here means "take originalName and call it newName."

Object Destructuring

Nested Destructuring

3
Destructure objects inside objects
Deep extract 
const employee = {
  name: "Alice",
  address: {
    city: "Mumbai",
    zip: "400001"
  }
};

// ✅ Nested destructuring  --  reach INSIDE the inner object
const { name, address: { city, zip } } = employee;
console.log(name);  // "Alice"
console.log(city);  // "Mumbai"
console.log(zip);   // "400001"

// ⚠ 'address' itself is NOT available as a variable!
// console.log(address);  // ❌ Error

// If you want BOTH the nested values AND the parent:
const { name: n, address } = employee;
const { city: c } = address;
console.log(n);        // "Alice"
console.log(address);  // { city: "Mumbai", zip: "400001" }
console.log(c);        // "Mumbai"
Advanced

Destructuring in Function Parameters

✓

Real-Life Example: Instead of handing someone a sealed envelope and saying "open it and read the name inside," you just hand them the name directly. Destructuring in parameters unwraps the data before the function even starts!

1
Destructure right in the parameter
Clean APIs 
// ❌ Old way  --  access properties inside function
function greetOld(person: { name: string; age: number }) {
  console.log(`Hi ${person.name}, you are ${person.age}`);
}

// ✅ Destructured  --  extract in the parameter itself
function greet({ name, age }: { name: string; age: number }) {
  console.log(`Hi ${name}, you are ${age}`);
}

greet({ name: "Alice", age: 25 });  // "Hi Alice, you are 25"

// With type alias  --  even cleaner:
type Person = { name: string; age: number };

function greetClean({ name, age }: Person) {
  console.log(`Hi ${name}, you are ${age}`);
}

// Works with arrays too:
function getFirst([first, second]: number[]) {
  return first;
}
console.log(getFirst([10, 20, 30]));  // 10
Advanced

Destructuring with Rest (...)

1
Collect remaining items with ...
...rest 
// Array rest  --  grab the first, collect the rest
const [winner, ...losers] = ["Gold", "Silver", "Bronze", "4th"];
console.log(winner);  // "Gold"
console.log(losers);  // ["Silver", "Bronze", "4th"]

// Object rest  --  pick some, collect the rest
const user = { name: "Bob", age: 30, role: "admin", city: "Delhi" };
const { name, ...remaining } = user;
console.log(name);       // "Bob"
console.log(remaining);  // { age: 30, role: "admin", city: "Delhi" }

// Great for removing a property:
const { password, ...safeUser } = { name: "Alice", password: "secret", email: "a@b.com" };
console.log(safeUser);  // { name: "Alice", email: "a@b.com" }
// password is extracted and thrown away  --  safeUser is clean!

...rest must always be the LAST item. You can't put anything after it.

Watch Out!

⚠ Common Beginner Mistakes

⚠
Mistake 1: Destructuring null or undefined -- crashes!

If the source is null or undefined, destructuring throws a TypeError. Always check first! 

// ❌ CRASH  --  cannot destructure null
const data = null;
const { name } = data;  // TypeError: Cannot destructure property 'name' of null

// ✅ SAFE  --  use default empty object
const { name = "unknown" } = data || {};
console.log(name);  // "unknown"

// ✅ In function parameters  --  default to empty object
function process({ id, name }: any = {}) {
  console.log(id, name);
}
process();  // undefined undefined (no crash!)

Rule: Always provide a fallback with || {} or = {} when the source might be null/undefined.

⚠
Mistake 2: Variable names MUST match property names (objects)

In object destructuring, the variable name must exactly match the property name. If you misspell it, you get undefined -- no error! 

const user = { name: "Alice", age: 25 };

// ❌ WRONG  --  'username' doesn't exist in user
const { username } = user;
console.log(username);  // undefined (silent bug!)

// ✅ CORRECT  --  name matches exactly
const { name } = user;
console.log(name);  // "Alice"

// ✅ If you WANT a different name, use renaming:
const { name: username2 } = user;
console.log(username2);  // "Alice"

Rule: Object destructuring matches by NAME, not position. Typos give undefined, not errors!

⚠
Mistake 3: Confusing : for type vs rename

In TypeScript, : is used for BOTH type annotation AND renaming in destructuring. This confuses beginners! 

// This is RENAMING (destructuring syntax):
const { name: fullName } = { name: "Alice" };
// fullName = "Alice", name doesn't exist

// This is TYPE ANNOTATION (TypeScript syntax):
const { name }: { name: string } = { name: "Alice" };
// name = "Alice", with type string

// ❌ WRONG  --  mixing them up:
// const { name: string } = { name: "Alice" };
// This RENAMES 'name' to 'string', doesn't set a type!

Rule: { prop: newName } = renaming. { prop }: { prop: Type } = typing. Don't mix them!

⚠
Mistake 4: Nested destructuring -- parent variable is NOT created

When you destructure nested properties, the parent object is NOT available as a variable unless you explicitly extract it too. 

const config = { db: { host: "localhost", port: 5432 } };

// ❌ WRONG  --  'db' is NOT a variable here!
const { db: { host, port } } = config;
console.log(host);  // "localhost" ✓
console.log(db);    // ❌ ReferenceError: db is not defined

// ✅ If you need 'db' too, destructure separately:
const { db } = config;
const { host: h, port: p } = db;
console.log(db);    // { host: "localhost", port: 5432 } ✓
console.log(h);     // "localhost" ✓

Rule: In { parent: { child } }, only child is created. parent is used as a path, not a variable.

Array vs Object Destructuring

Feature Array [ ] Object { }
Extracts by Position Property name
Order matters? Yes! No
Variable names Any name you want Must match property (or rename)
Skip items Empty commas [, , x] Just don't list them
Defaults [a = 10] { x = 10 }
Renaming Not needed (any name works) { name: newName }
Rest [first, ...rest] { name, ...rest }
Nesting [[a, b], [c, d]] { a: { b } }

Quick Reference Cheatsheet

Array Destructuring

const [a, b, c] = [1, 2, 3];         // Basic
const [x, , z] = [1, 2, 3];          // Skip second
const [a, b = 0] = [1];              // Default value
const [first, ...rest] = [1, 2, 3];  // Rest
let [a, b] = [1, 2]; [a, b] = [b, a]; // Swap

Object Destructuring

const { name, age } = person;              // Basic
const { name: fullName } = person;         // Rename
const { email = "none" } = person;         // Default
const { addr: { city } } = person;         // Nested
const { name, ...rest } = person;          // Rest

In Function Parameters

function greet({ name, age }: Person) { }  // Object param
function first([a]: number[]) { return a; } // Array param
