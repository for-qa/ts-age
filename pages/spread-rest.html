Spread & Rest Operators

The same ... syntax used for two opposite jobs — spreading values OUT and collecting values IN. Used everywhere in modern TypeScript.

Spread Operator

What is the Spread Operator?

✓

Real-Life Example: Imagine you have a jar of beads. The spread operator is like pouring all the beads onto the table — they're no longer inside the jar, they're spread out individually. You can then pick them up and put them into a new jar (a new array or object).

...
Spread = EXPAND
Takes items OUT of a container and lays them flat
...
Rest = COLLECT
Gathers remaining items INTO a container
!
Same syntax, opposite jobs
Key Concept 
// SPREAD — expanding items OUT
const arr = [1, 2, 3];
const copy = [...arr];        // [1, 2, 3] — poured out, then into new array

// REST — collecting items IN
function sum(...nums: number[]) {  // collects ALL arguments into one array
  return nums.reduce((a, b) => a + b, 0);
}
sum(1, 2, 3, 4);  // 10

Rule of thumb: On the LEFT side of = or in function params → Rest. On the RIGHT side of = or in function calls → Spread.

Spread Operator

Spread with Arrays

1
Merge, clone, and insert
Arrays 
const fruits = ["apple", "banana"];
const veggies = ["carrot", "pea"];

// ✅ Merge two arrays
const food = [...fruits, ...veggies];
// ["apple", "banana", "carrot", "pea"]

// ✅ Clone an array (not the same reference!)
const fruitsCopy = [...fruits];
fruitsCopy.push("mango");
console.log(fruits);      // ["apple", "banana"] — original untouched!
console.log(fruitsCopy);  // ["apple", "banana", "mango"]

// ✅ Insert in the middle
const nums = [1, 2, 5, 6];
const complete = [1, 2, ...([3, 4]), 5, 6];
// [1, 2, 3, 4, 5, 6]

// ✅ Add to front or back
const withStart = ["zero", ...nums];   // ["zero", 1, 2, 5, 6]
const withEnd = [...nums, 7, 8];       // [1, 2, 5, 6, 7, 8]
Spread Operator

Spread with Objects

✓

Real-Life Example: Think of it like photocopying a form and then writing over some fields. {...original, name: "New"} copies all fields, then overwrites just the name.

2
Merge, clone, and override
Objects 
const defaults = { theme: "dark", lang: "en", font: 14 };
const userPrefs = { lang: "hi", font: 18 };

// ✅ Merge — user overrides defaults (LAST ONE WINS)
const settings = { ...defaults, ...userPrefs };
// { theme: "dark", lang: "hi", font: 18 }
//   ↑ kept from defaults   ↑ overwritten by user

// ✅ Clone an object
const clone = { ...defaults };

// ✅ Add new properties
const extended = { ...defaults, notifications: true };
// { theme: "dark", lang: "en", font: 14, notifications: true }

// ✅ Override a specific field
const updated = { ...defaults, theme: "light" };
// { theme: "light", lang: "en", font: 14 }

Order matters! Properties from the LAST spread overwrite earlier ones. {...a, ...b} → b wins on conflicts.

Spread Operator

Shallow Copy — The Trap!

!
Spread only copies ONE level deep
⚠ Trap 
const original = {
  name: "Alice",
  scores: [90, 85, 78]  // nested array = reference type!
};

const copy = { ...original };

// Changing a primitive — safe, independent
copy.name = "Bob";
console.log(original.name);  // "Alice" ✅ untouched

// Changing nested data — DANGER! SHARED reference
copy.scores.push(100);
console.log(original.scores);  // [90, 85, 78, 100] 😱 MUTATED!

// ✅ FIX — deep copy nested values manually
const safeCopy = {
  ...original,
  scores: [...original.scores]  // spread the inner array too!
};
safeCopy.scores.push(200);
console.log(original.scores);  // [90, 85, 78] ✅ safe!

Spread copies values for primitives but references for objects/arrays. To truly copy nested data, spread each level or use structuredClone().

Spread Operator

Spread in Function Arguments

3
Pass array items as separate arguments
Calling 
const numbers = [5, 12, 3, 8, 1];

// ❌ Without spread — doesn't work!
// Math.max(numbers);  // NaN — expects individual numbers, not an array

// ✅ With spread — unpacks the array into separate arguments
console.log(Math.max(...numbers));  // 12
console.log(Math.min(...numbers));  // 1

// Another example
function greet(first: string, last: string) {
  console.log(`Hello ${first} ${last}`);
}
const names: [string, string] = ["Alice", "Smith"];
greet(...names);  // "Hello Alice Smith"
Rest Operator

Rest in Function Parameters

✓

Real-Life Example: Rest is like a bag at a buffet — "I'll take the first two items individually, and throw everything else into this bag." The bag (...rest) holds whatever is left.

1
Collect unlimited arguments
...rest 
// ✅ Accept any number of arguments
function sum(...nums: number[]): number {
  return nums.reduce((total, n) => total + n, 0);
}
console.log(sum(1, 2));           // 3
console.log(sum(1, 2, 3, 4, 5));  // 15
console.log(sum());               // 0

// ✅ Mix regular params + rest (rest MUST be last!)
function introduce(greeting: string, ...names: string[]) {
  names.forEach(name => console.log(`${greeting}, ${name}!`));
}
introduce("Hello", "Alice", "Bob", "Charlie");
// "Hello, Alice!"
// "Hello, Bob!"
// "Hello, Charlie!"

// ✅ TypeScript knows the types!
function log(level: string, ...messages: string[]) {
  messages.forEach(m => console.log(`[${level}] ${m}`));
}

...rest collects all remaining arguments into a real array (not the old arguments object).

Rest Operator

Rest in Destructuring

2
Collect remaining items
Destructuring 
// ✅ Array — grab first, collect the rest
const [gold, silver, ...others] = ["India", "USA", "UK", "Japan", "France"];
console.log(gold);    // "India"
console.log(silver);  // "USA"
console.log(others);  // ["UK", "Japan", "France"]

// ✅ Object — pick some, collect the rest
const user = { name: "Alice", age: 25, role: "admin", city: "Mumbai" };
const { name, ...profile } = user;
console.log(name);     // "Alice"
console.log(profile);  // { age: 25, role: "admin", city: "Mumbai" }

// ✅ Practical — remove sensitive data
const fullUser = { id: 1, password: "secret", email: "a@b.com", name: "Bob" };
const { password, ...safeUser } = fullUser;
console.log(safeUser);  // { id: 1, email: "a@b.com", name: "Bob" }
// 'password' is extracted and discarded — safeUser is clean!
Watch Out!

⚠ Common Beginner Mistakes

⚠
Mistake 1: Spread only does SHALLOW copy

Nested objects/arrays are still shared between original and copy. Changing nested data in the copy changes the original too! 

// ❌ WRONG — nested array is shared
const original = { name: "A", tags: ["ts", "js"] };
const copy = { ...original };
copy.tags.push("react");
console.log(original.tags);  // ["ts", "js", "react"] 😱

// ✅ CORRECT — spread nested values too
const safeCopy = { ...original, tags: [...original.tags] };
safeCopy.tags.push("react");
console.log(original.tags);  // ["ts", "js"] ✅

Rule: Always spread nested arrays/objects separately, or use structuredClone(obj) for a true deep copy.

⚠
Mistake 2: Object spread — LAST one wins (silent overwrite)

When merging objects, later properties silently overwrite earlier ones with the same key. No error, no warning! 

const config1 = { port: 3000, host: "localhost" };
const config2 = { port: 8080, debug: true };

// ❌ Surprise — port got overwritten silently!
const merged = { ...config1, ...config2 };
console.log(merged.port);  // 8080 (NOT 3000!)

// ✅ Be intentional — put important values LAST
const safe = { ...config2, ...config1 };
console.log(safe.port);  // 3000 (config1 wins)

Rule: In {...a, ...b}, b wins on duplicate keys. Put your priority source LAST.

⚠
Mistake 3: Spreading a string — surprise characters!

Strings are iterable, so spreading them gives individual characters, not what you might expect. 

// ❌ Surprise!
const word = "hello";
const chars = [...word];
console.log(chars);  // ["h", "e", "l", "l", "o"]

// This is technically correct — but often unexpected!
// Be careful when spreading variables that might be strings.

Rule: [...string] splits into characters. If you want the string in an array, use [string] (no spread).

⚠
Mistake 4: Rest must be the LAST parameter

You cannot put anything after a rest parameter — it collects everything remaining. 

// ❌ WRONG — rest is not last
// function bad(...nums: number[], label: string) { }
// Error: A rest parameter must be last in a parameter list.

// ✅ CORRECT — rest is last
function good(label: string, ...nums: number[]) {
  console.log(label, nums);
}
good("scores", 90, 85, 78);
// "scores" [90, 85, 78]

Rule: ...rest is always the LAST parameter — it's greedy and grabs everything.

⚠
Mistake 5: Confusing spread vs rest — same symbol, opposite jobs 
// SPREAD (right side / call site) — expands OUT
const merged = [...arr1, ...arr2];  // spread
Math.max(...numbers);               // spread

// REST (left side / definition) — collects IN
const [first, ...rest] = arr;                 // rest
function fn(...args: number[]) { }            // rest

Rule: Spread = unpacking a suitcase. Rest = packing leftovers into a bag.

Mistake What happens Fix
Shallow copy trap Nested data is shared Spread nested levels or structuredClone()
Last-one-wins overwrite Silent property loss Put priority source LAST
Spreading strings Gets char array Use [str] not [...str]
Rest not last Syntax error Always put ...rest last
Spread vs Rest confusion Wrong usage Right side = spread, left side = rest

Spread vs Rest — Side by Side

Feature Spread ... Rest ...
Purpose Expand / Unpack Collect / Gather
Where used Right side of =, function calls Left side of =, function params
With arrays [...arr1, ...arr2] const [a, ...rest] = arr
With objects {...obj1, ...obj2} const {a, ...rest} = obj
Functions fn(...args) (call) fn(...args) (define)
Analogy Pouring beads out of jar Sweeping beads into bag
Position Anywhere Must be LAST

Quick Reference Cheatsheet

Spread (Expand)

[...arr1, ...arr2]          // Merge arrays
{...obj1, ...obj2}          // Merge objects (last wins)
[...arr]                    // Clone array (shallow)
{...obj}                    // Clone object (shallow)
Math.max(...numbers)        // Pass array as arguments
fn(...args)                 // Unpack into function call

Rest (Collect)

function fn(...args: T[])       // Collect all args
function fn(a, ...rest: T[])   // Collect remaining args
const [first, ...rest] = arr   // Destructure + collect
const {a, ...rest} = obj       // Destructure + collect

Deep Copy

const deep = structuredClone(original);  // True deep copy
// OR manually spread each level:
const safe = { ...obj, nested: { ...obj.nested } };
