Callbacks

What is a Callback Function?

✓

Real-Life Example: A callback is like leaving your phone number with a restaurant. You say "Call me when my table is ready." You don't wait at the door — you go do other things, and THEY call YOU back when it's time. The restaurant (main function) calls your number (callback) when it's ready.

A callback function is a function passed as an argument to another function, which gets executed later — not immediately.

Normal call vs callback

// ❌ Normal — calling directly (immediate)
showMessage("Hello!");  // executes RIGHT NOW

// ✅ Callback — passing the function to someone else
greet("John", showMessage);  // greet decides WHEN to call showMessage
TermMeaning
ArgumentsValues passed INTO a function call
ParametersVariables that RECEIVE those values
CallbackA function passed as an argument to another function
Example

Callback Example — Greeting

1
Pass a function as an argument
Callback
// Step 1: Create the callback function
function showMessage(message: string) {
  console.log(message);
}

// Step 2: Create a function that ACCEPTS a callback
function greet(
  name: string,
  callback: (msg: string) => void   // ← callback type definition
) {
  console.log(name);        // do something first
  callback("Hello!");       // THEN call the callback
}

// Step 3: Call greet, passing showMessage as a callback
greet("John", showMessage);

// Output:
// John
// Hello!

We didn't call showMessage directly — we passed it INTO greet, and greet decided when to call it.

i

Callback type syntax: callback: (param: type) => returnType
This tells TypeScript: "this parameter is a function that takes param and returns returnType."

Example

Callback Example — Sum with Display

2
Compute, then pass result to callback
Callback
// Callback function — just displays the result
function displayResult(result: number) {
  console.log("Result:", result);
}

// Main function — does calculation, then calls the callback
function sum(
  a: number,
  b: number,
  callback: (result: number) => void
) {
  let result = a + b;       // do the work first
  callback(result);          // THEN pass result to callback
}

// Call it — passing displayResult as the callback
sum(10, 20, displayResult);

// Output: Result: 30

The sum function does the math, then hands the result to the callback. The callback decides what to do with it (here: print it).

Why Callbacks?

Synchronous vs Asynchronous

✓

Real-Life Example:
✓ Synchronous = A single checkout line — each person waits for the one in front. Everything happens in order.
⚡ Asynchronous = A food court — you order, walk away, do other things, and they call your number when food is ready. Tasks run in parallel.

FeatureSynchronousAsynchronous
ExecutionOne after another (sequential)Tasks run in parallel
WaitingEach step waits for the previousDoesn't wait — moves to next task
ProblemSlow if one step takes longOrder of completion is unpredictable
Solution—Callbacks, async/await, Promises
⚠

JavaScript/TypeScript is asynchronous by nature — this is its biggest challenge compared to languages like Java. Callbacks help enforce execution order when needed. In automation (Playwright), we use async/await instead — a cleaner alternative covered later.

Overloading

What is Function Overloading?

✓

Real-Life Example: Overloading is like a Swiss Army knife — ONE tool with MANY versions. The same knife (function name) can be a blade, scissors, or screwdriver depending on what you pull out (parameters). One name, multiple behaviors.

Function overloading lets you define multiple versions of a function with the same name but different parameters or return types.

What can be different?Allowed?
Parameter types (number vs string)✅ Yes
Number of parameters (2 vs 3)✅ Yes
Return types (string vs number)✅ Yes
Function name❌ Must be same!
Identical parameter types❌ Not allowed
How To

The 3-Step Process

①
Write Signatures
Function declarations without body (just semicolons)
②
Implement ONE Function
Single implementation that satisfies ALL signatures
③
Call the Function
Pass different params — the matching signature activates
⚠

Critical Rule: No matter how many signatures you write, there must be only ONE implementation that handles ALL of them. You don't write separate bodies for each signature!

Example 1

Overloading — Different Parameter Types

1
getInfo — accepts number OR string
Overload
// Step 1: SIGNATURES (no body — just semicolons!)
function getInfo(id: number): string;
function getInfo(name: string): string;

// Step 2: ONE IMPLEMENTATION (handles both)
function getInfo(param: number | string): string {
  if (typeof param === "number") {
    return `User ID is ${param}`;
  } else {
    return `Username is ${param}`;
  }
}

// Step 3: CALL with different types
console.log(getInfo(101));     // "User ID is 101"
console.log(getInfo("John"));  // "Username is John"

The typeof operator checks if param is a number or string, then runs the right logic.

Example 2

Overloading — Different Number of Parameters

2
add — 2 or 3 numbers
Optional param
// Signatures
function add(a: number, b: number): number;
function add(a: number, b: number, c: number): number;

// Implementation — c is OPTIONAL to satisfy both
function add(a: number, b: number, c?: number): number {
  if (c !== undefined) {
    return a + b + c;    // 3 params passed
  }
  return a + b;           // only 2 params passed
}

console.log(add(10, 20));       // 30
console.log(add(10, 20, 30));   // 60

Make the extra parameter optional (?) in the implementation so it satisfies both 2-param and 3-param signatures.

Example 3

Overloading — Different Return Types

3
processInput — returns string OR number
Union return
// Signatures — different param types AND return types!
function processInput(str: string): string;
function processInput(num: number): number;

// Implementation — union for both param and return
function processInput(input: string | number): string | number {
  if (typeof input === "string") {
    return input.toUpperCase();  // string → uppercase
  } else {
    return input * input;        // number → square
  }
}

console.log(processInput("hello"));  // "HELLO"
console.log(processInput(5));        // 25
4
greet — string, number, or boolean!
3 signatures
// 3 signatures!
function greet(name: string): string;
function greet(age: number): string;
function greet(isMarried: boolean): string;

// 1 implementation handles all 3
function greet(value: string | number | boolean): string {
  if (typeof value === "string") {
    return `Hello ${value}`;
  } else if (typeof value === "number") {
    return `You are ${value} years old`;
  } else {
    let result: string = value ? "Married" : "Single";
    return `You are ${result}`;
  }
}

console.log(greet("John"));   // "Hello John"
console.log(greet(30));        // "You are 30 years old"
console.log(greet(true));      // "You are Married"
console.log(greet(false));     // "You are Single"
Watch Out!

Invalid Overloading — Common Mistakes

❌
These will NOT work!
Errors
// ❌ 1. No implementation — just signatures
function test(a: number): string;
function test(a: string): string;
// ERROR: no function body!

// ❌ 2. Wrong return type in implementation
function calc(a: number): number;
function calc(a: string): string;
function calc(a: number | string): boolean { } // ← boolean ≠ number|string

// ❌ 3. Identical parameter types
function fn(a: number): number;
function fn(a: number): string;   // ← same param type!
// ERROR: "Cannot overload identical signatures"

// ❌ 4. Incompatible return type
function fetchData(url: string): string;
function fetchData(id: number): string;
function fetchData(url: string | number): number { } // ← should be string!
RuleValid?
Same name, different param types✅
Same name, different number of params✅
Same name, different return types✅
Same name, identical param types❌ Error
Missing implementation body❌ Error
Implementation return type doesn't match❌ Error

Quick Reference Cheatsheet

Callback function

// Define a callback
function logResult(result: number) {
  console.log("Result:", result);
}

// Function that accepts a callback
function compute(a: number, b: number, cb: (r: number) => void) {
  cb(a + b);
}

compute(5, 10, logResult);  // "Result: 15"

Function overloading

// Step 1: Signatures
function fn(a: number): number;
function fn(a: string): string;

// Step 2: ONE implementation
function fn(a: number | string): number | string {
  if (typeof a === "number") return a * 2;
  return a.toUpperCase();
}

// Step 3: Call
fn(5);       // 10
fn("hello"); // "HELLO"

Key rules

// ✅ Callbacks = pass function as argument, executed later
// ✅ Callback type: (param: type) => returnType
// ✅ Overloading = same name, different params/returns
// ✅ Only ONE implementation for ALL signatures
// ✅ Use typeof to check param types in implementation
// ✅ Use union types (|) in implementation to match all signatures
// ⚠ In automation: prefer async/await over callbacks
