Built-in Type Guards

0
Quick recap — typeof, instanceof, in
Recap 
// typeof — for primitives
function process(value: string | number) {
  if (typeof value === "string") {
    console.log(value.toUpperCase());  // TS knows it's string
  } else {
    console.log(value.toFixed(2));     // TS knows it's number
  }
}

// instanceof — for classes
function handleError(err: Error | TypeError) {
  if (err instanceof TypeError) {
    console.log("Type error:", err.message);
  }
}

// in — check if property exists
interface Dog { bark(): void; }
interface Cat { meow(): void; }
function speak(pet: Dog | Cat) {
  if ("bark" in pet) {
    pet.bark();   // TS knows it's Dog
  } else {
    pet.meow();   // TS knows it's Cat
  }
}

Custom Type Guards with is

✓

Real-Life: A custom type guard is like a bouncer at a club 🚪. You define the rules ("must have name AND age"), and the bouncer checks each person. If they pass, TypeScript treats them as the correct type inside the if block.

1
Tell TypeScript: "if this returns true, the type is X"
is Predicate 
interface User {
  name: string;
  age: number;
  email: string;
}

// Custom type guard — returns "value is User"
function isUser(value: unknown): value is User {
  return (
    typeof value === "object" &&
    value !== null &&
    "name" in value &&
    "age" in value &&
    "email" in value &&
    typeof (value as User).name === "string" &&
    typeof (value as User).age === "number"
  );
}

// Now TypeScript narrows automatically!
function processData(data: unknown) {
  if (isUser(data)) {
    // TypeScript KNOWS data is User here! ✅
    console.log(data.name);   // autocomplete works!
    console.log(data.email);  // no errors!
  } else {
    console.log("Not a valid user");
  }
}

// Works great with filter:
const mixed: unknown[] = [{ name: "A", age: 1, email: "a@b" }, "not user", 42];
const users = mixed.filter(isUser);
// users is User[] — TypeScript knows! ✅

Discriminated Unions

✓

Real-Life: Like a package label 📦. Every box has a label saying "fragile", "heavy", or "liquid". You check the label to know how to handle it. The type field is the label.

2
A shared property that identifies the variant
Discriminant 
// Each shape has a 'kind' property — the discriminant
interface Circle {
  kind: "circle";     // literal type — the label!
  radius: number;
}
interface Rectangle {
  kind: "rectangle";
  width: number;
  height: number;
}
interface Triangle {
  kind: "triangle";
  base: number;
  height: number;
}

type Shape = Circle | Rectangle | Triangle;

function area(shape: Shape): number {
  switch (shape.kind) {
    case "circle":
      // TS knows shape is Circle here!
      return Math.PI * shape.radius ** 2;
    case "rectangle":
      // TS knows shape is Rectangle!
      return shape.width * shape.height;
    case "triangle":
      return 0.5 * shape.base * shape.height;
  }
}
Advanced

Exhaustive Checking with never

3
Catch missing cases at COMPILE time
never 
// If you add a new shape but forget to handle it,
// TypeScript will ERROR at compile time!

function assertNever(value: never): never {
  throw new Error(`Unhandled case: ${value}`);
}

function area(shape: Shape): number {
  switch (shape.kind) {
    case "circle":
      return Math.PI * shape.radius ** 2;
    case "rectangle":
      return shape.width * shape.height;
    case "triangle":
      return 0.5 * shape.base * shape.height;
    default:
      return assertNever(shape);
      // If you add "pentagon" to Shape but forget a case,
      // TypeScript will error HERE: "Type 'Pentagon' is not
      // assignable to type 'never'" ✅
  }
}

// This guarantees you handle EVERY variant!

Why it works: After all cases are handled, shape should be type never (impossible). If it's not never, you missed a case.

Assertion Functions

4
Throw if wrong type — narrow if correct
asserts 
// Unlike 'is' guards (which return boolean),
// assertion functions THROW if the check fails

function assertIsString(value: unknown): asserts value is string {
  if (typeof value !== "string") {
    throw new Error(`Expected string, got ${typeof value}`);
  }
}

function processInput(input: unknown) {
  assertIsString(input);
  // If we get here, TypeScript KNOWS input is string! ✅
  console.log(input.toUpperCase());  // no error!
}

// Non-null assertion:
function assertDefined<T>(value: T | null | undefined): asserts value is T {
  if (value == null) {
    throw new Error("Value is null or undefined");
  }
}

const el = document.querySelector("#app");  // Element | null
assertDefined(el);
el.textContent = "Hello";  // TS knows el is Element ✅

API Response Pattern

5
Discriminated union for API results
Real-World 
// Common pattern: API responses with success/error variants
interface SuccessResponse<T> {
  status: "success";
  data: T;
}
interface ErrorResponse {
  status: "error";
  message: string;
  code: number;
}

type ApiResponse<T> = SuccessResponse<T> | ErrorResponse;

interface User { id: number; name: string; }

function handleResponse(response: ApiResponse<User>) {
  switch (response.status) {
    case "success":
      // TS knows: response.data exists and is User
      console.log(`Welcome, ${response.data.name}!`);
      break;
    case "error":
      // TS knows: response.message and response.code exist
      console.error(`Error ${response.code}: ${response.message}`);
      break;
  }
}

// Works perfectly with fetch:
async function fetchUser(id: number): Promise<ApiResponse<User>> {
  try {
    const res = await fetch(`/api/users/${id}`);
    const data = await res.json();
    return { status: "success", data };
  } catch {
    return { status: "error", message: "Network error", code: 500 };
  }
}
Watch Out!

⚠ Common Mistakes

⚠
Mistake 1: typeof null === "object" 
// ❌ This doesn't guard against null!
function isObj(val: unknown): val is object {
  return typeof val === "object";  // null passes! 😱
}

// ✅ Always check for null too
function isObj2(val: unknown): val is object {
  return typeof val === "object" && val !== null;
}
⚠
Mistake 2: Missing return/break in discriminated switch 
// ❌ Forgot return — falls through!
function area(s: Shape): number {
  switch (s.kind) {
    case "circle":
      Math.PI * s.radius ** 2;  // no return!
    case "rectangle":
      return s.width * s.height;
  }
}

// ✅ Always return or break
// Enable "noFallthroughCasesInSwitch" in tsconfig.json
⚠
Mistake 3: No default/never case — new types go unchecked 
// ❌ If you add "pentagon" to Shape, no error!
switch (shape.kind) {
  case "circle": ...
  case "rectangle": ...
  // pentagon silently ignored!
}

// ✅ Add assertNever in default
default: return assertNever(shape);
// Now adding pentagon without a case → compile error!
Mistake What happens Fix
typeof null null passes as object Add && val !== null
Missing return Falls through cases Always return/break
No never check New types unhandled assertNever() in default
