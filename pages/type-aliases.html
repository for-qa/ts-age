Quick Reference Cheatsheet

Type aliases

type Age = number;                          // primitive alias
type Point = { x: number; y: number };      // object alias
type Callback = (msg: string) => void;      // function alias

Union types

type ID = string | number;                  // accept either type
type Status = "active" | "inactive";        // literal union
type Result = Success | Failure;            // object union

Intersection types

type Staff = Person & Employee;             // must be BOTH

Narrowing

if (typeof x === "string") { }              // primitive check
if (x instanceof MyClass) { }              // class check
if ("prop" in x) { }                       // property check

Key rules

// âœ… type = gives a new NAME to any type
// âœ… | (union) = value can be ONE of the listed types
// âœ… & (intersection) = value must satisfy ALL types
// âœ… Literal types = restrict to exact values
// âœ… Always narrow union types before using type-specific methods
// âœ… Use type for unions/intersections, interface for class contracts
