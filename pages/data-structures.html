Stack — Last In, First Out (LIFO)

✓

Real-Life: Like a stack of plates 🍽. You add plates on top and remove from the top. Last plate added = first plate removed.

1
push() and pop() — that's it!
Stack
class Stack<T> {
  private items: T[] = [];

  push(item: T): void { this.items.push(item); }
  pop(): T | undefined { return this.items.pop(); }
  peek(): T | undefined { return this.items[this.items.length - 1]; }
  isEmpty(): boolean { return this.items.length === 0; }
  get size(): number { return this.items.length; }
}

const stack = new Stack<number>();
stack.push(1);  // [1]
stack.push(2);  // [1, 2]
stack.push(3);  // [1, 2, 3]
stack.pop();    // 3 — last in, first out!
stack.peek();   // 2 — look without removing

// Real uses: undo/redo, browser back button,
// function call stack, expression parsing

Queue — First In, First Out (FIFO)

✓

Real-Life: Like a line at a ticket counter 🎟. First person in line gets served first.

2
enqueue() and dequeue()
Queue
class Queue<T> {
  private items: Map<number, T> = new Map();
  private head = 0;
  private tail = 0;

  enqueue(item: T): void { this.items.set(this.tail++, item); }
  dequeue(): T | undefined {
    if (this.isEmpty()) return undefined;
    const item = this.items.get(this.head);
    this.items.delete(this.head++);
    return item;
  }
  peek(): T | undefined { return this.items.get(this.head); }
  isEmpty(): boolean { return this.head === this.tail; }
  get size(): number { return this.tail - this.head; }
}

const q = new Queue<string>();
q.enqueue("Alice");  // [Alice]
q.enqueue("Bob");    // [Alice, Bob]
q.enqueue("Charlie");// [Alice, Bob, Charlie]
q.dequeue();         // "Alice" — first in, first out!

// ⚠ Why not use array.shift()? It's O(n)! See mistakes section.
// Real uses: task queues, print queue, BFS, message queues

LinkedList

✓

Real-Life: Like a treasure hunt 🗺. Each clue tells you where the next clue is. You can only follow the chain — no jumping to the middle.

3
Nodes linked by references
LinkedList
class ListNode<T> {
  constructor(public value: T, public next: ListNode<T> | null = null) {}
}

class LinkedList<T> {
  private head: ListNode<T> | null = null;

  prepend(value: T): void {
    this.head = new ListNode(value, this.head);
  }

  append(value: T): void {
    const node = new ListNode(value);
    if (!this.head) { this.head = node; return; }
    let current = this.head;
    while (current.next) current = current.next;
    current.next = node;
  }

  toArray(): T[] {
    const result: T[] = [];
    let current = this.head;
    while (current) {
      result.push(current.value);
      current = current.next;
    }
    return result;
  }
}

const list = new LinkedList<number>();
list.append(1);   list.append(2);   list.prepend(0);
console.log(list.toArray());  // [0, 1, 2]

// Pros: O(1) insert at head, dynamic size
// Cons: O(n) access by index, no random access

Map — Key-Value Pairs

4
Better than plain objects for dynamic keys
Map
// Map vs Object — Map is better for dynamic key-value storage
const userCache = new Map<number, string>();

userCache.set(1, "Alice");
userCache.set(2, "Bob");
userCache.get(1);        // "Alice"
userCache.has(3);        // false
userCache.delete(1);
userCache.size;          // 1

// Map advantages over objects:
// ✅ Any type as key (objects, numbers, functions)
// ✅ Maintains insertion order
// ✅ .size property (object needs Object.keys().length)
// ✅ Better performance for frequent add/remove

// Iterating:
const scores = new Map<string, number>([["math", 90], ["eng", 85]]);
for (const [key, val] of scores) {
  console.log(`${key}: ${val}`);
}

// Convert to/from object:
const obj = Object.fromEntries(scores);
const map = new Map(Object.entries(obj));

Set — Unique Values Only

5
No duplicates allowed
Set
const tags = new Set<string>();
tags.add("typescript");
tags.add("javascript");
tags.add("typescript");  // ignored — already exists!
console.log(tags.size);  // 2

tags.has("typescript");  // true
tags.delete("javascript");

// Classic use: remove duplicates from array
const nums = [1, 2, 3, 2, 1, 4, 3];
const unique = [...new Set(nums)];  // [1, 2, 3, 4]

// Set operations:
const a = new Set([1, 2, 3]);
const b = new Set([2, 3, 4]);

const union = new Set([...a, ...b]);            // {1,2,3,4}
const intersection = new Set([...a].filter(x => b.has(x))); // {2,3}
const diff = new Set([...a].filter(x => !b.has(x)));        // {1}

Watch Out!

⚠ Common Mistakes

⚠
Mistake 1: Using array.shift() as a queue — it's O(n)! 
// ❌ shift() re-indexes the ENTIRE array!
const q = [1, 2, 3, 4, 5];
q.shift();  // O(n) — moves every element left

// ✅ Use a Map-based queue (see Queue class above)
// or a linked list for O(1) dequeue
⚠
Mistake 2: Using object when Map is better 
// ❌ Object keys are always strings
const cache: Record<string, any> = {};
cache[1] = "one";      // key becomes "1" (string!)
cache[{}] = "object";  // key becomes "[object Object]" 😱

// ✅ Map preserves key types
const map = new Map();
map.set(1, "one");     // key stays number
map.set({}, "object"); // key stays object reference
⚠
Mistake 3: Not using Set for uniqueness checks 
// ❌ O(n) check with array
const seen: string[] = [];
if (!seen.includes(item)) seen.push(item);  // O(n) per check!

// ✅ O(1) check with Set
const seen2 = new Set<string>();
seen2.add(item);  // auto-deduplicates, O(1)
