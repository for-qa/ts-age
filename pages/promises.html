Basics

What is a Promise?

✓

Real-Life Example: You order food on Swiggy/Zomato. You get a tracking number (= the promise). The food is either delivered (= resolved/fulfilled) or the order is cancelled (= rejected). You don't stand at the door waiting — you do other things while the order is being processed.

!
A Promise = a placeholder for a future value
Core Concept 
// A promise represents a value that doesn't exist YET
// but WILL exist in the future (or it will fail trying)

const order = new Promise<string>((resolve, reject) => {
  // Simulating a 2-second delivery
  setTimeout(() => {
    const success = true;
    if (success) {
      resolve("🍕 Pizza delivered!");  // Fulfilled!
    } else {
      reject("❌ Order cancelled");    // Rejected!
    }
  }, 2000);
});

// Right now, 'order' is PENDING — the pizza is on its way
console.log(order);  // Promise { <pending> }
Basics

The 3 States of a Promise

⏳
Pending
Waiting — operation in progress
✅
Fulfilled
Success — value is ready
❌
Rejected
Failed — error occurred
!
Once settled, a promise NEVER changes
Immutable 
// A promise moves from Pending → Fulfilled OR Pending → Rejected
// It can NEVER go back or change once settled

// Pending → Fulfilled
const good = new Promise((resolve) => resolve("done"));

// Pending → Rejected
const bad = new Promise((_, reject) => reject("error"));

// ⚠ Calling resolve() after reject() (or vice versa) does NOTHING
const confused = new Promise((resolve, reject) => {
  resolve("first");   // ← this wins
  reject("second");   // ← ignored! Already settled
  resolve("third");   // ← ignored! Already settled
});

A promise is like a one-way door — once it goes through fulfilled or rejected, it stays there forever.

Basics

Creating a Promise

1
The Promise constructor
new Promise() 
// The constructor takes an "executor" function
// with two callbacks: resolve and reject

function fetchUser(id: number): Promise<string> {
  return new Promise((resolve, reject) => {
    if (id > 0) {
      resolve(`User #${id} found`);
    } else {
      reject("Invalid user ID");
    }
  });
}

// Quick shortcuts:
const instant = Promise.resolve("already done");   // Fulfilled
const failure = Promise.reject("already failed");   // Rejected
Consuming

Consuming with .then() & .catch()

✓

Real-Life: .then() is like saying "when my food arrives, eat it." .catch() is like saying "if the order is cancelled, order from somewhere else."

2
Handle success and failure
.then / .catch 
fetchUser(1)
  .then(result => {
    console.log(result);  // "User #1 found"
  })
  .catch(error => {
    console.error(error); // only runs if rejected
  });

// With arrow function shorthand:
fetchUser(1)
  .then(r => console.log(r))
  .catch(e => console.error(e));

// ⚠ .catch() catches errors from ANY .then() above it
fetchUser(1)
  .then(r => {
    throw new Error("Processing failed!");  // error in .then()
  })
  .catch(e => {
    console.error(e.message);  // "Processing failed!" — caught!
  });
Consuming

Promise Chaining

3
Chain .then() calls — each passes result to the next
Pipeline 
// Each .then() returns a NEW promise, so you can chain!

fetchUser(1)
  .then(user => {
    console.log(user);       // "User #1 found"
    return user.toUpperCase(); // pass transformed value to next
  })
  .then(upper => {
    console.log(upper);      // "USER #1 FOUND"
    return upper.length;     // pass length to next
  })
  .then(len => {
    console.log(len);        // 13
  })
  .catch(err => {
    console.error(err);      // catches ANY error above
  });

// ❌ WRONG — nesting .then() inside .then() (callback hell!)
fetchUser(1).then(user => {
  fetchUser(2).then(user2 => {
    fetchUser(3).then(user3 => {
      // Nested pyramid of doom!
    });
  });
});

// ✅ RIGHT — flat chain
fetchUser(1)
  .then(u1 => { console.log(u1); return fetchUser(2); })
  .then(u2 => { console.log(u2); return fetchUser(3); })
  .then(u3 => { console.log(u3); })
  .catch(err => console.error(err));
Consuming

.finally() — Always Runs

4
Cleanup code that runs regardless of outcome
.finally() 
// .finally() runs whether the promise fulfilled or rejected
// Great for cleanup: hiding spinners, closing connections

let loading = true;

fetchUser(1)
  .then(user => console.log(user))
  .catch(err => console.error(err))
  .finally(() => {
    loading = false;  // always runs!
    console.log("Loading done");
  });
Static Methods

Promise.all() — Wait for ALL

✓

Real-Life: You ordered pizza, garlic bread, and coke. Promise.all() is like saying "I won't start eating until ALL three arrive." If any ONE is cancelled, the whole meal fails.

1
Run in parallel, wait for all to finish
All or nothing 
const p1 = Promise.resolve("Pizza");
const p2 = Promise.resolve("Garlic Bread");
const p3 = Promise.resolve("Coke");

Promise.all([p1, p2, p3])
  .then(results => {
    console.log(results);  // ["Pizza", "Garlic Bread", "Coke"]
  });

// ⚠ If ANY one rejects, the ENTIRE .all() rejects!
const p4 = Promise.reject("Out of stock!");

Promise.all([p1, p2, p4])
  .catch(err => {
    console.error(err);  // "Out of stock!" — all failed
  });

Use when: you need ALL results and can't proceed without any single one.

Static Methods

Promise.race() — First One Wins

2
Returns whichever settles first
Fastest wins 
const slow = new Promise(r => setTimeout(() => r("Slow"), 3000));
const fast = new Promise(r => setTimeout(() => r("Fast"), 1000));

Promise.race([slow, fast])
  .then(winner => {
    console.log(winner);  // "Fast" — settled first!
  });

// Real use: timeout pattern
const fetchData = fetch("https://api.example.com/data");
const timeout = new Promise((_, reject) =>
  setTimeout(() => reject("Timeout!"), 5000)
);

Promise.race([fetchData, timeout])
  .then(data => console.log("Got data"))
  .catch(err => console.error(err));  // "Timeout!" if slow
Static Methods

Promise.allSettled() — Get ALL Results

3
Waits for all — never short-circuits
All results 
const p1 = Promise.resolve("OK");
const p2 = Promise.reject("Failed");
const p3 = Promise.resolve("Also OK");

Promise.allSettled([p1, p2, p3])
  .then(results => {
    console.log(results);
    // [
    //   { status: "fulfilled", value: "OK" },
    //   { status: "rejected", reason: "Failed" },
    //   { status: "fulfilled", value: "Also OK" }
    // ]
  });

// Unlike .all(), it NEVER rejects — you get every result

Use when: you want results from ALL promises, even if some fail.

Static Methods

Promise.any() — First SUCCESS Wins

4
First fulfilled wins, ignores rejections
Optimistic 
const p1 = Promise.reject("Server 1 down");
const p2 = new Promise(r => setTimeout(() => r("Server 2 OK"), 500));
const p3 = new Promise(r => setTimeout(() => r("Server 3 OK"), 200));

Promise.any([p1, p2, p3])
  .then(first => {
    console.log(first);  // "Server 3 OK" — first SUCCESS
  });

// Only rejects if ALL promises reject
Promise.any([
  Promise.reject("A"),
  Promise.reject("B"),
])
  .catch(err => {
    console.error(err);  // AggregateError: All promises rejected
  });
✓

🧠 Memory Trick — Remember All 4 Static Methods:

"All Rabbits Are Adorable"
→ Promise.all(), Promise.race(), Promise.allSettled(), Promise.any()

💡 Tip: all = ALL must pass, race = FIRST wins, allSettled = ALL results (no fail), any = first SUCCESS wins

Feature Callbacks Promises
Readability Nested pyramid Flat chain
Error handling Manual in each callback Single .catch() at end
Parallel Hard to coordinate Promise.all()
Return value Nothing — fire and forget Returns a Promise (chainable)
Composition Callback hell Clean chaining
Context

How async/await Relates

!
async/await is SUGAR on top of promises
Key Insight 
// These two are IDENTICAL in behavior:

// Promise way:
fetchUser(1)
  .then(user => console.log(user))
  .catch(err => console.error(err));

// async/await way (same thing, different syntax):
async function main() {
  try {
    const user = await fetchUser(1);
    console.log(user);
  } catch (err) {
    console.error(err);
  }
}

// await just PAUSES until the promise settles
// Under the hood, it's still using .then() and .catch()

Learn promises FIRST — async/await won't make sense without understanding what it's built on.

Watch Out!

⚠ Common Beginner Mistakes

⚠
Mistake 1: Forgetting .catch() — unhandled rejection

If a promise rejects and there's no .catch(), Node.js will crash or show a warning! 

// ❌ WRONG — no error handling
fetchUser(-1).then(u => console.log(u));
// UnhandledPromiseRejectionWarning!

// ✅ CORRECT — always add .catch()
fetchUser(-1)
  .then(u => console.log(u))
  .catch(err => console.error("Error:", err));

Rule: EVERY promise chain must end with .catch() (or be in a try/catch with await).

⚠
Mistake 2: Not returning inside .then() — chain breaks 
// ❌ WRONG — forgot return, next .then() gets undefined
fetchUser(1)
  .then(user => {
    user.toUpperCase();  // no return!
  })
  .then(upper => {
    console.log(upper);  // undefined 😱
  });

// ✅ CORRECT — return the value
fetchUser(1)
  .then(user => {
    return user.toUpperCase();  // pass to next
  })
  .then(upper => {
    console.log(upper);  // "USER #1 FOUND" ✅
  });

Rule: Always return inside .then() to pass data to the next step.

⚠
Mistake 3: Nesting .then() inside .then() — callback hell returns! 
// ❌ WRONG — nested pyramid (defeats the purpose of promises!)
fetchUser(1).then(u1 => {
  fetchUser(2).then(u2 => {
    fetchUser(3).then(u3 => {
      console.log(u1, u2, u3);
    });
  });
});

// ✅ CORRECT — flat chain
fetchUser(1)
  .then(u1 => { console.log(u1); return fetchUser(2); })
  .then(u2 => { console.log(u2); return fetchUser(3); })
  .then(u3 => console.log(u3))
  .catch(err => console.error(err));

Rule: Return the next promise from .then() — chain flat, don't nest.

⚠
Mistake 4: Promise.all() fails if ANY one fails 
// ❌ SURPRISE — one failure kills everything
Promise.all([
  Promise.resolve("OK"),
  Promise.reject("FAIL"),
  Promise.resolve("Also OK"),
]).catch(e => console.error(e));  // "FAIL" — all lost!

// ✅ Use allSettled() if you want ALL results regardless
Promise.allSettled([
  Promise.resolve("OK"),
  Promise.reject("FAIL"),
  Promise.resolve("Also OK"),
]).then(results => console.log(results));
// All 3 results available!

Rule: Use .all() when you need ALL-or-nothing. Use .allSettled() when you want every result.

⚠
Mistake 5: Creating a promise but never consuming it 
// ❌ WRONG — promise created but ignored
function loadData() {
  fetchUser(1);  // returns a Promise... which is ignored!
}

// ✅ CORRECT — consume the promise
function loadData() {
  return fetchUser(1);  // return it so caller can handle it
}
loadData().then(d => console.log(d));

Rule: Always return, await, or .then() a promise. Never let it float.

Mistake What happens Fix
No .catch() Unhandled rejection crash Always add .catch()
Missing return Next .then() gets undefined Return value from .then()
Nesting .then() Callback hell in disguise Chain flat, return promises
.all() one fails Entire result lost Use .allSettled()
Unconsumed promise Silent no-op Return, await, or .then()

Method Resolves when Rejects when Use case
Promise.all() ALL fulfill ANY one rejects Need all results
Promise.race() First settles (any) First settles (any) Timeout patterns

Quick Reference Cheatsheet

Create

new Promise((resolve, reject) => { })   // Constructor
Promise.resolve(value)                  // Instant fulfilled
Promise.reject(error)                   // Instant rejected

Consume

p.then(val => { })      // On success
p.catch(err => { })     // On failure
p.finally(() => { })    // Always runs

Static Methods

Promise.all([p1, p2])           // ALL must succeed
Promise.race([p1, p2])          // First to settle
Promise.allSettled([p1, p2])    // All results, no fail
Promise.any([p1, p2])           // First success

Chain Pattern

doStep1()
  .then(r1 => doStep2(r1))
  .then(r2 => doStep3(r2))
  .catch(err => handleError(err))
  .finally(() => cleanup());
