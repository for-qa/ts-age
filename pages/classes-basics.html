Property Modifier

readonly — Assign Once, Read Forever

✓

Real-Life Example: A readonly property is like your Aadhaar number. It's assigned to you once at birth (constructor), and you can show it to anyone (read), but you can NEVER change it.

A readonly property can be assigned a value only once — inside the constructor. After that, it can be read but never modified. 

class Student {
  readonly studentID: number;  // ← readonly keyword
  name: string;

  constructor(id: number, name: string) {
    this.studentID = id;   // ✅ Assigning ONCE in constructor — allowed
    this.name = name;
  }
}

let s1 = new Student(101, "John");

console.log(s1.studentID);  // ✅ Reading — allowed (101)
s1.studentID = 999;         // ❌ ERROR: Cannot assign — it's readonly!
s1.name = "Jane";           // ✅ Regular property — can be changed
Action readonly Regular
Assign in constructor ✅ Once only ✅ Anytime
Read from object ✅ Unlimited ✅ Unlimited
Modify after creation ❌ Never ✅ Anytime
Property Modifier

Optional (?) — Value Not Required

✓

Real-Life Example: An optional property is like the "Middle Name" field on a form. Some people have one, some don't. If left blank, it's just undefined — and that's perfectly fine.

Add ? after the property name. If no value is passed, it defaults to undefined. 

class Student {
  readonly studentID: number;
  name: string;
  email?: string;  // ← optional (question mark)

  constructor(id: number, name: string, email?: string) {
    //                                    → must also be optional here!
    this.studentID = id;
    this.name = name;
    this.email = email;
  }
}

let s1 = new Student(101, "John");              // no email — OK!
let s2 = new Student(102, "Bob", "b@mail.com"); // with email — OK!

console.log(s1.email);  // undefined
console.log(s2.email);  // "b@mail.com"
⚠

Two places! You must add ? in BOTH the property declaration AND the constructor parameter. Otherwise, the constructor will still demand a value.

Property & Method Modifier

static — Shared Across ALL Objects

✓

Real-Life Example: static is like the school name on a student ID card. Every student at "Greenwood High" has the SAME school name. If the school changes its name, EVERY student's card reflects the new name. The school name isn't personal to one student — it's shared.

A static property or method belongs to the class itself, not to individual objects. It's shared across ALL objects.

!
Static property & method
static 
class Student {
  name: string;
  static schoolName: string = "Greenwood High";  // shared!

  constructor(name: string) {
    this.name = name;
  }

  // Static method — changes the shared property
  static changeSchoolName(newName: string): void {
    Student.schoolName = newName;  // use CLASS name, not 'this'
  }

  displayInfo(): void {
    console.log(`${this.name} - ${Student.schoolName}`);
    //                          → access static via CLASS name
  }
}

let s1 = new Student("John");
let s2 = new Student("Bob");

s1.displayInfo();  // "John - Greenwood High"
s2.displayInfo();  // "Bob - Greenwood High"

// Change school name — affects ALL objects!
Student.changeSchoolName("Sunrise Academy");

s1.displayInfo();  // "John - Sunrise Academy" ← changed!
s2.displayInfo();  // "Bob - Sunrise Academy"  ← changed!

4 Key Rules for Static

# Rule
1 Static properties/methods are shared across all objects
2 Access using class name directly: Student.schoolName
3 Cannot use this keyword for static — use class name instead
4 Cannot access via object: s1.schoolName → ❌ Error in TypeScript
⚠

TypeScript vs Java: In Java, you CAN access static members via objects. In TypeScript, you can ONLY access them via the class name. s1.schoolName will throw an error!

Complete Example

Student Class — All Property Types

☑
Full class with readonly, optional, static
Complete 
class Student {
  readonly studentID: number;        // assign once only
  name: string;                      // regular property
  email?: string;                    // optional (can be undefined)
  static schoolName: string = "Greenwood High";  // shared

  constructor(sid: number, sName: string, email?: string) {
    this.studentID = sid;
    this.name = sName;
    this.email = email;
  }

  displayInfo(): void {
    console.log(`ID: ${this.studentID}`);
    console.log(`Name: ${this.name}`);

    // Handle optional property
    if (this.email) {
      console.log(`Email: ${this.email}`);
    } else {
      console.log("Email: Not provided");
    }

    // Access static via class name (NOT 'this')
    console.log(`School: ${Student.schoolName}`);
  }

  static changeSchoolName(newName: string): void {
    Student.schoolName = newName;
  }
}
TypeScript Shorthand

Constructor Parameter Properties

✓

Real-Life Example: Instead of writing a shopping list AND then putting each item on the shelf separately, you just put items on the shelf as you write them down. One step instead of two.

TypeScript lets you declare AND initialize properties directly in the constructor parameters by adding an access modifier (public, private, protected, or readonly). This eliminates the need to declare properties separately and assign them with this.x = x.

!
Verbose way — declare + assign separately
❌ Verbose 
class Person {
  public name: string;
  private age: number;
  readonly id: number;

  constructor(name: string, age: number, id: number) {
    this.name = name;  // line 1: declare
    this.age = age;    // line 2: assign in constructor
    this.id = id;      // line 3: 6 lines of boilerplate!
  }
}
✓
Shorthand — declare + assign in ONE line
✅ Clean TS 
class Person {
  // No separate property declarations needed!
  constructor(
    public name: string,      // auto-creates this.name
    private age: number,      // auto-creates this.age (private)
    readonly id: number       // auto-creates this.id (readonly)
  ) {
    // No this.x = x assignments needed!
    // TypeScript does it automatically.
  }

  greet(): string {
    return `Hi, I'm ${this.name}, age ${this.age}, ID: ${this.id}`;
  }
}

const p = new Person("Alice", 25, 101);
console.log(p.name);    // "Alice" ✅ public — accessible
// console.log(p.age);  // ❌ Error — private
// p.id = 999;          // ❌ Error — readonly

The modifier keyword (public, private, protected, readonly) in the constructor parameter is what triggers this behavior. Without it, the parameter is just a regular parameter — no property is created.

i

When to use: Always prefer the shorthand for simple classes. It's the idiomatic TypeScript way and is used in every professional codebase. The verbose way is only needed when you need custom initialization logic beyond simple assignment.

Overloading

Constructor Overloading

✓

Real-Life Example: Like a vending machine that accepts different inputs. Insert 2 coins → get a small drink. Insert 3 coins → get a large drink. Same machine (class), different inputs (constructor signatures), one mechanism (implementation).

Multiple constructor signatures, but only ONE implementation. The implementation must satisfy all signatures using optional parameters.

C
Calculator with constructor overloading
Overloading 
class Calculator {
  // Signature 1: no parameters (default constructor)
  constructor();

  // Signature 2: two parameters
  constructor(a: number, b: number);

  // ONE implementation — must match BOTH signatures
  constructor(a?: number, b?: number) {
    //         → optional    → optional
    if (a !== undefined && b !== undefined) {
      console.log(`Sum: ${a + b}`);
    } else {
      console.log("Default constructor called");
    }
  }
}

let calc1 = new Calculator();        // "Default constructor called"
let calc2 = new Calculator(10, 20);  // "Sum: 30"

The implementation uses ? to make parameters optional, satisfying BOTH the no-param and two-param signatures.

i

How it works: Signature 1 has 0 params → implementation's optional params match. Signature 2 has 2 params → implementation accepts 2 params. ONE body handles both cases with an if check.

Overloading

Method Overloading

✓

Real-Life Example: Like a calculator's add button. Press it with 2 numbers → adds 2. Press it with 3 numbers → adds 3. Same button name, different number of inputs, one mechanism.

Same concept as function overloading — multiple method signatures, one implementation. Inside a class, it's called method overloading instead of function overloading.

M
add() with 2 or 3 parameters
Overloading 
class Calculator {
  constructor() {}

  // Signature 1: two numbers
  add(a: number, b: number): number;

  // Signature 2: three numbers
  add(a: number, b: number, c: number): number;

  // ONE implementation — satisfies both
  add(a: number, b: number, c?: number): number {
    if (c !== undefined) {
      return a + b + c;  // 3 params → Signature 2
    }
    return a + b;          // 2 params → Signature 1
  }
}

let calc = new Calculator();

console.log(calc.add(10, 20));      // 30  (Signature 1)
console.log(calc.add(10, 20, 30));  // 60  (Signature 2)
Term Where Name
Overloading outside a class Standalone functions Function overloading
Overloading inside a class Class methods Method overloading
Overloading constructors Class constructors Constructor overloading
⚠

TS vs Java overloading: In Java, each overloaded method has its OWN body. In TypeScript, multiple signatures share ONE implementation. The implementation uses optional params + if checks to handle all cases.

// ✅ Correct — use CLASS name
Example.company;
Example.staticMethod();

let obj = new Example();
obj.company;          // ERROR in TypeScript!

// ❌ Wrong — cannot use 'this'
this.company;         // ERROR for static!

Constructor overloading

constructor();                        // Signature 1

  // handle both cases

add(a: number, b: number): number;              // Sig 1
