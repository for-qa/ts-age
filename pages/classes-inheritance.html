Core Concept

What is Inheritance?

Real-Life Example: Inheritance is like a family business. The parent builds a restaurant with recipes, equipment, and staff. The child inherits ALL of that â€” and can add new dishes or change existing recipes. They don't start from scratch.

Inheritance lets a class reuse the properties and methods of another class. Instead of rewriting the same code, the child class acquires everything from the parent.

  year: number;  // additional property

  constructor(name: string, color: string, model: string, year: number) {
    super(name, color, model);  // invoke parent constructor
    this.year = year;           // assign child's own property
  }

  // Override: same name, different implementation
  start(): void { console.log("Honda started"); }

  // New method (only in Honda)
  yom(): void {
    console.log(`${this.name} | ${this.color} | ${this.model} | ${this.year}`);
  }
}

Without extends Car, Honda and Car are unrelated. The keyword creates the parent-child bond.

Overriding

Method Overriding â€” New Implementation, Same Name

ðŸª

Real-Life Example: Your parents built a house (old design). You renovate it â€” same house, same address, but a new interior. That's overriding: same method name, new body.

When a child class has a method with the same name, same parameters, and same return type as the parent â€” the child's version replaces the parent's. The overridden (child) version is always called by default.

}

// Child

  start(): void { console.log("Honda started"); } // overridden!

Constructor

Child Class Constructor â€” The super() Rule

ðŸª

Real-Life Example: When a new branch office opens (child), the first thing they do is call headquarters (parent constructor) to set up the shared systems. Only THEN do they set up their own local equipment.

When both parent and child have constructors, the child MUST call super() first to invoke the parent's constructor. This is mandatory â€” TypeScript will throw an error if you skip it.

class Honda extends Car {
  year: number;

  constructor(name: string, color: string, model: string, year: number) {
    // Step 1: Call parent constructor FIRST (mandatory!)
    super(name, color, model);

    // Step 2: Then assign child's own properties
    this.year = year;
  }
}
âš 

Mandatory rule: super() must be the FIRST line in the child constructor. Write any other code AFTER the super call. Skipping it = compile error: "Constructors for derived class must contain a super call."
