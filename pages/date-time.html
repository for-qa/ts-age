Date & Time

JavaScript's Date object is notoriously tricky. Months start at 0, timezones bite you, and formatting is manual. Master the gotchas here.

Basics

Creating Dates

1
4 ways to create a Date
new Date() 
// 1. Current date/time
const now = new Date();
console.log(now);  // 2024-06-15T10:30:00.000Z

// 2. From a string
const d1 = new Date("2024-06-15");          // ISO format (safest!)
const d2 = new Date("June 15, 2024");       // human readable
const d3 = new Date("2024-06-15T10:30:00"); // with time

// 3. From numbers (year, monthINDEX, day, hours, min, sec)
const d4 = new Date(2024, 5, 15);  // ⚠ Month 5 = JUNE (0-indexed!)
const d5 = new Date(2024, 5, 15, 10, 30, 0);

// 4. From timestamp (milliseconds since Jan 1, 1970)
const d6 = new Date(1718448000000);

// Current timestamp:
const timestamp = Date.now();  // milliseconds since epoch
console.log(timestamp);        // 1718448000000
✓

🧠 Memory Trick — 7 Date Getters:

"Your Mom Dances Daily — Her Moves Slay"
→ getFullYear, getMonth, getDate, getDay, getHours, getMinutes, getSeconds

⚠ Month is 0-indexed! Jan=0, Dec=11. Day is also 0-indexed (Sun=0).

Basics

Getting Date Parts

2
Extract year, month, day, etc.
Getters 
const date = new Date("2024-06-15T10:30:45");

date.getFullYear();   // 2024
date.getMonth();      // 5  ⚠ NOT 6! (0-indexed: Jan=0, Jun=5)
date.getDate();       // 15 (day of month)
date.getDay();        // 6  (day of week: 0=Sun, 6=Sat)
date.getHours();      // 10
date.getMinutes();    // 30
date.getSeconds();    // 45
date.getMilliseconds(); // 0
date.getTime();       // 1718444445000 (timestamp)

// ⚠ THE BIGGEST TRAP:
// getMonth() is 0-INDEXED → January = 0, December = 11
// getDate() is 1-INDEXED → 1st = 1, 31st = 31
// getDay() is 0-INDEXED → Sunday = 0, Saturday = 6
Basics

Setting Date Parts

3
Modify dates in place
Setters 
const date = new Date("2024-06-15");

date.setFullYear(2025);
date.setMonth(11);     // December (0-indexed!)
date.setDate(25);      // 25th

console.log(date);     // 2025-12-25 (Christmas!)

// ⚠ Setters MUTATE the original date object!
// To avoid mutation, clone first:
const original = new Date("2024-06-15");
const copy = new Date(original.getTime());
copy.setFullYear(2025);
console.log(original); // still 2024-06-15 ✅
Formatting

Formatting Dates

4
Display dates in human-readable formats
Format 
const date = new Date("2024-06-15T10:30:00");

// Built-in methods:
date.toString();          // "Sat Jun 15 2024 10:30:00 GMT+0530"
date.toDateString();      // "Sat Jun 15 2024"
date.toTimeString();      // "10:30:00 GMT+0530"
date.toISOString();       // "2024-06-15T05:00:00.000Z"

// ✅ BEST: toLocaleDateString() — locale-aware formatting
date.toLocaleDateString("en-IN");
// "15/6/2024"

date.toLocaleDateString("en-IN", {
  day: "numeric", month: "short", year: "numeric"
});
// "15 Jun 2024"

date.toLocaleDateString("en-IN", {
  weekday: "long", day: "numeric", month: "long", year: "numeric"
});
// "Saturday, 15 June 2024"

// Time formatting:
date.toLocaleTimeString("en-IN", {
  hour: "2-digit", minute: "2-digit", hour12: true
});
// "10:30 am"

ISO Strings & Timestamps

5
Machine-readable date formats
ISO/Unix 
// ISO 8601 — the universal date format for APIs
const date = new Date();
date.toISOString();  // "2024-06-15T10:30:00.000Z"
// Always UTC, always consistent — best for storage & APIs

// Unix Timestamp — milliseconds since Jan 1, 1970
Date.now();            // 1718448000000  (current time)
date.getTime();        // 1718448000000  (specific date)
+date;                 // 1718448000000  (shortcut)

// Convert timestamp back to Date:
const fromTimestamp = new Date(1718448000000);

// ✅ Use ISO for storage/APIs, locale for display

Comparing Dates

6
Which is earlier? Later? Same?
Compare 
const d1 = new Date("2024-06-15");
const d2 = new Date("2024-12-25");

// ✅ Compare with > < >= <=
console.log(d1 < d2);   // true  (June is before December)
console.log(d1 > d2);   // false

// ❌ DO NOT use === or == for equality!
const a = new Date("2024-06-15");
const b = new Date("2024-06-15");
console.log(a === b);   // false! (different objects)
console.log(a == b);    // false!

// ✅ Compare timestamps for equality
console.log(a.getTime() === b.getTime());  // true ✅

// Is date in the past?
function isPast(date: Date): boolean {
  return date.getTime() < Date.now();
}

Date Calculations

7
Add days, difference between dates
Math 
// Difference in days:
const start = new Date("2024-01-01");
const end = new Date("2024-06-15");
const diffMs = end.getTime() - start.getTime();
const diffDays = Math.floor(diffMs / (1000 * 60 * 60 * 24));
console.log(diffDays);  // 166 days

// Add days to a date:
function addDays(date: Date, days: number): Date {
  const result = new Date(date.getTime());  // clone!
  result.setDate(result.getDate() + days);
  return result;
}

const today = new Date();
const nextWeek = addDays(today, 7);
const yesterday = addDays(today, -1);

// Age calculator:
function getAge(birthday: Date): number {
  const diff = Date.now() - birthday.getTime();
  return Math.floor(diff / (1000 * 60 * 60 * 24 * 365.25));
}
console.log(getAge(new Date("1999-03-15")));  // 25
Watch Out!

⚠ Common Mistakes

⚠
Mistake 1: Months are 0-INDEXED 
// ❌ WRONG — this creates JULY, not June!
const june = new Date(2024, 6, 15);  // July 15!

// ✅ CORRECT — June = month 5
const juneCorrect = new Date(2024, 5, 15);

Rule: Jan=0, Feb=1, ... Dec=11. Always subtract 1 from the human month number.

⚠
Mistake 2: Comparing dates with === fails 
// ❌ Two Date objects are NEVER === even with same value
new Date("2024-06-15") === new Date("2024-06-15")  // false!

// ✅ Compare timestamps
d1.getTime() === d2.getTime()  // true ✅
⚠
Mistake 3: Setters MUTATE the original 
const date = new Date("2024-06-15");
date.setFullYear(2025);  // MUTATES date!

// ✅ Clone first if you need the original
const copy = new Date(date.getTime());
copy.setFullYear(2025);
⚠
Mistake 4: JSON.stringify converts Date to string 
const obj = { created: new Date() };
const json = JSON.stringify(obj);
const parsed = JSON.parse(json);
console.log(typeof parsed.created);  // "string" — NOT a Date!

// ✅ Convert back manually
parsed.created = new Date(parsed.created);
Mistake What happens Fix
Month 0-indexed Wrong month Subtract 1 from human month
=== on dates Always false Compare .getTime()
Setters mutate Original changed Clone before modifying
JSON loses Date Becomes string new Date(parsed.field)
