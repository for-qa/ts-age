The this Keyword

The second most confusing JS concept after closures. this changes meaning depending on HOW and WHERE a function is called.

5

3

🧠 Memory Trick — Remember All 5 this Contexts:

"Great Runners Often Achieve Championships"
→ Global, Regular function, Object method, Arrow function, Class

💡 Also remember BCA for control: Bind (returns new fn), Call (Comma args), Apply (Array args)

Contexts

What is this?

✓

Real-Life: this is like the word "I". When Alice says "I am hungry," "I" = Alice. When Bob says "I am hungry," "I" = Bob. Same word, different person depending on who is speaking. In JS, this depends on who is calling the function.

!
this = the object that is calling the function
Core Rule 
// The golden rule:
// this = whatever is LEFT of the dot when the function is called

const user = {
  name: "Alice",
  greet() {
    console.log(this.name);  // this = user (left of the dot)
  }
};

user.greet();  // "Alice" — user is left of the dot
//  ↑ this = user
Contexts

this in Global Scope

1
Global this = window (browser) or globalThis
Global 
// In browser:
console.log(this);  // Window object

// In Node.js (module):
console.log(this);  // {} (empty module object)

// Modern way to access global in any environment:
console.log(globalThis);  // Works everywhere
Contexts

this in Regular Functions

2
Depends on strict mode!
Functions 
// NON-STRICT mode:
function showThis() {
  console.log(this);  // Window (browser) or global (Node)
}
showThis();  // no object left of dot → defaults to global

// STRICT mode (TypeScript uses this!):
"use strict";
function showThisStrict() {
  console.log(this);  // undefined!
}
showThisStrict();  // no object → undefined in strict mode

// ⚠ TypeScript always compiles in strict mode
// So standalone function calls → this = undefined
Contexts

this in Object Methods

3
this = the object calling the method
Methods 
const user = {
  name: "Alice",
  age: 25,
  greet() {
    console.log(`Hi, I'm ${this.name}, age ${this.age}`);
  }
};

user.greet();  // "Hi, I'm Alice, age 25"
// this = user (left of the dot)

// ⚠ BUT if you extract the method...
const greetFn = user.greet;
greetFn();  // "Hi, I'm undefined, age undefined" 😱
// No object left of the dot! this = undefined (strict mode)
Contexts

this in Arrow Functions

✓

Key difference: Arrow functions don't have their own this. They inherit this from the surrounding scope (lexical this). This is their superpower AND their trap.

4
Arrow functions inherit this from parent
Arrow => 
// ✅ Arrow functions are GREAT inside methods:
const user = {
  name: "Alice",
  hobbies: ["reading", "coding"],

  showHobbies() {
    // Arrow function inherits 'this' from showHobbies()
    this.hobbies.forEach(hobby => {
      console.log(`${this.name} likes ${hobby}`);  // ✅ works!
    });
  }
};
user.showHobbies();
// "Alice likes reading"
// "Alice likes coding"

// ❌ Arrow functions are BAD as object methods:
const bad = {
  name: "Bob",
  greet: () => {
    console.log(this.name);  // undefined! Arrow has no own 'this'
  }
};
bad.greet();  // undefined — inherits global this, not 'bad'
Contexts

this in Classes

5
this = the instance
Classes 
class User {
  constructor(public name: string) {}

  greet() {
    console.log(`Hi, I'm ${this.name}`);
  }

  // ✅ Arrow method — binds this to instance automatically
  greetArrow = () => {
    console.log(`Hi, I'm ${this.name}`);
  };
}

const alice = new User("Alice");
alice.greet();       // "Hi, I'm Alice" ✅

// ⚠ But passing as callback...
const fn = alice.greet;
fn();  // undefined — 'this' is lost!

// ✅ Arrow method survives extraction:
const fn2 = alice.greetArrow;
fn2();  // "Hi, I'm Alice" ✅ — arrow binds 'this' at creation
Control this

bind(), call(), apply()

6
Manually set what this should be
Control 
function greet(greeting: string, punctuation: string) {
  console.log(`${greeting}, ${this.name}${punctuation}`);
}

const alice = { name: "Alice" };
const bob = { name: "Bob" };

// CALL — invoke immediately, args one by one
greet.call(alice, "Hello", "!");    // "Hello, Alice!"
greet.call(bob, "Namaste", ".");    // "Namaste, Bob."

// APPLY — invoke immediately, args as array
greet.apply(alice, ["Hi", "!!"]);   // "Hi, Alice!!"

// BIND — returns NEW function with this permanently set
const aliceGreet = greet.bind(alice);
aliceGreet("Hey", "~");             // "Hey, Alice~"

// bind() is most useful for callbacks:
const user = {
  name: "Alice",
  greet() { console.log(this.name); }
};
setTimeout(user.greet.bind(user), 100);  // "Alice" ✅

call = C for Comma (args separated). apply = A for Array (args as array). bind = returns a new Bound function.

Watch Out!

⚠ Common Mistakes

⚠
Mistake 1: Arrow function as object method 
// ❌ Arrow has no own 'this' — inherits from outer scope
const obj = {
  name: "Alice",
  greet: () => console.log(this.name),  // undefined!
};

// ✅ Use regular function for methods
const obj2 = {
  name: "Alice",
  greet() { console.log(this.name); },  // "Alice" ✅
};
⚠
Mistake 2: Losing this when passing method as callback 
class Timer {
  count = 0;
  start() {
    // ❌ this is lost in callback!
    setInterval(function() {
      this.count++;  // TypeError!
    }, 1000);

    // ✅ FIX 1: arrow function
    setInterval(() => { this.count++; }, 1000);

    // ✅ FIX 2: bind
    setInterval(function() { this.count++; }.bind(this), 1000);
  }
}
⚠
Mistake 3: this inside setTimeout 
const user = {
  name: "Alice",
  delayedGreet() {
    // ❌ regular function — this = undefined
    setTimeout(function() {
      console.log(this.name);  // undefined!
    }, 100);

    // ✅ arrow function — inherits this
    setTimeout(() => {
      console.log(this.name);  // "Alice" ✅
    }, 100);
  }
};
⚠
Mistake 4: Forgetting bind() when extracting methods 
class Logger {
  prefix = "[LOG]";
  log(msg: string) { console.log(`${this.prefix} ${msg}`); }
}
const logger = new Logger();

// ❌ Extracted — this is lost
const logFn = logger.log;
logFn("test");  // undefined test

// ✅ Bind it
const boundLog = logger.log.bind(logger);
boundLog("test");  // [LOG] test
Mistake What happens Fix
Arrow as method this = outer scope Use regular function
Method as callback this = undefined Arrow fn or .bind()
this in setTimeout this = undefined Arrow fn
Extracted method this = undefined .bind(this)

Cheatsheet

// RULE: this = whatever is LEFT of the dot
obj.method()     // this = obj
func()           // this = undefined (strict) or global

// ARROW: no own this — inherits from parent scope
// Use arrows INSIDE methods, not AS methods

// CONTROL:
fn.call(obj, a, b)    // call with this=obj, args separate
fn.apply(obj, [a, b]) // call with this=obj, args array
fn.bind(obj)           // returns new function, this=obj forever

// CLASS: use arrow methods to auto-bind
class C { method = () => { this... }; }
