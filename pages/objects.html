What is an Object?

✓

Real-Life Example: Every real-world thing is an object. A car has properties (color, brand, speed) and behavior (drive, brake, honk). An employee has properties (name, salary, department) and behavior (calculateBonus, getDetails).

An object is a group of properties (data) and behavior (methods/functions).

ConceptReal-WorldIn Code
PropertiesName, age, color, sizeVariables (key-value pairs)
BehaviorWalk, eat, calculateMethods (functions inside object)

JS/TS specific definition

In JavaScript and TypeScript, an object is a collection of key-value pairs. Each key is a property name, and the value is the data or a function (method).

// Properties = key-value pairs
// Behavior  = methods (functions inside the object)
// A function inside an object = METHOD
ℹ

4 ways to create objects in TypeScript: (1) Direct object type, (2) Inline type, (3) Type alias, (4) Class. We'll cover each one.

Approach 1

Direct Object Type — Just Assign Data

✓

Real-Life Example: Like filling out a sticky note right away — you write Name: John, Age: 30 directly. No form template needed. Quick and simple.

Directly assign values to properties without declaring types. Works in both JavaScript and TypeScript (type inference handles the rest).

1
Employee object with properties & method
Direct
let employee = {
  name: "John",
  age: 30,
  salary: 50000,
  job: "Engineer",

  getDetails: function() {
    return `${this.name} is a ${this.job} earning ${this.salary}`;
  }
};

// Verify it's an object
console.log(typeof employee);  // "object"

// Access properties
console.log(employee.name);    // "John"
console.log(employee.salary);  // 50000

// Call method
console.log(employee.getDetails());
// "John is a Engineer earning 50000"
⚠

Don't use : object type annotation! If you write let employee: object = {...}, you won't be able to access properties later (TypeScript will complain "property does not exist on type object"). Just omit it — TS will infer the type automatically.

The this keyword

// Inside an object, 'this' refers to the CURRENT object
// this.name → accesses the 'name' property of THIS object

getDetails: function() {
  return this.name;  // ✅ Correct — accesses object's 'name'
  return name;       // ❌ Wrong — looks for external variable 'name'
}
Approach 2

Inline Type Object — Declare Structure, Then Assign

✓

Real-Life Example: Like designing a form first (Name: ___, Age: ___, Grade: ___), and THEN filling it out. Two-step process: structure first, data second.

First declare the structure with data types, then assign values. TypeScript only (because we specify types explicitly).

2
Student object with inline types
Inline
// Step 1: Declare structure (types only, no data)
let student: {
  name: string;
  age: number;
  grade: string;
  getSummary: () => string;  // arrow function declaration
}

// Step 2: Assign data
= {
  name: "Scott",
  age: 15,
  grade: "A",
  getSummary: function() {
    return `${this.name} is ${this.age} years old and scored ${this.grade}`;
  }
};

console.log(student.getSummary());
// "Scott is 15 years old and scored A"
⚠

Problem with inline types: If you create ANOTHER student, you have to re-write the entire structure again! For every object, the same structure must be repeated. This leads to the next approach — type aliases.

Approach 3

Type Alias — Define Once, Reuse Everywhere

✓

Real-Life Example: Like creating a reusable form template at a print shop. You design the form once, then photocopy it for every new student/employee. Same structure, different data.

Create the structure once with the type keyword, then reuse it for multiple objects. TypeScript only.

3
Product type with multiple objects
Type Alias
// Step 1: Define the type ONCE
type Product = {
  name: string;
  price: number;
  getInfo: () => string;
};

// Step 2: Create MULTIPLE objects using the same type!
let book1: Product = {
  name: "Learn Java",
  price: 300,
  getInfo: function() {
    return `${this.name} costs ${this.price}`;
  }
};

let book2: Product = {
  name: "Learn Python",
  price: 500,
  getInfo: function() {
    return `${this.name} costs ${this.price}`;
  }
};

let book3: Product = {
  name: "Learn JavaScript",
  price: 600,
  getInfo: function() {
    return `${this.name} costs ${this.price}`;
  }
};

console.log(book1.getInfo());  // "Learn Java costs 300"
console.log(book2.getInfo());  // "Learn Python costs 500"
console.log(book3.getInfo());  // "Learn JavaScript costs 600"

One type Product → reused for 3 objects. No repeated structure!

Approach 4 — Preferred ⭐

Class — The Blueprint Approach

✓

Real-Life Example: A class is like an architectural blueprint for a house. The blueprint itself isn't a house — but you can build many houses (objects) from the same blueprint. Each house has its own paint color and furniture (different data), but the same layout (structure).

A class is a blueprint containing properties and methods. Use the new keyword to create objects from it. This is the most preferred approach — pure object-oriented programming.

4
Person class with constructor
Class ⭐
class Person {
  // Properties (key: type)
  ssn: string;
  firstName: string;
  lastName: string;

  // Constructor — assigns data to properties
  constructor(ssn: string, firstName: string, lastName: string) {
    this.ssn = ssn;
    this.firstName = firstName;
    this.lastName = lastName;
  }

  // Methods
  getFullName(): string {
    return `${this.firstName} ${this.lastName}`;
  }

  getDetails(): string {
    return `${this.ssn} - ${this.getFullName()}`;
  }
}

// Create MULTIPLE objects with 'new'
let person1 = new Person("1111", "John", "Kennedy");
let person2 = new Person("2222", "David", "DV");
let person3 = new Person("3333", "Scott", "S");

console.log(person1.getDetails());  // "1111 - John Kennedy"
console.log(person2.getDetails());  // "2222 - David DV"
console.log(person3.getDetails());  // "3333 - Scott S"
ℹ

Why classes are preferred: This is the ONLY approach that's truly object-oriented. Java, Python, C# — all use classes. TypeScript's class syntax is very similar to Java. In automation and real projects, always use classes.

Accessing Data

Dot Notation vs Bracket Notation

✓

Real-Life Example: Two ways to find a book in a library:
Dot notation = "Go to the Science shelf, pick the Physics book" (library.science.physics)
Bracket notation = "Go to shelf ['Science'], pick book ['Physics']" (library["science"]["physics"])

NotationSyntaxWhen to use
Dotemployee.nameMost common — use this 90% of the time
Bracketemployee["name"]When key is dynamic or has special characters
 
let employee = { name: "John", salary: 50000, job: "Engineer" };

// Dot notation (preferred)
console.log(employee.name);       // "John"
console.log(employee.salary);     // 50000

// Bracket notation (key as string)
console.log(employee["name"]);    // "John"
console.log(employee["salary"]); // 50000

// Calling methods
console.log(employee.getDetails());      // dot
console.log(employee["getDetails"]());   // bracket
Modifying Data

Changing Object Values

let employee = { name: "John", job: "Engineer" };

// Modify using dot notation
employee.job = "Manager";
console.log(employee.job);  // "Manager" ✅

// Modify using bracket notation
employee["job"] = "Director";
console.log(employee.job);  // "Director" ✅
Advanced

Intersection Types — Combining Multiple Types

✓

Real-Life Example: Think of a job application form. Page 1 asks for personal info (name, age). Page 2 asks for contact info (email, phone). The complete form combines BOTH pages. That's intersection — merging multiple types into one.

Use the & operator to combine multiple types into one super-type.

&
Combining Personal + Contact types
Intersection
// Type 1: Personal info
type Personal = {
  name: string;
  age: number;
};

// Type 2: Contact info
type Contact = {
  email: string;
  phone: number;
};

// Combined type: has ALL properties from both!
type Candidate = Personal & Contact & {
  getContactInfo: () => string;
};

// Object using the combined type
let candidate: Candidate = {
  name: "Scott",        // from Personal
  age: 25,              // from Personal
  email: "s@gmail.com", // from Contact
  phone: 9876543210,    // from Contact
  getContactInfo: function() {
    return `${this.name} — ${this.email}, ${this.phone}`;
  }
};

console.log(candidate.getContactInfo());
// "Scott — s@gmail.com, 9876543210"

Candidate = Personal + Contact + its own method. All properties from all types are required.

Class Basics

Constructor — Assigning Data at Creation

✓

Real-Life Example: A constructor is like a hospital birth registration. The moment a baby (object) is born (new), the nurse (constructor) immediately records the name, weight, and time. You don't call the nurse later — she acts automatically at birth.

FeatureConstructorMethod
CalledAutomatically at object creationManually with obj.method()
ReturnsNothing (never)Can return values
PurposeAssign data to propertiesPerform operations / return data
Keywordconstructor (in TS/JS)Any name you choose
 
class Person {
  ssn: string;
  firstName: string;

  // Constructor — runs AUTOMATICALLY when 'new Person(...)' is called
  constructor(ssn: string, firstName: string) {
    this.ssn = ssn;            // assign parameter to class variable
    this.firstName = firstName;
  }
}

// 'new' triggers the constructor, passing data
let p = new Person("111", "John");
// constructor runs: this.ssn = "111", this.firstName = "John"

Why use this?

constructor(ssn: string) {
  this.ssn = ssn;
  // this.ssn → class property
  // ssn      → parameter (local)
  // Without 'this', TS can't tell which is which!
}

Objects are independent

let p1 = new Person("111", "John");   // p1 has its OWN copy
let p2 = new Person("222", "David");  // p2 has its OWN copy
let p3 = new Person("333", "Scott");  // p3 has its OWN copy

// p1, p2, p3 are INDEPENDENT — changing one doesn't affect others
// Same class structure, different data
Watch Out!

⚠ Common Beginner Mistakes

⚠
Mistake 1: Using object type blocks property access

Typing a variable as object (lowercase) tells TypeScript "this is some object" but gives NO information about what's inside. You lose all property access and autocomplete!

// ❌ WRONG — 'object' type blocks everything
let person: object = { name: "Alice", age: 25 };
console.log(person.name); // ❌ Error: Property 'name' does not exist on type 'object'
person.age = 26;          // ❌ Error: same reason

// ✅ CORRECT — use inline type or type alias
let person2: { name: string; age: number } = { name: "Alice", age: 25 };
console.log(person2.name); // "Alice" ✓
person2.age = 26;          // ✓ Works!

// ✅ EVEN BETTER — use a class or type alias
type Person = { name: string; age: number };
let person3: Person = { name: "Alice", age: 25 };

Rule: Never use object as a type. Always specify the shape (properties and their types) so TypeScript can help you.

⚠
Mistake 2: Arrow functions lose this inside objects

Arrow functions (=>) do NOT have their own this. If you use an arrow function as a method inside an object, this won't point to the object — it points to whatever this was outside!

let employee = {
  name: "Bob",
  // ❌ WRONG — arrow function: 'this' is NOT the object!
  greetArrow: () => {
    return `Hi, I'm ${this.name}`;  // 'this' is undefined or window!
  },
  // ✅ CORRECT — regular function: 'this' IS the object
  greetRegular: function() {
    return `Hi, I'm ${this.name}`;  // 'this' = employee object ✓
  }
};

console.log(employee.greetArrow());   // "Hi, I'm undefined" ❌
console.log(employee.greetRegular()); // "Hi, I'm Bob" ✓

Rule: For object methods that use this, always use function() syntax, NOT arrow functions. Save arrows for callbacks and standalone functions.

Quick Reference Cheatsheet

1. Direct object (no types)

let emp = { name: "John", age: 30,
  greet: function() { return `Hi, ${this.name}`; }
};

2. Inline type (declare + assign)

let student: { name: string; age: number } = { name: "Scott", age: 15 };

3. Type alias (reusable)

type Product = { name: string; price: number };
let book: Product = { name: "Learn TS", price: 300 };

4. Class (preferred ⭐)

class Person {
  name: string;
  constructor(name: string) { this.name = name; }
  greet(): string { return `Hi, ${this.name}`; }
}
let p = new Person("John");

Intersection types

type A = { x: number };
type B = { y: string };
type C = A & B;  // has BOTH x and y

Access & modify

obj.name           // dot notation (preferred)
obj["name"]        // bracket notation
obj.name = "New";  // modify value

Key rules

// ✅ 'this' keyword = current object inside methods
// ✅ 'new' keyword = creates object from class
// ✅ Constructor runs AUTOMATICALLY at object creation
// ✅ Constructor NEVER returns a value
// ✅ Objects from same class are INDEPENDENT
// ✅ Use colon (:) not equals (=) for key-value pairs
// ✅ Separate properties with commas (,) — class uses semicolons (;)
