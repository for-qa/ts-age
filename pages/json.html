Basics

What is JSON?

✓

Real-Life Example: JSON is like English at an international airport. Every country (programming language / system) speaks a different language, but they all understand English (JSON). It's the universal way to share data between servers, browsers, databases, and APIs.

!
JSON = JavaScript Object Notation
Core Concept 
// JSON is a TEXT FORMAT for storing and transporting data
// It LOOKS like a JavaScript object, but it's actually a STRING

// This is a JavaScript object:
const user = { name: "Alice", age: 25 };

// This is JSON (a STRING):
const json = '{"name": "Alice", "age": 25}';

// JSON is used everywhere:
// 1. API responses from servers
// 2. Config files (package.json, tsconfig.json)
// 3. Storing data in localStorage
// 4. Sending data between systems
Basics

JSON Syntax Rules

!
Strict rules — even one violation = crash
Rules 
// ✅ VALID JSON
{
  "name": "Alice",
  "age": 25,
  "isAdmin": true,
  "scores": [90, 85, 78],
  "address": {
    "city": "Mumbai",
    "pin": "400001"
  },
  "spouse": null
}

// ❌ INVALID JSON — common syntax errors:

// 1. Single quotes (must use double quotes!)
// { 'name': 'Alice' }           ❌

// 2. Unquoted keys
// { name: "Alice" }             ❌

// 3. Trailing comma
// { "name": "Alice", }          ❌

// 4. Comments
// { "name": "Alice" /* hi */ }  ❌

// 5. undefined or functions
// { "fn": undefined }           ❌

// 6. Single value without object/array wrapper
// "hello"  ← technically valid JSON, but unusual
✅
Allowed Types
string, number, boolean, null, object, array
❌
NOT Allowed
undefined, functions, Date, RegExp, Symbol, comments
Basics

JSON vs JavaScript Object

Feature JS Object JSON
Type Object (in memory) String (text)
Keys Unquoted or quoted MUST be double-quoted
Values Any JS value Only string, number, bool, null, obj, array
Trailing comma Allowed NOT allowed
Comments Allowed NOT allowed
Functions Can have methods Cannot
undefined Valid value NOT valid (gets dropped)
Core Methods

JSON.stringify() — Object → String

✓

Real-Life: Stringify is like packing your clothes into a suitcase (flat text) so you can travel (send data over the network). At the destination, you unpack (parse) them back into usable form.

1
Convert objects to JSON strings
stringify 
const user = { name: "Alice", age: 25, isAdmin: true };

// Convert to JSON string
const json = JSON.stringify(user);
console.log(json);
// '{"name":"Alice","age":25,"isAdmin":true}'

console.log(typeof json);  // "string" — it's text now!

// Arrays work too:
const nums = [1, 2, 3];
console.log(JSON.stringify(nums));  // '[1,2,3]'

// Nested objects:
const data = {
  users: [
    { name: "Alice", role: "admin" },
    { name: "Bob", role: "user" },
  ],
};
console.log(JSON.stringify(data));
// '{"users":[{"name":"Alice","role":"admin"},{"name":"Bob","role":"user"}]}'
Core Methods

JSON.parse() — String → Object

2
Convert JSON strings back to objects
parse 
const json = '{"name":"Alice","age":25}';

// Convert back to an object
const user = JSON.parse(json);
console.log(user.name);  // "Alice"
console.log(user.age);   // 25

// ⚠ The result type is 'any' — TypeScript doesn't know the shape!
// user.whatever   // No error, no autocomplete — dangerous!

// Arrays:
const arr = JSON.parse('[1, 2, 3]');
console.log(arr);  // [1, 2, 3]
Core Methods

Pretty Printing

3
Make JSON human-readable
Formatting 
const user = { name: "Alice", age: 25, scores: [90, 85] };

// Default — compact, one line
console.log(JSON.stringify(user));
// '{"name":"Alice","age":25,"scores":[90,85]}'

// Pretty — 2 spaces indentation
console.log(JSON.stringify(user, null, 2));
// {
//   "name": "Alice",
//   "age": 25,
//   "scores": [
//     90,
//     85
//   ]
// }

// 4 spaces:
console.log(JSON.stringify(user, null, 4));

// Tab indentation:
console.log(JSON.stringify(user, null, "\t"));

// The 3 arguments:
// JSON.stringify(value, replacer, spaces)
//   value    — what to convert
//   replacer — filter function (null = include everything)
//   spaces   — indentation (number or string)
TypeScript

Typing Parsed JSON — TypeScript-Specific!

4
JSON.parse() returns 'any' — fix it!
Type Safety 
interface User {
  name: string;
  age: number;
  isAdmin: boolean;
}

const json = '{"name":"Alice","age":25,"isAdmin":true}';

// ❌ No type safety — user is 'any'
const user1 = JSON.parse(json);
user1.whatever;  // no error! dangerous!

// ✅ Method 1: Type assertion (simple but trusting)
const user2 = JSON.parse(json) as User;
user2.name;     // ✅ autocomplete works!
// user2.xyz;   // ❌ TypeScript error

// ✅ Method 2: Assign to typed variable
const user3: User = JSON.parse(json);

// ⚠ WARNING: Both methods TRUST you!
// If the JSON doesn't match the type, TypeScript won't catch it
// at runtime — it only checks at compile time
TypeScript

Runtime Validation with Type Guards

5
Don't just trust — VERIFY!
Safe Parsing 
interface User {
  name: string;
  age: number;
}

// ✅ Type guard — validates at RUNTIME
function isUser(data: unknown): data is User {
  return (
    typeof data === "object" &&
    data !== null &&
    "name" in data &&
    typeof (data as User).name === "string" &&
    "age" in data &&
    typeof (data as User).age === "number"
  );
}

// Safe parsing pattern:
function parseUser(json: string): User | null {
  try {
    const data = JSON.parse(json);
    if (isUser(data)) {
      return data;  // ✅ confirmed User
    }
    console.error("Invalid user data");
    return null;
  } catch {
    console.error("Invalid JSON");
    return null;
  }
}

const user = parseUser('{"name":"Alice","age":25}');
if (user) {
  console.log(user.name);  // ✅ safe!
}
Real-World

localStorage — Store Data in Browser

6
localStorage only stores STRINGS — need JSON!
Storage 
interface Settings {
  theme: string;
  fontSize: number;
}

// SAVE — stringify before storing
const settings: Settings = { theme: "dark", fontSize: 16 };
localStorage.setItem("settings", JSON.stringify(settings));

// LOAD — parse after reading
const raw = localStorage.getItem("settings");
if (raw) {
  const loaded = JSON.parse(raw) as Settings;
  console.log(loaded.theme);     // "dark"
  console.log(loaded.fontSize);  // 16
}

// ⚠ localStorage.getItem() can return null!
// Always check before parsing!

// REMOVE
localStorage.removeItem("settings");

// CLEAR ALL
localStorage.clear();
Real-World

API Responses — fetch + JSON

7
APIs send JSON — you parse it
APIs 
interface Post {
  id: number;
  title: string;
  body: string;
}

// fetch returns a Response — you need to parse the JSON body
async function getPosts(): Promise<Post[]> {
  const response = await fetch("https://jsonplaceholder.typicode.com/posts");
  const data: Post[] = await response.json();  // auto-parses JSON!
  return data;
}

// response.json() is basically JSON.parse(responseBody)
// but it also returns a Promise, so you need 'await'

// Sending JSON to an API:
async function createPost(title: string, body: string) {
  const response = await fetch("https://api.example.com/posts", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ title, body }),  // stringify!
  });
  const result = await response.json();
  return result;
}
Watch Out!

⚠ Common Beginner Mistakes

⚠
Mistake 1: JSON.parse() on invalid JSON — crashes!

JSON.parse() throws a SyntaxError if the string isn't valid JSON. Always wrap in try/catch! 

// ❌ CRASH — invalid JSON
const bad = "{ name: 'Alice' }";  // unquoted key + single quotes
JSON.parse(bad);  // SyntaxError: Unexpected token n

// ✅ SAFE — always try/catch
try {
  const data = JSON.parse(bad);
} catch (err) {
  console.error("Invalid JSON:", err);
}

Rule: NEVER call JSON.parse() without try/catch if the data comes from user input, APIs, or localStorage.

⚠
Mistake 2: JSON.parse() returns 'any' — lost type safety 
// ❌ DANGEROUS — no type checking
const data = JSON.parse('{"name":"Alice"}');
console.log(data.xyz);  // undefined, no TS error!

// ✅ SAFE — assert or validate
interface User { name: string; }
const user = JSON.parse('{"name":"Alice"}') as User;
// user.xyz;  // TS error! ✅

Rule: Always use as Type or a type guard after JSON.parse().

⚠
Mistake 3: undefined, functions, and Dates get LOST in JSON 
const obj = {
  name: "Alice",
  age: undefined,        // ← will be REMOVED
  greet: () => "hi",     // ← will be REMOVED
  createdAt: new Date(), // ← becomes a STRING, not a Date!
};

const json = JSON.stringify(obj);
console.log(json);
// '{"name":"Alice","createdAt":"2024-01-15T10:30:00.000Z"}'
// age and greet are GONE!

const parsed = JSON.parse(json);
console.log(typeof parsed.createdAt);  // "string" — NOT a Date object!
// parsed.createdAt.getFullYear()  // TypeError!

Rule: undefined and functions disappear. Date becomes a string. Convert dates back manually after parsing.

⚠
Mistake 4: Single quotes in JSON — INVALID! 
// ❌ INVALID — single quotes are NOT valid JSON
const bad = "{'name': 'Alice'}";
JSON.parse(bad);  // SyntaxError!

// ✅ VALID — must use double quotes
const good = '{"name": "Alice"}';
JSON.parse(good);  // ✅ works

Rule: JSON requires double quotes for both keys and string values. No exceptions.

⚠
Mistake 5: Circular references crash stringify 
// ❌ CRASH — object references itself
const obj: any = { name: "Alice" };
obj.self = obj;  // circular reference!

JSON.stringify(obj);
// TypeError: Converting circular structure to JSON

// ✅ FIX — remove circular refs or use a replacer
JSON.stringify(obj, (key, value) => {
  if (key === "self") return undefined;  // skip circular
  return value;
});

Rule: Objects that reference themselves cannot be stringified. Remove circular refs or use a custom replacer.

Mistake What happens Fix
Parse invalid JSON SyntaxError crash Wrap in try/catch
Parse returns any No type safety as Type or type guard
undefined/functions/Date Lost or converted Handle manually after parse
Single quotes Invalid JSON Always use double quotes
Circular references TypeError crash Custom replacer function

const data = JSON.parse(json) as MyType;   // assertion
const data: MyType = JSON.parse(json);     // typed variable
// For runtime safety, use a type guard!

// Save
localStorage.setItem("key", JSON.stringify(data));

// Load
const raw = localStorage.getItem("key");
