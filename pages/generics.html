Core Concept

What are Generics?

ðŸª

Real-Life Example: Generics are like a gift box. The box doesn't care what's inside â€” it could hold a book, a phone, or shoes. The box design (function) stays the same, but the content (type) changes depending on what you put in. The box adapts to whatever you give it.

Generics let you write code that works with any type without losing type safety. Instead of hardcoding string or number, you use a type parameter (placeholder) like <T>.

// âŒ Without generics â€” need separate functions for each type
function identityString(value: string): string { return value; }
function identityNumber(value: number): number { return value; }

// âŒ Using 'any' â€” loses type safety
function identityAny(value: any): any { return value; }
let result = identityAny("hello");  // result is 'any' â€” no autocomplete!

// âœ… With generics â€” ONE function, works with ALL types, keeps safety
function identity<T>(value: T): T { return value; }

let a = identity<string>("hello");  // a is string âœ…
let b = identity<number>(42);       // b is number âœ…
let c = identity("auto");           // c is string (type inferred!) âœ…

Convention: T = Type, K = Key, V = Value, E = Element. You can use any letter, but these are standard. The <T> is declared after the function name and before the parentheses.

Functions

Generic Functions â€” Flexible Parameters

ðŸª

Real-Life Example: Like a printer. It doesn't care if you print a photo, a document, or a label. The printer adapts to whatever you feed it. Similarly, a generic function adapts to whatever type you pass.

// Returns the first element of any array
function getFirst<T>(arr: T[]): T {
  return arr[0];
}

let first1 = getFirst<number>([10, 20, 30]);    // number â†’ 10
let first2 = getFirst<string>(["a", "b", "c"]); // string â†’ "a"
let first3 = getFirst([true, false]);             // boolean â†’ true (inferred)

// Reverse any array
function reverseArray<T>(arr: T[]): T[] {
  return arr.reverse();
}

console.log(reverseArray([1, 2, 3]));       // [3, 2, 1]
console.log(reverseArray(["x", "y", "z"])); // ["z", "y", "x"]
// Arrow function syntax
const wrapInArray = <T>(value: T): T[] => [value];

console.log(wrapInArray(42));       // [42]
console.log(wrapInArray("hello"));  // ["hello"]
Multiple Types

Multiple Type Parameters

ðŸª

Real-Life Example: Like a shipping label with two fields â€” "From" (one type) and "To" (another type). Both are flexible but independent of each other.

// Two type parameters: T and U
function makePair<T, U>(first: T, second: U): [T, U] {
  return [first, second];
}

let pair1 = makePair("Alice", 25);       // [string, number]
let pair2 = makePair(true, [1, 2, 3]);   // [boolean, number[]]

console.log(pair1);  // ["Alice", 25]
console.log(pair2);  // [true, [1, 2, 3]]

// Key-value mapper
function mapKeyValue<K, V>(key: K, value: V): { key: K; value: V } {
  return { key, value };
}

let entry = mapKeyValue("name", "John");  // { key: string, value: string }
console.log(entry);  // { key: "name", value: "John" }

Generic Interfaces

ðŸª

Real-Life Example: Like a standardized report template. The format is always the same (title, date, data), but the "data" section can hold anything â€” sales numbers, student marks, inventory counts. The template adapts to the data type.

// Generic interface â€” T can be anything

Classes

Generic Classes

ðŸª

Real-Life Example: Like a storage locker. The locker design is the same, but you decide what goes inside â€” clothes, tools, books. A generic class is a reusable container that adapts to whatever type you store in it.

class Box<T> {
  private content: T;

  constructor(item: T) {
    this.content = item;
  }

  getContent(): T {
    return this.content;
  }

  replaceContent(newItem: T): void {
    this.content = newItem;
  }
}

// String box
let nameBox = new Box<string>("TypeScript");
console.log(nameBox.getContent());  // "TypeScript"

// Number box
let numBox = new Box<number>(42);
console.log(numBox.getContent());   // 42

// Object box
let userBox = new Box({ name: "Alice", age: 30 });
console.log(userBox.getContent().name);  // "Alice"
// Generic stack (LIFO data structure)
class Stack<T> {
  private items: T[] = [];

  push(item: T): void { this.items.push(item); }
  pop(): T | undefined { return this.items.pop(); }
  peek(): T | undefined { return this.items[this.items.length - 1]; }
  size(): number { return this.items.length; }
}

let numStack = new Stack<number>();
numStack.push(10);
numStack.push(20);
console.log(numStack.pop());   // 20
console.log(numStack.peek());  // 10
Advanced

Constraints â€” Limiting T with extends

ðŸª

Real-Life Example: A photo frame accepts any picture, but it must be a specific size. Similarly, a constrained generic accepts any type, but that type must have certain properties.

Sometimes T can be "too flexible" â€” you need to guarantee it has certain properties. Use extends to add constraints.

// âŒ Problem â€” T could be anything (no .length guarantee)
function logLength<T>(item: T): void {
  console.log(item.length);  // ERROR: Property 'length' doesn't exist on type T
}

// âœ… Solution â€” constrain T to types that have 'length'
interface HasLength {
  length: number;
}

function logLength<T extends HasLength>(item: T): void {
  console.log(item.length);  // âœ… Now TS knows T has .length
}

logLength("hello");      // 5 âœ… (strings have .length)
logLength([1, 2, 3]);    // 3 âœ… (arrays have .length)
logLength({ length: 7 }); // 7 âœ… (object with .length)
logLength(42);           // âŒ ERROR (numbers don't have .length)
// Constraint with keyof â€” ensure key exists in object
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

let person = { name: "Alice", age: 30 };
console.log(getProperty(person, "name"));  // "Alice" âœ…
console.log(getProperty(person, "email")); // âŒ ERROR â€” "email" not in person

Default Type Parameters

// Default type = string (if not specified)
interface Container<T = string> {
  value: T;
}

let c1: Container = { value: "hello" };        // T defaults to string
let c2: Container<number> = { value: 42 };     // T overridden to number

// Function with default
function createArray<T = number>(length: number, value: T): T[] {
  return Array(length).fill(value);
}

console.log(createArray(3, 0));        // [0, 0, 0]  â€” T = number (default)
console.log(createArray(3, "hi"));     // ["hi", "hi", "hi"] â€” T = string
