Core Concept

What is an Enum?

✓

Real-Life Example: An enum is like a TV remote with preset channels. Button 1 = "News", Button 2 = "Sports", Button 3 = "Movies". Instead of remembering channel numbers, you press named buttons. Enums give meaningful names to a fixed set of values.

An enum (enumeration) is a way to define a set of named constants. Instead of using magic numbers like 0, 1, 2, you give them descriptive names.

// ❌ Without enums — magic numbers (unreadable!)
let status = 0;  // What does 0 mean? Active? Pending? ✓

// ✅ With enums — clear, readable names
enum Status {
  Active,    // 0
  Inactive,  // 1
  Suspended  // 2
}

let userStatus: Status = Status.Active;  // Clear! ✅
ℹ

Syntax: enum EnumName { Member1, Member2, Member3 } — use PascalCase for both the enum name and its members by convention.

Default Behavior

Numeric Enums — Auto-Incrementing Numbers

✓

Real-Life Example: Like floor numbers in a building. Ground floor = 0, First floor = 1, Second floor = 2. Each floor automatically gets the next number.

By default, enum members start at 0 and auto-increment by 1.

enum Direction {
  Up,      // 0
  Down,    // 1
  Left,    // 2
  Right    // 3
}

console.log(Direction.Up);     // 0
console.log(Direction.Down);   // 1
console.log(Direction.Left);   // 2
console.log(Direction.Right);  // 3

// Use in comparisons
let playerMove: Direction = Direction.Left;

if (playerMove === Direction.Left) {
  console.log("Moving left!");  // ✅ This runs
}
Custom Values

Custom Start Values

✓

Real-Life Example: Like numbering exam questions starting from 1 instead of 0. You set the first question as 1, and the rest follow: Q1, Q2, Q3...

// Start from 1 instead of 0
enum StatusCode {
  OK = 200,
  NotFound = 404,
  ServerError = 500
}

console.log(StatusCode.OK);          // 200
console.log(StatusCode.NotFound);    // 404
console.log(StatusCode.ServerError); // 500

// Partially custom — auto-increments from last value
enum Priority {
  Low = 1,     // 1
  Medium,      // 2 (auto-incremented)
  High,        // 3 (auto-incremented)
  Critical     // 4 (auto-incremented)
}

console.log(Priority.Medium);   // 2
console.log(Priority.Critical); // 4
String Values

String Enums — Named String Constants

✓

Real-Life Example: Like a colour palette in a design tool. Instead of remembering "#FF5733", you name it "Sunset Orange". String enums map names to meaningful string values.

Each member must be explicitly assigned a string value — no auto-increment for strings.

enum Color {
  Red = "RED",
  Green = "GREEN",
  Blue = "BLUE"
}

console.log(Color.Red);    // "RED"
console.log(Color.Green);  // "GREEN"

// Great for API values or configurations
enum Environment {
  Dev = "development",
  Staging = "staging",
  Prod = "production"
}

let currentEnv: Environment = Environment.Dev;
console.log(currentEnv);  // "development"
⚠

No auto-increment! Unlike numeric enums, string enums require EVERY member to be explicitly assigned a value. You can't skip values.

Mixed Types

Heterogeneous Enums — Mixing Numbers & Strings

// Mix number and string values (not recommended, but possible)
enum Mixed {
  No = 0,
  Yes = "YES"
}

console.log(Mixed.No);   // 0
console.log(Mixed.Yes);  // "YES"
⚠

Not recommended! Mixing types makes enums confusing. Stick to either all-numeric or all-string enums for clarity.

Optimization

const Enums — Compile-Time Only

✓

Real-Life Example: Regular enums are like printed menus that stay on the table. Const enums are like the waiter reading the menu aloud — the info is delivered, but no menu card is left behind (no JavaScript output).

// Regular enum → generates JavaScript object at runtime
enum Direction {
  Up = "UP",
  Down = "DOWN"
}
// Compiles to: var Direction; Direction["Up"] = "UP"; ...

// const enum → inlined at compile time (no runtime object!)
const enum Speed {
  Slow = 1,
  Medium = 5,
  Fast = 10
}

let s = Speed.Fast;
// Compiles to: let s = 10;  (value directly inlined!)

Feature

Reverse Mapping — Value → Name

✓

Real-Life Example: Like a phonebook that works both ways. Look up "John" → get 12345 (forward). Look up 12345 → get "John" (reverse). Numeric enums support both directions!

enum Day {
  Monday = 1,
  Tuesday = 2,
  Wednesday = 3
}

// Forward: Name → Value
console.log(Day.Monday);     // 1

// Reverse: Value → Name
console.log(Day[1]);         // "Monday"
console.log(Day[3]);         // "Wednesday"
⚠

Numeric enums only! Reverse mapping works only with numeric enums, not string enums. String enums do NOT support Day["Monday"] reverse lookup.

Practical

Real-World Usage Patterns

1
API Response Handling
Pattern
enum HttpStatus {
  OK = 200,
  Created = 201,
  BadRequest = 400,
  NotFound = 404,
  ServerError = 500
}

function handleResponse(status: HttpStatus): void {
  switch (status) {
    case HttpStatus.OK:
      console.log("Success!");
      break;
    case HttpStatus.NotFound:
      console.log("Resource not found");
      break;
    case HttpStatus.ServerError:
      console.log("Server issue, try again later");
      break;
  }
}

handleResponse(HttpStatus.OK);  // "Success!"
2
Test Configuration
Automation
enum Browser {
  Chrome = "chrome",
  Firefox = "firefox",
  Safari = "safari",
  Edge = "edge"
}

enum TestEnv {
  Local = "http://localhost:3000",
  Staging = "https://staging.app.com",
  Production = "https://app.com"
}

function runTest(browser: Browser, env: TestEnv): void {
  console.log(`Running on ${browser} at ${env}`);
}

runTest(Browser.Chrome, TestEnv.Staging);
// "Running on chrome at https://staging.app.com"

Key rules

// ✅ Numeric enums auto-increment from 0 (or custom start)
// ✅ String enums require explicit values for EVERY member
// ✅ const enums are inlined — no runtime object generated
// ✅ Reverse mapping works for numeric enums ONLY
// ✅ Use enums for fixed sets of related constants
// ❌ Avoid mixing numbers and strings in one enum
