Introduction

What is an Array?

✓

Real-Life Example: Imagine you have a grocery list on a piece of paper. Each item is written on a line — Line 1: Milk, Line 2: Bread, Line 3: Eggs. That paper with multiple items is an array. A normal variable is like a sticky note — it can only hold ONE item. An array is like a full page — it holds MANY items in order.

A normal variable can hold only one value at a time. If you put a new value? The old one gets replaced. But what if you need to store a bunch of things — like a list of your friends' names or your exam scores? That's where an array comes in.

ℹ

Simple Definition: An array is a list variable — it stores multiple values under one name, like a to-do list that holds many tasks.

TypeScript vs Java — important difference

In Java, there's a regular array (fixed number of slots, like a parking lot with 10 spots) and an ArrayList (can grow, like a waiting line). In TypeScript, arrays work like the flexible version by default — they grow and shrink as needed. If you want a fixed-size collection in TypeScript, you use a tuple (covered later).

Core properties

  • Ordered — items stay in the order you put them in, just like songs in a playlist.
  • Numbered positions — each item has an index (position number) starting from 0, not 1. Think of elevator floors where Ground Floor = 0.
  • Grows automatically — you can keep adding items without defining a size upfront.
  • Can mix types — you can store text, numbers, true/false — all in one array if needed.
Fundamentals

Key concepts before you start

Indexing

✓

Real-Life Example: Think of an array like seats in a movie theater row. Seat numbers start from 0 (not 1!). So if there are 5 seats, they are numbered 0, 1, 2, 3, 4. The "index" is just the seat number.

Each slot in an array has an index — think of it as the slot's address. The first slot is always 0 (not 1!). If you have n items, the last one sits at position n − 1.

Array size (n)First index (lower bound)Last index (upper bound)
504  n − 1
1009  n − 1
100  n − 1
⚠

Accessing index n (equal to the length) is out of bounds. In Java this throws ArrayIndexOutOfBoundsException; in TypeScript/JavaScript it silently returns undefined.

The length attribute

arrayName.length returns the total number of elements stored — not the last valid index. Use it in loop conditions as i < arr.length (or i <= arr.length - 1).

let names: string[] = ["John", "Smith", "Peter", "Scott"];
console.log(names.length); // 4  ← count of elements, NOT the last index

Arrays

Declaration & initialisation

✓

Real-Life Example: Creating an array is like getting a new notebook. You can get an empty notebook and write names later (declaration only), or get one that already has names printed inside (declaration + initialisation). TypeScript gives you two ways to label this notebook.

There are two ways to write an array in TypeScript. Think of them as two different styles of handwriting — both produce the same result.

Approach 1
Literal / Bracket Syntax
Most common
Uses string[], number[] etc. Works in all languages with similar syntax. Preferred for typed arrays.
Approach 2
Generic Array Type
JS / TS only
Uses Array<string>, Array<number | string> etc. Specific to JavaScript and TypeScript — not available in Java or Python.
Variant
Declaration only
Two-step
Declare with empty brackets [], then assign values individually by index. Useful when values arrive dynamically.
Variant

Preferred

Declaration — Approach 1

Literal (bracket) syntax

✓

Real-Life Example: This is like writing Friends List: [] on a whiteboard. The [] tells everyone "this is a list." You can fill it in now or add names later.

The most popular way: write the data type, then [] after it. It's like saying "give me a list that only holds this type of stuff."

A
Declaration only — then assign by index
Two-step
let names: string[] = [];          // declaration — empty array

names[0] = "John";
names[1] = "Smith";
names[2] = "Peter";
names[3] = "Scott";

console.log(names);                // [ 'John', 'Smith', 'Peter', 'Scott' ]

Use this pattern when values arrive one at a time (e.g., from user input or a loop).

B
Declaration + initialisation — one statement
Preferred
let names: string[] = ["John", "Smith", "Peter", "Scott"];

console.log(names);                // [ 'John', 'Smith', 'Peter', 'Scott' ]
console.log(names[1]);             // Smith  ← index 1

Values are auto-indexed 0, 1, 2, 3 … without needing to write the index explicitly.

⚠

No fixed size in TypeScript arrays. Writing let names: string[5] causes a type error. Fixed-size collections are tuples, not arrays.

Declaration — Approach 2

Generic array type

✓

Real-Life Example: This is like using a labeled container. Instead of writing string[], you write Array<string> — it's like putting a "STRINGS ONLY" label on a container. Same result, different style.

Another way to write arrays using the Array<T> format. This style is only available in TypeScript/JavaScript — you won't see it in Java or Python.

1
Single type — strings
Array<string>
let empNames: Array<string> = ["Alice", "Bob", "Carol"];
2
Single type — numbers
Array<number>
let empIds: Array<number> = [101, 102, 103, 104];
3
Union type — string or number
Array<string | number>
let data: Array<string | number> = ["John", 101, "Alice", 102];

The union operator | allows either type — but no other type.

4
Any type — mixed data
Array<any>
let mixedData: Array<any> = [42, "hello", true, null];
⚠

Using any disables TypeScript's type checking for that array. Prefer specific types or unions where possible.

Type mode comparison

Type argumentAllowed valuesTypical use case
Array<string>Strings onlyNames, labels, IDs stored as text
Array<number>Numbers onlyIDs, prices, quantities
Array<string | number>Strings or numbersMixed tabular data, records
Array<any>AnythingRapid prototyping; avoid in production
Iterating Arrays

Three ways to loop over an array

✓

Real-Life Example: Imagine you have a tray of 10 cupcakes. You want to check each one. You can: (1) count slot by slot — "Slot 0, Slot 1, Slot 2..." (classic for). (2) ask "which slot am I on?" automatically (for…in). (3) just grab each cupcake directly without caring about the slot number (for…of).

To look at each item in an array one by one, you use a loop. TypeScript gives you three styles — each works differently with the position number (index).

Loop 1
Classic for
index mandatory
Loop 2
for…in
yields index
Loop 3
for…of
yields value directly
ℹ

A fourth option — forEach() — is an array method that accepts a callback function. Covered in Day 10 / Day 11.

Loop 1

Classic for loop

✓

Real-Life Example: Like a security guard checking rooms in a hotel — Room 0, Room 1, Room 2... He starts at Room 0, checks it, moves to the next, and stops when there are no more rooms.

The old-school loop where YOU control everything — where to start, when to stop, and how to move forward. You always work with the position number (index).

1
Classic for loop — using length
Index-based
let empNames: Array<string> = ["Alice", "Bob", "Carol", "Dave"];

for (let i = 0; i < empNames.length; i++) {
  console.log(empNames[i]);
}
// Alice  Bob  Carol  Dave

Parts of the loop

PartExpressionPurpose
Initlet i = 0Start at index 0
Conditioni < empNames.lengthRun while index is within bounds. length = 4, last index = 3 → use < not <=
Incrementi++Move to the next index after each iteration
AccessempNames[i]Retrieve the element at the current index
⚠

i <= empNames.length causes an off-by-one error — the last iteration reads index 4 which doesn't exist. Always use < with length, or <= with length - 1.

Loop 2

for…in loop

✓

Real-Life Example: Like a postman who reads house numbers — he knows the address (index) but has to open the mailbox to see what's inside (value). He gives you the house number, not the letter.

A shorter way to loop. But careful: the loop variable gives you the position number (index), NOT the actual value. You still need to use that number to get the item.

2
for…in — index-based, compact
Index yielded
let empIds: Array<number> = [101, 102, 103, 104];

for (let i in empIds) {
  console.log(empIds[i]);
}
// 101  102  103  104
ℹ

i in a for…in loop is the index, not the value. You must still write empIds[i] to get the element. Writing console.log(i) gives 0 1 2 3.

Loop 3

for…of loop

✓

Real-Life Example: Like a waiter bringing dishes to your table — he hands you each dish directly. You don't care if it was Dish #1 or Dish #3 — you just get the food itself. No position numbers involved!

The easiest loop for getting values. The loop variable holds the actual value — no index numbers, no extra steps. Just the item itself.

3
for…of — value-based, no index
Value yielded
let data: Array<string | number> = ["John", 101, "Alice", 202];

for (let value of data) {
  console.log(value);
}
// John  101  Alice  202

The variable name (value here) is user-defined — it is not a keyword.

Loop comparison summary

Loop typeVariable holdsNeed to write arr[i]?Condition required?
for (let i = 0; …)IndexYesYes
for (let i in arr)IndexYesNo — auto
for (let v of arr)ValueNoNo — auto
Functions & Arrays

Using arrays with functions

✓

Real-Life Example: Think of a function as a kitchen blender. You can put in a basket of fruits (pass an array) and get back a basket of juices (return an array). The blender works on the whole basket, not one fruit at a time.

Arrays can travel in and out of functions — you can send a list into a function and get a list back. This is how functions work with groups of data.

✓

Pro tip: A function can only return ONE thing. So if you need to send back multiple values? Pack them in an array — it's like putting multiple gifts in one bag.

Functions — Example 4

Passing an array to a function

4
Search an element inside an array
Array as parameter
function search(element: number, arr: number[]): boolean {
  for (let i = 0; i < arr.length; i++) {
    if (arr[i] === element) {
      return true;   // element found
    }
  }
  return false;      // element not found
}

let ar: number[] = [10, 20, 30, 40, 50];

console.log(search(30, ar));   // true
console.log(search(100, ar));  // false
ElementCodeNotes
Param 1element: numberThe value we are searching for
Param 2arr: number[]Array parameter — the collection to search in
Returnbooleantrue = found, false = not found
Functions — Example 5

Returning an array from a function

5
Convert all strings to uppercase — returns an array
Array as return type
function capitalizeWords(ar: string[]): string[] {
  let result: string[] = [];

  for (let i = 0; i < ar.length; i++) {
    result[i] = ar[i].toUpperCase();
  }

  return result;
}

let words: string[] = ["hello", "world", "typescript"];
console.log(capitalizeWords(words));
// [ 'HELLO', 'WORLD', 'TYPESCRIPT' ]
ℹ

The result is stored at result[i] — the same index as the source element. Position is preserved. The original ar array is unchanged.

Tuples

What is a Tuple?

✓

Real-Life Example: Think of a tuple like a passport — it has exactly 3 things in a fixed order: Name (text), Age (number), Photo (image). You can't add a 4th field, you can't swap the order, and the Name slot can't suddenly become a number. That's a tuple — a fixed-size box where every slot has a specific type.

A tuple is like an array's strict older sibling — it has two rules that can't be broken: fixed length (you can't add or remove items) and fixed types per position (each slot only accepts one specific type).

ℹ

Simple Definition: A tuple is a list with a locked size and locked types. Slot 1 = always text, Slot 2 = always number. No exceptions.

T1
Two-element tuple — [string, number]
Fixed 2 values
let person: [string, number] = ["John", 25];

console.log(person);      // [ 'John', 25 ]
console.log(person[0]);   // John
console.log(person[1]);   // 25

[string, number] is not a union operator — it is the tuple type definition.

T2
Multi-element tuple — mixed types
Fixed 5 values
let user: [number, string, boolean, number, string] = [101, "John", true, 99, "Admin"];
console.log(user);        // [ 101, 'John', true, 99, 'Admin' ]
console.log(user.length); // 5

Tuple constraints

ViolationTypeScript behaviour
Adding a 3rd element to a [string, number] tupleCompile error
Putting a number in position 0 of [string, number]Compile error
Swapping the two valuesCompile error
Tuples

Iterating tuples with for loops

All three loop styles that work for arrays work identically for tuples. The length attribute is also available.

1
Classic for loop on a tuple
Index-based
for (let i = 0; i < user.length; i++) { console.log(user[i]); }
2
for…in on a tuple
Index yielded
for (let i in user) { console.log(user[i]); }
3
for…of on a tuple
Value yielded
for (let value of user) { console.log(value); }
Tuples — Advanced

Array of tuples

By appending [] to a tuple type, you create an array whose elements are tuples.

T3
Array of [number, string] tuples — student records
Array of Tuples
let students: [number, string][] = [
  [101, "John"],
  [102, "Peter"],
  [103, "Scott"]
];

console.log(students.length);    // 3
console.log(students[0]);        // [ 101, 'John' ]

// Extract a single field — two-step
let tp = students[1];            // [ 102, 'Peter' ]
console.log(tp[0]);              // 102
console.log(tp[1]);              // Peter
ℹ

students[1][0] is not valid here — this implies a 2D array. Always extract the tuple first, then index into it.

Edge Cases

Common mistakes & gotchas

EC-01
Off-by-one
Using <= with length
Writing i <= arr.length causes the last iteration to access an out-of-bounds index, returning undefined. Always use i < arr.length or i <= arr.length - 1.
EC-02
for…in value
for…in yields index, not value
Beginners often expect for (let i in arr) to give values. It gives indexes. Write arr[i] to get the element. For direct values, use for…of.
EC-03
Fixed size
Attempting to set array size with a number
Writing let names: string[5] is a TypeScript error. Array size is dynamic. If you need fixed size, use a tuple.
EC-04
Tuple order
Swapping tuple values
If a tuple is declared as [number, string] then [25, "John"] is valid but ["John", 25] is a compile error. Type and position are both fixed.
EC-05
Tuple union
Confusing [string, number] with a union
[string, number] is a tuple type. string | number is a union type. They look similar but behave completely differently.
EC-06
2D Array
2D arrays vs array of tuples
students[0][1] implies a 2D array. An array of tuples requires two steps: extract the tuple, then index into it.

Introduction

Array methods overview

✓

Real-Life Example: Think of array methods as tools in a toolbox. Your array (the list) is a piece of wood. You have tools like a hammer (push — add a nail), pliers (pop — pull out a nail), a saw (slice — cut a piece), and a drill (splice — make holes and add things). Each tool does a specific job so you don't have to do everything by hand!

Array methods are ready-made tools that every array comes with. Instead of writing loops to add, remove, or find items, you just call a method — one line does the work.

ℹ

Method vs attribute — what's the difference? arr.length is like checking a label (no parentheses needed). arr.push() is like pressing a button (needs parentheses () to work). This comes up in interviews!

Arrays used throughout this document

let numbers: number[] = [1, 2, 3, 4, 5];

Attribute (not a method)

length

Returns the total number of elements in the array. It is an attribute, not a method — no parentheses.

let numbers: number[] = [1, 2, 3, 4, 5];
console.log(numbers.length); // 5

let fruits: string[] = ["apple", "banana", "orange", "mango"];
console.log(fruits.length);  // 4
⚠

length returns the element count, not the last valid index. The last valid index is always length - 1.

Method 1

push()

✓

Real-Life Example: Like adding people to the end of a queue at a movie theater. New people always join at the back. That's push() — it adds items to the END of the list.

Adds one or more items to the end of your array. The original array gets changed (the new item is actually added to it).

1
push() — add to end
Modifies original
let numbers: number[] = [1, 2, 3, 4, 5];

numbers.push(6, 7);            // add multiple values at once

console.log(numbers);
// [ 1, 2, 3, 4, 5, 6, 7 ]

Method signature

ParameterTypeDescription
...itemsone or more valuesElements to add at the end. Comma-separate multiple values.
ReturnsnumberNew length of the array after insertion
ℹ

Use push() when you don't know the current last index but want to append values. No index management needed.

Method 2

pop()

✓

Real-Life Example: Like taking the top plate off a stack of plates. The last plate added is the first one you grab. pop() removes the LAST item and gives it back to you.

Removes the last item from your array and hands it to you. Only removes ONE item at a time. The array shrinks by one.

2
pop() — remove from end
Returns removed element
let fruits: string[] = ["apple", "banana", "orange", "mango"];

let lastFruit = fruits.pop();       // removes "mango" and returns it

console.log(fruits);                // [ 'apple', 'banana', 'orange' ]
console.log("Removed:", lastFruit); // Removed: mango

Method signature

ParameterDescription
NoneTakes no parameters
ReturnsThe removed element, or undefined if the array was empty
Method 3

shift()

✓

Real-Life Example: Like a bus stop queue — the person at the FRONT gets on the bus and leaves the queue. Everyone else shifts forward by one position. That's shift() — it removes the FIRST item.

Removes the first item from your array and gives it back. Everyone else in the array moves one position forward.

3
shift() — remove from start
Returns removed element
let numbers: number[] = [1, 2, 3, 4, 5, 6, 7];

let firstNum = numbers.shift();     // removes 1 and returns it

console.log(numbers);               // [ 2, 3, 4, 5, 6, 7 ]
console.log("Removed:", firstNum);  // Removed: 1
Method 4

unshift()

✓

Real-Life Example: Like a VIP cutting to the front of a line. New items are added at the BEGINNING, and everyone else gets pushed back. That's unshift().

Adds one or more items to the very beginning of your array. All existing items shift to make room. Returns the new total count.

4
unshift() — add to start
Modifies original
let fruits: string[] = ["apple", "banana", "orange"];

fruits.unshift("kiwi", "pear");     // add two values at the beginning

console.log(fruits);
// [ 'kiwi', 'pear', 'apple', 'banana', 'orange' ]

push / pop / shift / unshift — comparison

MethodOperationPositionReturnsInterview tip
push()AddEndNew lengthOpposite of pop

⚠

Common interview question: "What is the difference between push/pop and shift/unshift?" — Answer: push/pop operate on the end of the array; shift/unshift operate on the start.

Method 5

concat()

✓

Real-Life Example: Like merging two playlists into a new one. Your original playlists stay untouched — you just get a new combined playlist. That's concat().

Joins two or more arrays together into a brand new array. The originals stay the same — concat doesn't mess with your existing lists.

5
concat() — combine multiple arrays
Returns new array
let numbers: number[] = [2, 3, 4, 5, 6, 7];

let combinedArray = numbers.concat([8, 9], [10]);

console.log(combinedArray);
// [ 2, 3, 4, 5, 6, 7, 8, 9, 10 ]

console.log(numbers);       // original unchanged: [ 2, 3, 4, 5, 6, 7 ]
⚠

You can only concat arrays of the same type. Concatenating a number[] with a string[] causes a TypeScript type error.

Method 6

slice()

✓

Real-Life Example: Like photocopying a few pages from a book. The book stays complete — you just get copies of pages 2 to 5. slice() copies a portion of your array without damaging the original.

Extracts a portion of your array and gives it as a new array. The original? Completely untouched. You tell it where to start (included) and where to stop (NOT included).

6
slice(start, end) — extract a section
Returns new array · non-destructive
// Current fruits: [ 'kiwi', 'pear', 'apple', 'banana', 'orange' ]
//   index:             0       1       2        3         4

let extracted = fruits.slice(1, 3);
console.log(extracted);   // [ 'pear', 'apple' ]

let last3 = fruits.slice(2, 5);
console.log(last3);       // [ 'apple', 'banana', 'orange' ]

Index rules for slice()

ParameterBehaviourExample
startInclusive — element at this index IS includedslice(1, …) starts from index 1
endExclusive — element at this index is NOT included (takes end − 1)slice(…, 3) takes up to index 2

Memory aid: slice(1, 3) gives you elements at positions 1 and 2 — think of it as "from 1, up to but not including 3".

Method 7

splice()

✓

Real-Life Example: Like a surgeon operating on your array. It can remove items (cut out a section), add new items (insert new parts), or replace items (swap out old for new). Unlike slice, this one actually changes your original array — it's like actual surgery, not just an X-ray.

The Swiss army knife of array methods. It can add, remove, or replace items ANYWHERE in your array. Warning: it changes the original array permanently.

ℹ

slice vs splice — how to remember: slice = safe photocopy (original untouched). splice = surgery (original gets modified). The extra p stands for "permanent change."

Ex 1
Delete elements only
splice(start, deleteCount)
// fruits: [ 'kiwi', 'pear', 'apple', 'banana', 'orange' ]

let removed = fruits.splice(1, 2);  // from index 1, delete 2 elements

console.log(fruits);    // [ 'kiwi', 'banana', 'orange' ]
console.log(removed);   // [ 'pear', 'apple' ]  ← returned as array

Second parameter is the delete count (how many to remove), NOT an ending index.

Ex 2
Insert elements without deleting
splice(start, 0, ...items)
// fruits: [ 'kiwi', 'banana', 'orange' ]

fruits.splice(1, 0, "pineapple", "grape");

console.log(fruits);
// [ 'kiwi', 'pineapple', 'grape', 'banana', 'orange' ]

Delete count 0 means nothing is removed — only the new items are inserted at the given position.

Ex 3
Replace elements (delete + insert)
splice(start, deleteCount, ...items)
// fruits: [ 'kiwi', 'pineapple', 'grape', 'banana', 'orange' ]

fruits.splice(1, 2, "mango", "cherry");

console.log(fruits);
// [ 'kiwi', 'mango', 'cherry', 'banana', 'orange' ]
// pineapple + grape were removed; mango + cherry were inserted at index 1

splice() parameter reference

ParameterRequired?Description
startRequiredIndex at which to begin the operation
deleteCountOptionalNumber of elements to remove from start. Use 0 to insert without removing.
...itemsOptionalElements to insert at start (after deletion). Can be zero or more.
Returns—Array of removed elements (empty array if none removed)

Method 8

indexOf()

✓

Real-Life Example: Like asking a librarian "Which shelf is this book on?" She tells you the shelf number (index). If the book doesn't exist? She says "-1" (not found). That's indexOf().

Finds the position number of the first matching item in your array. If the item doesn't exist, it returns −1 (a special "not found" signal).

8
indexOf() — find the index of an element
Returns index or −1
// fruits: [ 'kiwi', 'mango', 'cherry', 'banana', 'orange' ]

let bananaIndex = fruits.indexOf("banana");
console.log("Index of banana:", bananaIndex);  // 3

let papayaIndex = fruits.indexOf("papaya");
console.log("Index of papaya:", papayaIndex);  // -1  (not found)

// Search starting from a specific index (second parameter)
let idx = fruits.indexOf("banana", 2);         // start searching from index 2
console.log(idx);                              // 3

Method 9

includes()

✓

Real-Life Example: Like asking "Is pizza on the menu?" The answer is simply Yes (true) or No (false). You don't care WHERE it is on the menu — you just want to know if it exists. That's includes().

Checks if a specific item exists in the array. Gives you a simple yes/no answer — true or false. No position number, just existence.

9
includes() — existence check
Returns boolean
let isAppleExist  = fruits.includes("apple");
console.log("Has apple?",  isAppleExist);   // false  (not in current list)

let isBananaExist = fruits.includes("banana");
console.log("Has banana?", isBananaExist);  // true

Method 10

toString()

✓

Real-Life Example: Like reading out your grocery list to someone on the phone — instead of handing them the paper list (array), you say the items as one sentence separated by commas: "milk, bread, eggs." That's toString() — it turns your list into a single text string.

Converts your array into a single text string with commas between each item. The original array stays as-is — you just get a text version.

10
toString() — array to string
Returns string
let numbers: number[] = [2, 3, 4, 5, 6, 7];
let numbersStr = numbers.toString();

console.log(numbers);     // [ 2, 3, 4, 5, 6, 7 ]   ← still an array
console.log(numbersStr);  // "2,3,4,5,6,7"           ← string representation

// Another example — character array
let myArray: string[] = ["W", "E", "L", "C", "O", "M", "E"];
let strArray = myArray.toString();
console.log(strArray);    // "W,E,L,C,O,M,E"
