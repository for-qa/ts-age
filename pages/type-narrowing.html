Type Narrowing

What is Type Narrowing?

✓

Real-Life Example: You receive a mystery package. It could be a book or a toy. Before you can READ it (book) or PLAY with it (toy), you need to OPEN it and check what's inside. That "check" is narrowing — you go from "could be anything" to "definitely a book."

Narrowing is the process of refining a broad type to a more specific one. When a variable can be multiple types (union), TypeScript requires you to check which type it is before using type-specific features.

// The problem: TS doesn't know if 'value' is a string or number
function process(value: string | number) {
  value.toUpperCase();  // ❌ ERROR — might be a number!
  value.toFixed(2);     // ❌ ERROR — might be a string!
}

// The solution: narrow the type first
function process(value: string | number) {
  if (typeof value === "string") {
    value.toUpperCase();  // ✅ TS knows it's a string here
  } else {
    value.toFixed(2);     // ✅ TS knows it's a number here
  }
}
Guard #1

typeof Guard — Check Primitive Types

✓

Real-Life Example: Like a metal detector at security. It checks: "Is this metal?" → Yes = handle one way. No = handle another way. typeof checks the primitive type of a value.

function describe(value: string | number | boolean): string {
  if (typeof value === "string") {
    return `Text: "${value.toUpperCase()}"`;
  } else if (typeof value === "number") {
    return `Number: ${value.toFixed(2)}`;
  } else {
    return `Boolean: ${value ? "YES" : "NO"}`;
  }
}

console.log(describe("hello"));  // Text: "HELLO"
console.log(describe(3.14));     // Number: 3.14
console.log(describe(true));     // Boolean: YES

⚠

Watch out! typeof null === "object" — this is a famous JavaScript bug. Always check for null separately: if (value !== null && typeof value === "object")

Guard #2

Truthiness Guard — Check for null/undefined

✓

Real-Life Example: Like checking if your wallet is empty before paying. If the wallet exists AND has money → pay. If it's empty or missing → can't pay.

function greet(name: string | null | undefined): string {
  if (name) {
    // TS narrows: name is definitely a string here
    return `Hello, ${name.toUpperCase()}!`;
  }
  return "Hello, stranger!";
}

console.log(greet("Alice"));     // "Hello, ALICE!"
console.log(greet(null));        // "Hello, stranger!"
console.log(greet(undefined));   // "Hello, stranger!"
console.log(greet(""));          // "Hello, stranger!" (empty string is falsy!)

Guard #3

Equality Narrowing

function compare(a: string | number, b: string | boolean): void {
  if (a === b) {
    // Both must be string (the only common type!)
    console.log(a.toUpperCase());  // ✅ TS narrows to string
  }
}

// Null check with equality
function processId(id: string | null): void {
  if (id !== null) {
    console.log(id.toUpperCase());  // ✅ Definitely string
  }
}
Guard #4

in Operator — Check if Property Exists

✓

Real-Life Example: Like checking a person's ID card. If it has a "Student ID" field → they're a student. If it has an "Employee ID" field → they're an employee. You check which field exists to determine the type.

type Fish = { swim: () => void };
type Bird = { fly: () => void };

function move(animal: Fish | Bird): void {
  if ("swim" in animal) {
    animal.swim();   // ✅ TS knows it's a Fish
  } else {
    animal.fly();    // ✅ TS knows it's a Bird
  }
}

// Practical example
type Admin = { name: string; role: string; permissions: string[] };
type Guest = { name: string; visitDate: string };

function identify(user: Admin | Guest): void {
  if ("role" in user) {
    console.log(`Admin: ${user.name}, Role: ${user.role}`);
  } else {
    console.log(`Guest: ${user.name}, Visited: ${user.visitDate}`);
  }
}
Guard #5

instanceof Guard — Check Class Instances

✓

Real-Life Example: Like asking "Were you made in Factory A or Factory B?" instanceof checks which class constructor was used to create the object.

class Dog {
  bark() { console.log("Woof!"); }
}

class Cat {
  meow() { console.log("Meow!"); }
}

function makeSound(pet: Dog | Cat): void {
  if (pet instanceof Dog) {
    pet.bark();   // ✅ TS knows it's a Dog
  } else {
    pet.meow();   // ✅ TS knows it's a Cat
  }
}

makeSound(new Dog());  // "Woof!"
makeSound(new Cat());  // "Meow!"

// Works with built-in classes too!
function formatDate(value: string | Date): string {
  if (value instanceof Date) {
    return value.toISOString();  // ✅ Date methods available
  }
  return value;  // Already a string
}
Advanced

Discriminated Unions — Tagged Object Types

✓

Real-Life Example: Like parcels in a warehouse. Each parcel has a sticker: "Fragile", "Heavy", or "Perishable." The sticker tells the handler exactly how to process it without opening the box.

type Circle = { kind: "circle"; radius: number };
type Rectangle = { kind: "rectangle"; width: number; height: number };
type Triangle = { kind: "triangle"; base: number; height: number };

type Shape = Circle | Rectangle | Triangle;

function area(shape: Shape): number {
  switch (shape.kind) {
    case "circle":
      return Math.PI * shape.radius ** 2;
    case "rectangle":
      return shape.width * shape.height;
    case "triangle":
      return (shape.base * shape.height) / 2;
  }
}

console.log(area({ kind: "circle", radius: 5 }));           // 78.54
console.log(area({ kind: "rectangle", width: 4, height: 6 })); // 24
console.log(area({ kind: "triangle", base: 10, height: 3 })); // 15
i

The "kind" field (discriminant) must be a literal type and present in ALL union members. It acts as a tag that TypeScript uses to narrow the type automatically inside switch or if blocks.

Advanced

Custom Type Predicates — is Keyword

✓

Real-Life Example: Like hiring a specialist inspector. Instead of you checking if something is gold, you call a gold inspector who says "Yes, this IS gold." TypeScript trusts the inspector's verdict.

// Custom type guard function
function isString(value: unknown): value is string {
  return typeof value === "string";
}

function processInput(input: unknown): void {
  if (isString(input)) {
    console.log(input.toUpperCase());  // ✅ TS trusts our guard
  }
}

// More practical example
type User = { name: string; email: string };
type Admin = User & { role: string; permissions: string[] };

function isAdmin(user: User): user is Admin {
  return "role" in user && "permissions" in user;
}

function greet(user: User): void {
  if (isAdmin(user)) {
    console.log(`Welcome Admin ${user.name}! Role: ${user.role}`);
  } else {
    console.log(`Welcome ${user.name}!`);
  }
}
Advanced

Exhaustive Checks with never

type Shape = Circle | Rectangle | Triangle;

function area(shape: Shape): number {
  switch (shape.kind) {
    case "circle":
      return Math.PI * shape.radius ** 2;
    case "rectangle":
      return shape.width * shape.height;
    case "triangle":
      return (shape.base * shape.height) / 2;
    default:
      // If all cases are handled, this line is unreachable
      // If you ADD a new shape but forget to handle it,
      // TypeScript throws a COMPILE-TIME error here!
      const _exhaustive: never = shape;
      return _exhaustive;
  }
}
i

The never type represents "should never happen." If TypeScript can still assign a value to never, it means you missed a case — catching bugs at compile time!
