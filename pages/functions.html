What is a Function?

✓

Real-Life Example: A function is like a coffee machine. You put in ingredients (parameters), press a button (call the function), and it gives you coffee (return value). You don't rebuild the machine every time — you just press the button again. That's reusability!

①
Define
Write the code once
②
Call
Invoke it whenever needed
③
Reuse
Call it again and again!

Three types of functions in TypeScript

ℹ

Function vs Method: Same concept, different context. A function lives independently. A method lives inside a class and is accessed through objects. We'll cover methods when we learn classes.

Type 1

Named Functions

✓

Real-Life Example: A named function is like a restaurant with a signboard — it has a clear name ("Pizza Palace"), and anyone can find it and visit (call it) by that name.

Syntax

function functionName(param1: type, param2: type): returnType {
  // block of code
  return value;
}
1
No parameters, no return value
Simple
function display(): void {
  console.log("Welcome to TypeScript!");
}

display();  // Output: "Welcome to TypeScript!"
// void = this function doesn't return anything
2
With parameters and return value
Params + Return
function addNumbers(x: number, y: number): number {
  return x + y;
}

console.log(addNumbers(2, 3));  // 5
// Pass 2 params → get back a number

let result: number = addNumbers(10, 20);
console.log(result);  // 30

Must pass exactly the right number of parameters. Passing 1 or 3 → compile error!

Parameter Style 1

Rest Parameters — "Pass as many as you want"

✓

Real-Life Example: Rest parameters are like an all-you-can-eat buffet — bring 2 friends or 20 friends, the restaurant accepts everyone. There's no fixed limit on how many "guests" (values) you can pass.

Syntax: three dots ...

function functionName(...variableName: type[]): returnType {
  // variableName is an array holding ALL passed values
}
1
Sum any number of values
...rest
function addNumbers(...nums: number[]) {
  let sum = 0;
  for (let n of nums) {
    sum += n;
  }
  console.log("Sum:", sum);
}

addNumbers(1, 2);          // Sum: 3
addNumbers(1, 2, 3);       // Sum: 6
addNumbers(10, 20, 30, 40, 50);  // Sum: 150
// Pass 2 values or 50 values — it works!
2
Rest params with union types
Multi-type
function findElements(...elements: (number | string)[]): number {
  return elements.length;
}

console.log(findElements(3, "John", 42, 99, "test"));  // 5
console.log(findElements(10, 20, 30, 40, 50, 60, 70)); // 7
console.log(findElements("a", "b"));  // 2
// Accepts numbers AND strings — union type!
Parameter Style 2

Optional Parameters — "Pass it or don't"

✓

Real-Life Example: Optional parameters are like toppings on a pizza — crust and sauce are mandatory, but extra cheese is optional. You can order with or without it, and the pizza still works!

Syntax: question mark ?

function fn(required: type, optional?: type): void { }
1
Optional email parameter
?
function displayDetails(id: number, name: string, mail?: string): void {
  console.log("ID:", id);
  console.log("Name:", name);
  if (mail !== undefined) {
    console.log("Email:", mail);
  }
}

displayDetails(123, "Scott", "scott@gmail.com");
// ID: 123, Name: Scott, Email: scott@gmail.com

displayDetails(456, "John");
// ID: 456, Name: John   (no email — no error!)

If not passed, the optional parameter is undefined. Use an if check before using it!

⚠

Rule: Once you make a parameter optional, ALL parameters AFTER it must also be optional. A required parameter cannot follow an optional one!

// ✅ CORRECT — optional at the end
function fn(a: number, b: string, c?: string): void { }

// ❌ ERROR — required 'b' follows optional 'a'
function fn(a?: number, b: string): void { }

// ✅ FIX — make 'b' optional too
function fn(a?: number, b?: string): void { }
Parameter Style 3

Default Parameters — "Use this if nothing is passed"

✓

Real-Life Example: Default parameters are like a thermostat preset — it's set to 22°C by default. You CAN change it, but if you don't touch it, it stays at 22°C.

Syntax: = value

function fn(param: type = defaultValue): void { }
1
Discount calculator with default rate
= default
function calculateDiscount(price: number, rate: number = 0.50): void {
  let discount = price * rate;
  console.log("Discount:", discount);
}

calculateDiscount(1000, 0.30);  // Discount: 300  (0.30 overrides default)
calculateDiscount(1000);        // Discount: 500  (uses default 0.50)

If you pass a value → it overrides the default. If you don't → the default kicks in.

Type 2

Anonymous Functions — "No name, stored in a variable"

✓

Real-Life Example: An anonymous function is like a food truck with no signboard — it doesn't have an official name, but everyone knows it by where it parks (the variable). You visit "the truck at the corner" (the variable name) instead of a restaurant name.

Syntax

let variableName = function(params): returnType {
  // body
};

variableName();  // call using the variable name
1
Anonymous function — no params, returns string
Anonymous
let msg = function(): string {
  return "Hello TypeScript";
};

console.log(msg());  // "Hello TypeScript"
2
Anonymous function — with params
With params
let multiply = function(a: number, b: number): number {
  return a * b;
};

console.log(multiply(10, 20));  // 200

All parameter styles (rest, optional, default) work with anonymous functions too — same syntax!

Type 3 (Most Used!)

Arrow Functions / Lambda — "Short & sweet"

✓

Real-Life Example: Arrow functions are like text message abbreviations — instead of writing "laughing out loud," you write "lol." Same meaning, way shorter. Arrow functions are the shorthand version of anonymous functions.

Syntax: => (fat arrow / "goes to" operator)

let variableName = (params): returnType => {
  // body
};
// No 'function' keyword! No function name! Just =>.
1
No params, no return
Arrow
let greet = (): void => {
  console.log("Hello TypeScript!");
};

greet();  // "Hello TypeScript!"
2
With params and return
Arrow
let add = (a: number, b: number): number => {
  return a + b;
};

console.log(add(10, 20));  // 30
3
Arrow with optional, default, rest params
All styles
// Optional
let showInfo = (id: number, name: string, email?: string): void => {
  console.log(id, name);
  if (email !== undefined) console.log(email);
};

// Default
let calcDiscount = (price: number, rate: number = 0.50): void => {
  console.log("Discount:", price * rate);
};

// Rest
let findLength = (...items: (number | string)[]): number => {
  return items.length;
};
Arrow Bonus

Implicit Return — One-liner arrow functions

✓

Real-Life Example: Implicit return is like a drive-through — you don't park, go inside, order, sit down. You just drive up and get your food in ONE step. If your function has only ONE return statement, you can skip the { } and return keyword entirely!

vs
Normal vs implicit return
Shorthand
// ❌ Normal (verbose)
let add = (a: number, b: number): number => {
  return a + b;
};

// ✅ Implicit return (concise!) — same result
let add = (a: number, b: number): number => a + b;

// No { }, no 'return' keyword — just the expression!

let multiply = (a: number, b: number): number => a * b;

console.log(add(10, 20));       // 30
console.log(multiply(10, 20));  // 200

Only works when you have ONE single expression. Multiple statements still need { }.

Quick Reference Cheatsheet

Named function

function greet(name: string): string {
  return `Hello, ${name}`;
}
console.log(greet("Alice"));

Anonymous function

let greet = function(name: string): string {
  return `Hello, ${name}`;
};
console.log(greet("Bob"));

Arrow function

let greet = (name: string): string => {
  return `Hello, ${name}`;
};
// Or with implicit return:
let greet = (name: string): string => `Hello, ${name}`;

Parameter styles

// Rest — unlimited params
function sum(...nums: number[]) { }

// Optional — pass or skip
function fn(a: number, b?: string) { }

// Default — fallback value
function fn(a: number, b: number = 10) { }

Key rules

// ✅ Arrow functions = modern, preferred
// ✅ Implicit return = one expression, no { } needed
// ✅ Rest params use ... (three dots)
// ✅ Optional params use ? (question mark)
// ⚠ Optional param rule: all params AFTER an optional must also be optional
// ✅ Default params use = value
