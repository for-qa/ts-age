Basics

What is Regex?

✓

Real-Life: Regex is like a search-and-rescue dog 🐕 trained to sniff out specific patterns. You describe the pattern ("find anything that looks like a phone number"), and the dog finds every match in the text.

!
A mini-language for pattern matching
Concept
// Does this string contain a number?
const hasNumber = /\d/.test("hello123");  // true

// Find all numbers:
const nums = "price: 99, tax: 18".match(/\d+/g);  // ["99", "18"]

// Replace pattern:
const clean = "hello   world".replace(/\s+/g, " ");  // "hello world"
Basics

Creating a Regex

1
Two ways to create
Create
// 1. Literal (most common)  --  /pattern/flags
const regex1 = /hello/i;  // case-insensitive

// 2. Constructor  --  new RegExp("pattern", "flags")
const regex2 = new RegExp("hello", "i");

// Use constructor when pattern is DYNAMIC:
const searchTerm = "alice";
const dynamic = new RegExp(searchTerm, "i");
dynamic.test("Hello Alice");  // true
Basics

Character Classes

2
Match types of characters
Classes
// \d = any digit (0-9)
/\d/.test("abc3");         // true

// \w = word character (letter, digit, underscore)
/\w/.test("hello");        // true

// \s = whitespace (space, tab, newline)
/\s/.test("hello world");  // true

// . = ANY character (except newline)
/./.test("x");             // true

// NEGATED (uppercase = opposite):
// \D = NOT a digit
// \W = NOT a word character
// \S = NOT whitespace

// Character sets  --  [abc] matches a, b, or c
/[aeiou]/.test("hello");   // true (has 'e' and 'o')
/[0-9]/.test("abc5");      // true (same as \d)
/[a-zA-Z]/.test("Hello");  // true (any letter)
/[^0-9]/.test("abc");      // true (^ inside [] = NOT)
Basics

Quantifiers

3
How many times to match
Quantity
// +   = one or more
/\d+/.test("123");         // true (1+ digits)

// *   = zero or more
/\d*/.test("abc");         // true (0 digits is fine)

// ?   = zero or one (optional)
/colou?r/.test("color");   // true (u is optional)
/colou?r/.test("colour");  // true

// {n}   = exactly n times
/\d{3}/.test("123");       // true (exactly 3 digits)

// {n,m} = between n and m times
/\d{2,4}/.test("12");      // true (2 to 4 digits)

// {n,}  = n or more
/\d{3,}/.test("12345");    // true (3+ digits)
Basics

Anchors

4
Match position, not characters
Anchors
// ^ = start of string
/^hello/.test("hello world");  // true  --  starts with "hello"
/^hello/.test("say hello");    // false

// $ = end of string
/world$/.test("hello world");  // true  --  ends with "world"

// Both = exact match
/^hello$/.test("hello");       // true  --  ONLY "hello"
/^hello$/.test("hello world"); // false

// \b = word boundary
/\bcat\b/.test("the cat sat");    // true
/\bcat\b/.test("concatenate");    // false (cat is inside word)
Basics

Flags

FlagNameEffect
gGlobalFind ALL matches, not just first
iCase-insensitive/hello/i matches "HELLO"
mMultiline^ and $ match per line
5
Flags change matching behavior
Flags
// Without g  --  only first match
"aaa".match(/a/);    // ["a"]

// With g  --  all matches
"aaa".match(/a/g);   // ["a", "a", "a"]

// Case insensitive
/hello/i.test("HELLO");  // true

// Combine flags
"Hello hello HELLO".match(/hello/gi);  // ["Hello", "hello", "HELLO"]
Methods

test() / match() / replace() / split()

6
4 ways to use regex
Methods
const text = "Call 123-456 or 789-012";

// TEST  --  does it match? → boolean
/\d{3}/.test(text);         // true

// MATCH  --  find matches → array | null
text.match(/\d{3}-\d{3}/g); // ["123-456", "789-012"]

// REPLACE  --  swap matches → new string
text.replace(/\d/g, "X");   // "Call XXX-XXX or XXX-XXX"

// SPLIT  --  split by pattern → array
"one,  two,three".split(/,\s*/);  // ["one", "two", "three"]

// SEARCH  --  find position → index | -1
text.search(/\d{3}/);       // 5 (index of first match)
Practical

5 Useful Patterns

1-5
Real-world validation patterns
Patterns
// 1. Email (simple validation)
const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
emailRegex.test("user@example.com");  // true

// 2. Phone (Indian 10-digit)
const phoneRegex = /^[6-9]\d{9}$/;
phoneRegex.test("9876543210");  // true

// 3. URL
const urlRegex = /^https?:\/\/.+/;
urlRegex.test("https://example.com");  // true

// 4. Password (min 8 chars, 1 uppercase, 1 digit)
const passRegex = /^(?=.*[A-Z])(?=.*\d).{8,}$/;
passRegex.test("Hello123");   // true
passRegex.test("hello123");   // false (no uppercase)

// 5. Extract numbers from text
const numbers = "Price: ₹1,200 and ₹350".match(/\d+/g);
// ["1", "200", "350"]
Watch Out!

⚠ Common Mistakes

⚠
Mistake 1: Forgetting the g flag 
"aaa".match(/a/);   // ["a"]      --  only first!
"aaa".match(/a/g);  // ["a","a","a"]  --  all matches ✅
⚠
Mistake 2: test() vs match() -- different returns 
/\d/.test("abc1");   // true (boolean)
"abc1".match(/\d/);  // ["1"] (array)
// Don't use match() when you just need true/false!
⚠
Mistake 3: . matches ANY character -- escape it! 
/file.txt/.test("fileXtxt");   // true! . = any char
/file\.txt/.test("fileXtxt");  // false ✅ \. = literal dot
⚠
Mistake 4: Greedy vs Lazy 
// Greedy (default)  --  matches as MUCH as possible
"bold".match(/<.*>/);    // ["bold"]  --  too much!

// Lazy (add ?)  --  matches as LITTLE as possible
"bold".match(/<.*?>/);   // [""] ✅
MistakeWhat happensFix
No g flagOnly first matchAdd /g
test vs matchWrong return typetest→bool, match→array
Unescaped .Matches anythingUse \.
Greedy matchingToo much capturedUse *? or +?

Cheatsheet

// CHARACTERS
\d  digit     \D  not digit     .   any char
\w  word      \W  not word      [abc]  a or b or c
\s  space     \S  not space     [^abc] not a,b,c

// QUANTIFIERS
+  one+    *  zero+    ?  optional
{3}  exactly 3    {2,5}  2 to 5    {3,}  3+

// ANCHORS
^  start    $  end    \b  word boundary

// FLAGS
/g  global    /i  case-insensitive    /m  multiline

// METHODS
regex.test(str)       → boolean
str.match(regex)      → array | null
str.replace(regex, x) → string
str.split(regex)      → array
