Mapped

Mapped Types

✓

Real-Life: Like a photocopier with settings 🖨. You feed in a document (type), and the copier creates a new version where every page (property) is modified the same way -- make all pages read-only, all pages optional, etc.

1
Loop over properties and transform them
Mapped 
// Syntax: { [K in keyof T]: NewType }
// "For each key K in type T, create a property with NewType"

interface User {
  name: string;
  age: number;
  email: string;
}

// Make all properties optional (this IS Partial!)
type MyOptional<T> = {
  [K in keyof T]?: T[K];
};
type OptionalUser = MyOptional<User>;
// { name?: string; age?: number; email?: string; }

// Make all properties readonly
type MyReadonly<T> = {
  readonly [K in keyof T]: T[K];
};

// Make all properties nullable
type Nullable<T> = {
  [K in keyof T]: T[K] | null;
};
type NullableUser = Nullable<User>;
// { name: string | null; age: number | null; email: string | null; }
Mapped

Build Partial Yourself!

2
How built-in utilities actually work
Under the Hood 
// Partial  --  add - to every property
type Partial<T> = { [K in keyof T]?: T[K] };

// Required  --  remove - from every property
type Required<T> = { [K in keyof T]-?: T[K] };
//                                    ^^ -? removes optional

// Readonly  --  add readonly to every property
type Readonly<T> = { readonly [K in keyof T]: T[K] };

// Pick  --  keep only listed keys
type Pick<T, K extends keyof T> = { [P in K]: T[P] };

// Record  --  create from key type and value type
type Record<K extends string, V> = { [P in K]: V };

// All utility types are just mapped types! 🤯
Conditional

Conditional Types

✓

Analogy: Like an if/else for types. "If T is a string, give me X. Otherwise, give me Y."

3
T extends U - X : Y
Conditional 
// Syntax: T extends U - TrueType : FalseType

// Is it a string?
type IsString<T> = T extends string - "yes" : "no";
type A = IsString<string>;   // "yes"
type B = IsString<number>;   // "no"

// Flatten arrays (if array, get element type)
type Flatten<T> = T extends Array<infer U> - U : T;
type C = Flatten<string[]>;  // string
type D = Flatten<number>;    // number (not an array, return as-is)

// Extract non-nullable
type NonNull<T> = T extends null | undefined - never : T;
type E = NonNull<string | null | undefined>;  // string

// This is how Exclude works!
type Exclude<T, U> = T extends U - never : T;
type F = Exclude<"a" | "b" | "c", "a">;  // "b" | "c"

The infer Keyword

4
Extract types from within other types
infer 
// infer = "figure out this type for me"

// Get return type of a function
type MyReturnType<T> = T extends (...args: any[]) => infer R - R : never;
//                                                   ^^^^^^^ "infer R"
// "If T is a function, extract its return type as R"

type X = MyReturnType<() => string>;           // string
type Y = MyReturnType<(a: number) => boolean>; // boolean

// Get element type of array
type ElementOf<T> = T extends (infer E)[] - E : never;
type Z = ElementOf<number[]>;  // number

// Get promise value
type Unwrap<T> = T extends Promise<infer V> - V : T;
type W = Unwrap<Promise<string>>;  // string
type Q = Unwrap<number>;           // number

Build Your Own Utility Types

5
Combine mapped + conditional for powerful types
Custom 
// 1. Make only string properties optional
type OptionalStrings<T> = {
  [K in keyof T]: T[K] extends string - T[K] | undefined : T[K];
};

// 2. Deep Readonly (recursive!)
type DeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends object - DeepReadonly<T[K]> : T[K];
};

// 3. Make specific keys required, rest optional
type RequireKeys<T, K extends keyof T> = Required<Pick<T, K>> & Partial<Omit<T, K>>;

interface Form {
  name: string;
  email: string;
  phone: string;
  age: number;
}
type SignupForm = RequireKeys<Form, "name" | "email">;
// name & email required, phone & age optional!
Watch Out!

⚠ Common Mistakes

⚠
Mistake 1: Infinite recursion in recursive types 
// ❌ Can crash the compiler
type Bad<T> = { [K in keyof T]: Bad<T[K]> };  // infinite!

// ✅ Add a base case
type Good<T> = {
  [K in keyof T]: T[K] extends object - Good<T[K]> : T[K];
};
⚠
Mistake 2: Overly complex types

If a type takes 10 lines to read, split it into smaller named types. Your teammates need to understand it too.

⚠
Mistake 3: Forgetting distributive behavior 
// Conditional types DISTRIBUTE over unions!
type X = Exclude<"a" | "b" | "c", "a">;
// Checks each: "a" extends "a"? never : "a" → never
//              "b" extends "a"? never : "b" → "b"
//              "c" extends "a"? never : "c" → "c"
// Result: "b" | "c"

Cheatsheet

// MAPPED  --  transform all properties
{ [K in keyof T]: NewType }        // loop over keys
{ [K in keyof T]?: T[K] }         // add optional (Partial)
{ readonly [K in keyof T]: T[K] }  // add readonly
{ [K in keyof T]-?: T[K] }        // remove optional (Required)

// CONDITIONAL  --  if/else for types
T extends U - X : Y

// INFER  --  extract inner type
T extends Array<infer E> - E : never    // array element
T extends (...args: any[]) => infer R - R : never  // return type
T extends Promise<infer V> - V : T      // unwrap promise
