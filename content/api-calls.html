What is an API Call?

✓

Real-Life: An API call is like ordering from a restaurant menu 🍽. You (the client) send an order (request) to the kitchen (server). The kitchen prepares the food and sends it back (response). You don't need to know HOW the kitchen works -- just what's on the menu.

Basics

GET Requests -- Fetch Data

1
The simplest API call
GET 
// Basic GET  --  fetch returns a Promise
const response = await fetch("https://jsonplaceholder.typicode.com/posts/1");
const data = await response.json();
console.log(data);

// With query parameters:
const page = 1;
const limit = 10;
const url = `https://api.example.com/users?page=${page}&limit=${limit}`;
const res = await fetch(url);

// With headers (e.g., auth token):
const res2 = await fetch("https://api.example.com/me", {
  headers: {
    "Authorization": "Bearer YOUR_TOKEN_HERE",
    "Content-Type": "application/json",
  },
});

POST / PUT / DELETE

2
Send data to the server
POST/PUT/DELETE 
// POST  --  create new resource
const newUser = { name: "Alice", email: "alice@example.com" };
const res = await fetch("https://api.example.com/users", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify(newUser),  // must stringify!
});
const created = await res.json();

// PUT  --  update existing resource
await fetch("https://api.example.com/users/1", {
  method: "PUT",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ name: "Alice Updated" }),
});

// DELETE  --  remove resource
await fetch("https://api.example.com/users/1", {
  method: "DELETE",
});

Typing API Responses

3
Type safety for API data
Typed fetch 
// Define the shape of the API response
interface Post {
  id: number;
  title: string;
  body: string;
  userId: number;
}

// ✅ Type the parsed JSON
async function getPost(id: number): Promise<Post> {
  const res = await fetch(`https://jsonplaceholder.typicode.com/posts/${id}`);
  const data: Post = await res.json();
  return data;
}

const post = await getPost(1);
console.log(post.title);  // autocomplete works! ✅

// ✅ Generic fetch wrapper
async function fetchJson<T>(url: string): Promise<T> {
  const res = await fetch(url);
  if (!res.ok) throw new Error(`HTTP ${res.status}`);
  return res.json() as Promise<T>;
}

const posts = await fetchJson<Post[]>("https://jsonplaceholder.typicode.com/posts");

Error Handling

4
fetch doesn't throw on 404/500!
⚠ Critical 
// ⚠ fetch only throws on NETWORK errors (no internet)
// It does NOT throw on 404, 500, etc.!

// ❌ WRONG  --  doesn't catch HTTP errors
try {
  const res = await fetch("/api/users/999");
  const data = await res.json();  // might be error HTML!
} catch (err) {
  // Only catches network failures, NOT 404!
}

// ✅ CORRECT  --  check response.ok
async function safeFetch<T>(url: string): Promise<T> {
  try {
    const res = await fetch(url);

    if (!res.ok) {
      throw new Error(`HTTP Error: ${res.status} ${res.statusText}`);
    }

    return await res.json() as T;
  } catch (err) {
    if (err instanceof TypeError) {
      throw new Error("Network error  --  check your connection");
    }
    throw err;
  }
}

Critical: response.ok is true for status 200-299, false for 400+. ALWAYS check it.

Loading States Pattern

5
Track loading, data, and error
Pattern 
// Common pattern for UI state management
interface ApiState<T> {
  data: T | null;
  loading: boolean;
  error: string | null;
}

async function loadUsers(): Promise<void> {
  const state: ApiState<User[]> = { data: null, loading: true, error: null };

  try {
    const res = await fetch("/api/users");
    if (!res.ok) throw new Error(`Error: ${res.status}`);
    state.data = await res.json();
  } catch (err) {
    state.error = err instanceof Error ? err.message : "Unknown error";
  } finally {
    state.loading = false;
  }

  // Use state:
  if (state.loading) console.log("Loading...");
  if (state.error) console.log("Error:", state.error);
  if (state.data) console.log("Users:", state.data);
}

⚠ Common Mistakes

⚠
Mistake 1: Not awaiting fetch 
// ❌ Forgot await  --  data is a Promise, not the value!
const data = fetch("/api/users");  // Promise, not data!

// ✅ Await both fetch AND .json()
const res = await fetch("/api/users");
const data = await res.json();
⚠
Mistake 2: Not checking response.ok 
// ❌ fetch doesn't throw on 404!
const res = await fetch("/api/missing");
const data = await res.json();  // might be error page HTML!

// ✅ Always check
if (!res.ok) throw new Error(`${res.status}`);
⚠
Mistake 3: Not typing the response 
// ❌ res.json() returns 'any'  --  no type safety
const data = await res.json();
data.whateverrrr;  // no error! 😱

// ✅ Type it
const data: User[] = await res.json();
⚠
Mistake 4: Forgetting Content-Type header on POST 
// ❌ Server doesn't know it's JSON!
await fetch(url, { method: "POST", body: JSON.stringify(data) });

// ✅ Tell the server
await fetch(url, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify(data),
});

No res.ok check 404 errors undetected if (!res.ok) throw
