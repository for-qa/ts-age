Foundation

Synchronous vs Asynchronous

✓

Real-Life Example: Synchronous = standing in a queue at a bank. You wait for each person to finish before you can go. Asynchronous = ordering food at a restaurant. You order, then chat with friends while the kitchen prepares your food. You don't wait idle. 

// Synchronous  --  each line waits for the previous one
console.log("Step 1");  // runs first
console.log("Step 2");  // runs second
console.log("Step 3");  // runs third
// Output: Step 1, Step 2, Step 3 (in order)

// Asynchronous  --  some tasks run "in the background"
console.log("Step 1");
setTimeout(() => console.log("Step 2 (delayed)"), 2000);
console.log("Step 3");
// Output: Step 1, Step 3, Step 2 (delayed)
// Step 3 runs before Step 2 because setTimeout is async!
Synchronous Asynchronous
Runs line by line, in order Some tasks run in background
Blocks until each line completes Doesn't block -- continues executing
Simple but slow for I/O Complex but fast for I/O
Regular function calls API calls, timers, file reads
Old Way

Callbacks -- The Original Approach

✓

Real-Life Example: Like leaving your phone number at a busy restaurant: "Call me BACK when my table is ready." You give them a function (your number) to execute later. 

// Callback pattern  --  pass a function to call when done
function fetchUser(id: number, callback: (user: string) => void): void {
  setTimeout(() => {
    callback(`User_${id}`);  // "call back" with the result
  }, 1000);
}

fetchUser(1, (user) => {
  console.log(`Got: ${user}`);  // "Got: User_1" (after 1 second)
});
// ❌ Callback Hell  --  nested callbacks become unreadable!
fetchUser(1, (user) => {
  fetchOrders(user, (orders) => {
    fetchDetails(orders[0], (details) => {
      processPayment(details, (result) => {
        console.log(result);  // ✓ 4 levels deep!
      });
    });
  });
});
// This is why Promises were invented!
⚠

Callback hell! Deeply nested callbacks are hard to read, debug, and maintain. Promises and async/await solve this problem elegantly.

Promises

What is a Promise?

✓

Real-Life Example: A promise is like an online order tracking number. When you place an order, you get a tracking number (promise). The order is either: still processing (pending), delivered successfully (fulfilled/resolved), or failed (rejected).

A Promise is an object representing the eventual result of an asynchronous operation. It has three states:

State Meaning Handler
Pending Operation still in progress --
Fulfilled Operation succeeded → has a value .then()
Rejected Operation failed → has an error .catch() 
// Promise flow
//  Pending → Fulfilled (success) → .then() handles it
//  Pending → Rejected  (failure) → .catch() handles it
Creating

Creating a Promise

// Create a new Promise with TypeScript typing
let myPromise: Promise<string> = new Promise((resolve, reject) => {
  let success = true;

  if (success) {
    resolve("Operation completed!");   // → Fulfilled
  } else {
    reject("Something went wrong!");   // → Rejected
  }
});

// Handle the result
myPromise
  .then((result) => console.log(result))      // if fulfilled
  .catch((error) => console.log(error));       // if rejected
// Output: "Operation completed!"
// Simulating an API call
function fetchUser(id: number): Promise<{ name: string; age: number }> {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      if (id > 0) {
        resolve({ name: "Alice", age: 25 });
      } else {
        reject("Invalid user ID");
      }
    }, 1000);
  });
}

fetchUser(1)
  .then((user) => console.log(`Name: ${user.name}`))
  .catch((err) => console.log(`Error: ${err}`));
// Output after 1 second: "Name: Alice"
Chaining

Chaining .then() -- Sequential Steps

✓

Real-Life Example: Like an assembly line. Step 1: cut the fabric. Step 2: stitch it. Step 3: add buttons. Each step passes its result to the next. 

fetchUser(1)
  .then((user) => {
    console.log(`Got user: ${user.name}`);
    return user.name.toUpperCase();     // pass to next .then()
  })
  .then((upperName) => {
    console.log(`Uppercase: ${upperName}`);
    return upperName.length;            // pass to next .then()
  })
  .then((length) => {
    console.log(`Name length: ${length}`);
  })
  .catch((error) => {
    console.log(`Error: ${error}`);     // catches ANY error in chain
  })
  .finally(() => {
    console.log("Done!");               // runs always (success or fail)
  });
Modern Syntax

async / await -- Cleaner Promises

✓

Real-Life Example: async/await is like telling a friend: "Go order my coffee. I'll wait right here until you come back." It makes asynchronous code LOOK and READ like synchronous code.

async marks a function as asynchronous. await pauses execution until the promise resolves. 

// ❌ With .then() chains (harder to read)
fetchUser(1)
  .then(user => fetchOrders(user.name))
  .then(orders => processOrder(orders[0]))
  .then(result => console.log(result))
  .catch(err => console.log(err));

// ✅ With async/await (reads like normal code!)
async function processUserOrder(): Promise<void> {
  const user = await fetchUser(1);
  const orders = await fetchOrders(user.name);
  const result = await processOrder(orders[0]);
  console.log(result);
}
Rule Detail
async Put before function keyword -- function now returns a Promise
await Put before a Promise -- pauses until it resolves
await location Can ONLY be used inside an async function
Return type async functions always return Promise<T> 
// async function always returns a Promise
async function getMessage(): Promise<string> {
  return "Hello!";  // automatically wrapped in Promise.resolve()
}

// await extracts the value from the Promise
async function main(): Promise<void> {
  const msg = await getMessage();  // msg is string, not Promise!
  console.log(msg);                // "Hello!"
}
Error Handling

Error Handling with try/catch

✓

Real-Life Example: Like wearing a safety net while tightrope walking. try = walk the rope. catch = the net catches you if you fall. finally = you get down from the platform either way. 

async function loadUserData(id: number): Promise<void> {
  try {
    const user = await fetchUser(id);        // might fail!
    const orders = await fetchOrders(user.name); // might fail!
    console.log(`${user.name} has ${orders.length} orders`);
  } catch (error) {
    // Catches ANY error from the try block
    console.log(`Failed: ${error}`);
  } finally {
    // Runs ALWAYS  --  success or failure
    console.log("Loading complete");
  }
}

loadUserData(1);   // Success path
loadUserData(-1);  // Error path → catch block runs
⚠

Always wrap await in try/catch! Without it, rejected promises will crash your application with an unhandled promise rejection error.

Utility

Promise.all() -- Run in Parallel

✓

Real-Life Example: Like ordering 3 dishes at the same time instead of one after another. All kitchens work simultaneously. You get everything when the LAST dish is ready. But if ANY dish fails → the entire order is cancelled. 

async function loadDashboard(): Promise<void> {
  try {
    // All three run at the SAME TIME (parallel)
    const results = await Promise.all([
      fetchUser(1),
      fetchOrders("Alice"),
      fetchNotifications("Alice")
    ]);
    const user = results[0];
    const orders = results[1];
    const notifications = results[2];

    console.log(user, orders, notifications);
  } catch (error) {
    // If ANY one fails, entire Promise.all rejects
    console.log(`Dashboard failed: ${error}`);
  }
}

// Sequential (slow): 3 seconds total
// Parallel with Promise.all: ~1 second total (all run together!)
Utility

Promise.race() -- First One Wins

✓

Real-Life Example: Like a 100m sprint. Multiple runners start together. Whoever finishes FIRST wins -- the rest don't matter. 

// Timeout pattern  --  cancel if too slow
function timeout(ms: number): Promise<never> {
  return new Promise((_, reject) =>
    setTimeout(() => reject("Timeout!"), ms)
  );
}

async function fetchWithTimeout(): Promise<void> {
  try {
    const result = await Promise.race([
      fetchUser(1),      // actual API call
      timeout(3000)      // 3 second limit
    ]);
    console.log(result);
  } catch (error) {
    console.log(error);  // "Timeout!" if API is too slow
  }
}
Method Waits for Fails when
Promise.all() ALL to finish ANY one rejects
Promise.race() FIRST to finish FIRST one rejects
Promise.allSettled() ALL to finish Never fails
Promise.any() FIRST success ALL reject

Real-World Patterns

1
API Data Fetching
Pattern 
interface User {
  id: number;
  name: string;
  email: string;
}

async function getUser(id: number): Promise<User> {
  const response = await fetch(`https://api.example.com/users/${id}`);

  if (!response.ok) {
    throw new Error(`HTTP ${response.status}`);
  }

  const data: User = await response.json();
  return data;
}

// Usage
async function main() {
  try {
    const user = await getUser(1);
    console.log(`Welcome, ${user.name}!`);
  } catch (error) {
    console.log(`Failed to load user: ${error}`);
  }
}
2
Automation -- Wait for Element
Testing 
async function waitForElement(
  selector: string,
  timeoutMs: number = 5000
): Promise<Element> {
  const start = Date.now();

  while (Date.now() - start < timeoutMs) {
    const element = document.querySelector(selector);
    if (element) return element;
    await new Promise(r => setTimeout(r, 100)); // wait 100ms
  }

  throw new Error(`Element ${selector} not found in ${timeoutMs}ms`);
}

// Usage in a test
async function testLogin(): Promise<void> {
  try {
    const button = await waitForElement("#login-btn");
    console.log("Login button found!");
  } catch (error) {
    console.log(error);  // timeout error
  }
}

Quick Reference Cheatsheet

Create a promise

const p = new Promise<string>((resolve, reject) => {
  resolve("success");  // or reject("error");
});

Consume with .then/.catch

p.then(val => console.log(val))
 .catch(err => console.log(err))
 .finally(() => console.log("done"));

async/await

async function getData(): Promise<string> {
  const result = await someAsyncOperation();
  return result;
}

Error handling

try { await riskyOp(); }
catch (error) { console.log(error); }
finally { cleanup(); }

Parallel execution

const results = await Promise.all([p1, p2, p3]);
// results[0] = p1 result, results[1] = p2 result, results[2] = p3 result

Key rules

// ✅ async function always returns Promise<T>
// ✅ await can ONLY be used inside async functions
// ✅ Always wrap await in try/catch for error handling
// ✅ Promise.all = parallel execution (faster!)
// ✅ Promise.race = first one wins (timeout patterns)
// ❌ Don't forget error handling  --  unhandled rejections crash apps
