TypeScript Glossary

Every key term explained in plain English with a real-life analogy and a code snippet. Grouped by topic, sorted A-Z within each group.

Terms
8
Topics
Variables

const

Real life: A name carved in stone -- you can't erase and write a new name.

Declares a variable that cannot be reassigned. Objects/arrays declared with const can still have their contents changed. 

const PI = 3.14;
// PI = 3.15; // Error! Cannot reassign
const arr = [1, 2];
arr.push(3); // OK  --  contents can change

Hoisting

Real life: A student who raises their hand before the teacher calls on them -- JS moves var and function declarations to the top. 

console.log(x); // undefined (hoisted, but no value yet)
var x = 10;

// let/const are NOT hoisted  --  they throw ReferenceError

let

Real life: A notebook page -- you can update it, but it stays within its chapter (block scope). 

let count = 0;
count = 5;     // OK  --  can reassign
// let count = 10; // Error! Cannot redeclare

Scope

Real life: Rooms in a house -- a variable in the kitchen can't be seen from the bedroom. 

if (true) {
  let x = 10; // only visible inside this { }
}
// console.log(x); // Error! x is not defined here

var

Real life: An old whiteboard in a shared room -- anyone can see and erase it. Avoid in modern code. 

var x = 10;   // function-scoped, hoisted, can redeclare
// Problems: leaks out of blocks, gets hoisted
// Always use let or const instead
Data Types

any

Real life: A bag with no label -- could be anything, no safety checks.

any disables type checking. Use it sparingly; prefer unknown for safety. 

let x: any = "hello";
x = 42;       // no error
x.foo.bar();  // no error at compile  --  crashes at runtime!

bigint

Real life: A calculator with infinite digits -- for numbers too large for number to handle precisely (beyond 2⁵³). 

const huge: bigint = 9007199254740993n; // note the 'n' suffix
const also: bigint = BigInt("9007199254740993");
// Cannot mix bigint and number in math:
// huge + 1; // ❌ Error! Use huge + 1n instead

Immutable

Real life: A printed book -- you can't change the words. You can only buy a new edition. 

let name = "hello";
name.toUpperCase(); // returns "HELLO" but name is still "hello"
// Strings, numbers, booleans are immutable

never

Real life: A black hole -- nothing ever comes out. Used for functions that never return (throw or infinite loop) and exhaustive checks. 

function throwError(msg: string): never {
  throw new Error(msg); // never returns
}

// Exhaustive check  --  if you reach this, you missed a case
type Shape = "circle" | "square";
function area(s: Shape): number {
  switch (s) {
    case "circle": return 1;
    case "square": return 2;
    default:
      const _exhaustive: never = s; // TS error if a case is missing
      return _exhaustive;
  }
}

null / undefined

Real life: undefined = empty parking spot (nobody parked). null = spot with "Reserved" sign (intentionally empty). 

let x;              // undefined  --  never assigned
let y: string | null = null; // null  --  intentionally empty

symbol

Real life: A unique serial number stamped on each item -- even two symbols with the same description are never equal. 

const id1: symbol = Symbol("id");
const id2: symbol = Symbol("id");
console.log(id1 === id2); // false  --  always unique!

// Used as unique object keys:
const KEY = Symbol("secret");
const obj = { [KEY]: "hidden value" };
console.log(obj[KEY]); // "hidden value"

typeof

Real life: Asking "what kind of animal is this?" before deciding how to handle it. 

let val: unknown = "hello";
if (typeof val === "string") {
  console.log(val.toUpperCase()); // safe!
}
// typeof returns: "string", "number", "bigint", "boolean",
//                 "symbol", "undefined", "object", "function"

unknown

Real life: A new employee -- must show ID before accessing any room. Safer than any. 

let val: unknown = "hello";
// val.toUpperCase(); // Error! Must check type first
if (typeof val === "string") {
  val.toUpperCase(); // OK after check
}

void

Real life: A doorbell -- it does something (rings) but doesn't give you anything back. 

function logMessage(msg: string): void {
  console.log(msg); // does something, returns nothing
}
Strings & Arrays

Array

Real life: A grocery list -- items in order, each with a position number starting from 0. 

let fruits: string[] = ["apple", "banana", "cherry"];
console.log(fruits[0]); // "apple"
fruits.push("date");    // add to end

Destructuring

Real life: Unpacking a gift box -- you pull out each item and give it its own name. 

const user = { name: "Alice", age: 25 };
const { name, age } = user; // unpack!
console.log(name); // "Alice"

const [first, second] = [10, 20]; // array version

Rest Parameters (...)

Real life: A buffet -- take as many items as you want; they all go on one plate (array). 

function sum(...nums: number[]): number {
  return nums.reduce((a, b) => a + b, 0);
}
sum(1, 2, 3, 4); // 10

Spread Operator (...)

Real life: Pouring a box of LEGO onto the floor -- all pieces spread out individually. 

const a = [1, 2];
const b = [...a, 3, 4]; // [1, 2, 3, 4]

const obj = { x: 1 };
const obj2 = { ...obj, y: 2 }; // { x: 1, y: 2 }

Template Literal

Real life: A fill-in-the-blank form -- "Hello, ____!" where you plug in the name. 

const name = "Alice";
console.log(`Hello, ${name}!`); // "Hello, Alice!"
// Use backticks (`) not quotes

Tuple

Real life: A fixed meal combo -- exactly 1 burger, 1 fries, 1 drink. No more, no less, in that order. 

let meal: [string, number, boolean] = ["Burger", 299, true];
// Position 0 = string, 1 = number, 2 = boolean
Operators

Ternary Operator

Real life: A quick yes/no decision -- "Is it raining? Take umbrella : wear sunglasses." 

const age = 20;
const status = age >= 18 - "adult" : "minor";
// Same as: if (age >= 18) { ... } else { ... }
Functions

Arrow Function

Real life: A sticky note -- short, quick, borrows context from its surroundings. 

const add = (a: number, b: number) => a + b;
// Same as: function add(a, b) { return a + b; }
// Key difference: arrow inherits 'this' from parent

Callback

Real life: Giving your phone number to a shop -- "call me BACK when my order is ready." 

function greet(name: string, done: () => void) {
  console.log("Hi " + name);
  done(); // call the callback
}
greet("Alice", () => console.log("Done!"));

Function

Real life: A coffee machine -- put in ingredients (parameters), get coffee (return value). 

function greet(name: string): string {
  return "Hello, " + name;
}
greet("Bob"); // "Hello, Bob"

Optional (?)

Real life: A pizza order form -- name is required, but "extra toppings" is optional. 

function greet(name: string, title?: string) {
  console.log("Hi " + (title || "") + " " + name);
}
greet("Alice");          // OK
greet("Alice", "Dr.");   // also OK

Overloading

Real life: A Swiss Army knife -- one tool, many uses depending on what you need. 

function format(val: string): string;
function format(val: number): string;
function format(val: any): string {
  return typeof val === "string" - val.trim() : val.toFixed(2);
}
Type System

Enum

Real life: A TV remote with fixed buttons -- Power, Volume, Channel. Only these options exist. 

enum Color { Red, Green, Blue }
let fav: Color = Color.Green; // 1
enum Status { Active = "ACTIVE", Inactive = "INACTIVE" }

Generic <T>

Real life: A universal bottle -- it works with water, juice, or milk. You decide what goes in. 

function wrap<T>(value: T): T[] {
  return [value];
}
wrap("hi");  // string[]
wrap(42);    // number[]

Type Inference

Real life: A smart friend who guesses your order -- "You always get coffee, so I ordered it for you." 

let x = 10;       // TS infers: number
let y = "hello";  // TS infers: string
// No need to write: let x: number = 10;

Intersection Type (&)

Real life: A person who is BOTH a doctor AND a teacher -- has skills of both. 

type HasName = { name: string };
type HasAge = { age: number };
type Person = HasName & HasAge; // must have BOTH
const p: Person = { name: "Alice", age: 25 };

Literal Type

Real life: A vending machine with only 3 buttons -- you can ONLY pick from those exact options. 

type Direction = "up" | "down" | "left" | "right";
let d: Direction = "up"; // OK
// let d2: Direction = "diagonal"; // Error!

Narrowing

Real life: Airport security -- checking your ID narrows "unknown person" to "verified passenger." 

function print(val: string | number) {
  if (typeof val === "string") {
    console.log(val.toUpperCase()); // TS knows it's string
  } else {
    console.log(val.toFixed(2));    // TS knows it's number
  }
}

type (alias)

Real life: A nickname -- instead of saying "string or number" every time, you say "ID". 

type ID = string | number;
type Point = { x: number; y: number };
let userId: ID = "abc123";

Union Type (|)

Real life: A gift that can be a book OR a toy -- one value, multiple possible types. 

let id: string | number;
id = "abc";  // OK
id = 123;    // also OK
// id = true; // Error! boolean not in the union

Utility Types

Real life: Pre-made templates -- instead of building from scratch, use a ready-made form and customize. 

interface User { name: string; age: number; email: string; }

Partial<User>   // all fields optional
Required<User>  // all fields required
Pick<User, "name" | "age">  // only name & age
Omit<User, "email">         // everything except email
Record<string, number>      // { [key: string]: number }
OOP

abstract

Real life: A recipe title with no instructions -- someone else must write the steps.

An abstract class or method defines the shape but cannot be used directly. Child classes must provide the actual code. 

abstract class Shape {
  abstract area(): number; // no body  --  child MUST implement
}
class Circle extends Shape {
  constructor(public r: number) { super(); }
  area() { return Math.PI * this.r ** 2; }
}

Class

Real life: A cookie cutter -- one mold, many cookies. The class is the blueprint, objects are the cookies. 

class Car {
  constructor(public brand: string, public speed: number) {}
  drive() { console.log(this.brand + " goes vroom!"); }
}
const myCar = new Car("Toyota", 120);

Constructor

Real life: The factory assembly line -- runs once when a new product (object) is built. 

class Student {
  name: string;
  constructor(name: string) {
    this.name = name; // runs when you say "new Student()"
  }
}

extends

Real life: A child inheriting traits from a parent -- you GET everything automatically. 

class Animal { move() { console.log("moving"); } }
class Dog extends Animal { bark() { console.log("woof!"); } }
// Dog has BOTH move() and bark()

implements

Real life: Signing a contract -- you PROMISE to provide everything listed. 

interface Printable { print(): void; }
class Report implements Printable {
  print() { console.log("Printing report..."); } // MUST provide
}

Interface

Real life: A job description -- it lists what skills are needed but doesn't do the work itself. 

interface User {
  name: string;
  age: number;
  email?: string; // optional
}
const u: User = { name: "Alice", age: 25 };

Method

Real life: An action a person can do -- a chef can cook(), a driver can drive(). 

class Dog {
  bark() { console.log("Woof!"); } // method = function inside a class
}
new Dog().bark();

Object

Real life: A person's ID card -- has properties (name, age) and actions (walk, talk). 

const user = {
  name: "Alice",
  age: 25,
  greet() { console.log("Hi!"); }
};

readonly

Real life: A museum exhibit -- look but don't touch. You can read the value but can't change it. 

class User {
  readonly id: number;
  constructor(id: number) { this.id = id; }
}
const u = new User(1);
// u.id = 2; // Error! readonly

static

Real life: A school's principal -- belongs to the school (class) itself, not to any single student (instance). 

class MathHelper {
  static PI = 3.14159;
  static square(n: number) { return n * n; }
}
MathHelper.square(5); // 25  --  no "new" needed

super

Real life: Calling your parent for help -- "Hey Mom, I need your recipe before I add my twist." 

class Animal {
  constructor(public name: string) {}
}
class Dog extends Animal {
  constructor(name: string, public breed: string) {
    super(name); // calls parent's constructor FIRST
  }
}
Modules & Async

async / await

Real life: Ordering food at a restaurant -- you place the order (async), do other things, and the food arrives later (await). 

async function fetchUser(id: number) {
  const res = await fetch(`/api/users/${id}`); // waits here
  return await res.json(); // then continues
}

export / import

Real life: A library lending system -- the library exports books, you import the ones you need. 

// utils.ts
export function add(a: number, b: number) { return a + b; }

// app.ts
import { add } from "./utils";
console.log(add(2, 3)); // 5

Module

Real life: Separate LEGO boxes -- each box has specific pieces. You pick the box you need.

Any file with export or import is a module. Keeps code organized and reusable.

Promise

Real life: A food delivery order -- pending (cooking), fulfilled (delivered), or rejected (cancelled). 

const p = new Promise<string>((resolve, reject) => {
  setTimeout(() => resolve("Done!"), 1000);
});
p.then(msg => console.log(msg)); // "Done!" after 1 second
