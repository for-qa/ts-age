What are Operators?

✓

Real-Life Example: Operators are like tools in a toolbox. Variables are the raw materials (wood, nails, screws). Data types tell you what KIND of material you have. Operators are the tools -- a hammer (+), a saw (−), a measuring tape (>), a level (===) -- that let you DO things with those materials.

An operator is a symbol that tells TypeScript to perform a specific action on variables or values. Without operators, variables just sit there doing nothing!

Category ①

Arithmetic Operators -- Math Operations

✓

Real-Life Example: Arithmetic operators are like the buttons on a calculator. You punch in two numbers, press an operator button, and get a result. Addition (+), subtraction (−), multiplication (×), division (÷) -- same buttons, same logic.

Operator Name Example Result
+ Addition 10 + 20 30
- Subtraction 20 - 10 10
* Multiplication 10 * 20 200
/ Division (quotient) 20 / 10 2
% Modular division (remainder) 20 % 10 0
** Exponentiation (power) 5 ** 2 25
1
All arithmetic operators in action
Calculator 
let a: number = 10, b: number = 20;

console.log(a + b);   // 30    --  Addition
console.log(b - a);   // 10    --  Subtraction
console.log(a * b);   // 200   --  Multiplication
console.log(b / a);   // 2     --  Division (quotient)
console.log(b % a);   // 0     --  Modular division (remainder)
console.log(5 ** 2);  // 25    --  5 to the power of 2 (5 × 5)
i

Division vs Modular: / gives you the quotient (how many times it fits). % gives you the remainder (what's left over). Think: 20 ÷ 10 = 2 remainder 0.

Category ②

Assignment Operators -- Shortcuts

✓

Real-Life Example: Assignment operators are like text shortcuts on your phone. Instead of typing "On my way," you type "omw" and it auto-expands. Instead of writing a = a + b, you write a += b. Same result, less typing!

The basic assignment operator is =. But TypeScript also has shorthand operators that combine an arithmetic operation with assignment:

Shorthand Full version What it does
a += b a = a + b Add b to a, store in a
a -= b a = a - b Subtract b from a, store in a
a *= b a = a * b Multiply a by b, store in a
a /= b a = a / b Divide a by b, store in a
a %= b a = a % b Remainder of a÷b, store in a
1
Shorthand operators -- step by step
Shortcuts 
let a: number = 10, b: number = 5;

a += b;   // a = 10 + 5  → a is now 15
console.log(a);  // 15

a -= b;   // a = 15 - 5  → a is now 10
console.log(a);  // 10

a *= b;   // a = 10 * 5  → a is now 50
console.log(a);  // 50

a /= b;   // a = 50 / 5  → a is now 10
console.log(a);  // 10

a %= b;   // a = 10 % 5  → a is now 0 (no remainder)
console.log(a);  // 0

⚠ Each line CHANGES the value of a. The next line uses the NEW value, not the original. Track the value step by step!

Category ③

Relational / Comparison Operators

✓

Real-Life Example: Comparison operators are like a referee at a sports event. The referee compares two players and says "Yes, Player A scored more" (true) or "No, Player B didn't win" (false). The result is always a clear yes or no -- never "maybe."

Relational operators compare two values and always return a boolean -- either true or false. Nothing else.

Operator Meaning Example (a=10, b=20) Result
> Greater than a > b false
< Less than a < b true
>= Greater than or equal a >= b false
<= Less than or equal a <= b true
== Equal to (value only) a == b false
!= Not equal to a != b true
=== Strict equal (value + type) see next section --
1
All comparison operators
true / false 
let a: number = 10, b: number = 20;

console.log(a > b);   // false  --  10 is NOT greater than 20
console.log(a < b);   // true   --  10 IS less than 20
console.log(a <= b);  // true   --  10 is less than OR equal to 20
console.log(a >= b);  // false  --  10 is NOT greater than or equal to 20
console.log(a == b);  // false  --  10 is NOT equal to 20
console.log(a != b);  // true   --  10 IS not equal to 20
⚠

Common Mistake: Don't confuse = (assignment -- puts a value IN) with == (comparison -- checks if values are EQUAL). Single = assigns. Double == compares!

Category ③ -- Special

== vs === -- Equality vs Strict Equality

✓

Real-Life Example:
✓ == (Equality) = Asking "Do these two containers have the same amount?" -- A glass with 500ml and a bottle with 500ml → "Yes, same amount!" (true). It only checks the value.
✓ === (Strict Equality) = Asking "Are these the exact same container?" -- A glass with 500ml and a bottle with 500ml → "No! One's a glass, the other's a bottle!" (false). It checks value AND type.

==
Equality -- compares VALUES only
⚠ Loose 
let num1: any = 10;     // number
let num2: any = "10";   // string (same value, different type!)

console.log(num1 == num2);  // ✅ true
// ✓ == only checks: is 10 equal to 10? YES!
// It ignores that one is a number and the other is a string.
===
Strict Equality -- compares VALUES + TYPES
✅ Strict 
let num1: any = 10;     // number
let num2: any = "10";   // string

console.log(num1 === num2);  // ❌ false
// ✓ === checks TWO things:
//   1. Are the values equal?  10 == 10 → YES ✅
//   2. Are the types equal?   number == string → NO ❌
// Since types don't match → false!
Operator Name Checks 10 == "10" 10 === "10"
== Equality Value only true --
=== Strict Equality Value + Type -- false
⚠

Interview Question: "What's the difference between == and ===?" -- == compares values only. === compares values AND data types. The === operator is specific to JavaScript/TypeScript -- most other languages don't have it. Always prefer === for safer comparisons.

Category ④

Logical Operators -- AND, OR, NOT

✓

Real-Life Example:
✓ AND (&&) = To enter a VIP club, you need BOTH: a ticket AND valid ID. Missing either one? Rejected!
✓ OR (||) = To pay, you can use cash OR card. Either one works -- you only need ONE.
✓ NOT (!) = A light switch. If it's ON, NOT makes it OFF. If it's OFF, NOT makes it ON. Flip it!

Logical operators work with boolean values (true/false) and return a boolean result.

! NOT Flips the value. true → false. false → true. Exclamation

Category ④ -- Advanced

Combining Logical + Relational Operators

✓

Real-Life Example: "Can I ride this roller coaster?" The rule: you must be taller than 120cm AND older than 10. That's a relational check (>) combined with a logical check (&&). Both conditions must be true!

You can mix relational operators (which return true/false) with logical operators to build complex conditions:

1
Mixing relational + logical
Combo 
// Both comparisons return true → AND returns true
console.log(20 > 10 && 10 > 5);   // true && true → true

// First is true, second is false → OR still returns true
console.log(10 < 20 || 5 > 10);   // true || false → true

// Real-world example: checking if someone can vote
let age: number = 20;
let isCitizen: boolean = true;

// Must be 18+ AND a citizen to vote
let canVote: boolean = (age >= 18) && isCitizen;
console.log("Can vote:", canVote);  // true
i

This combination is commonly used in if statements and loops -- which you'll learn in upcoming topics. For now, just understand how the operators work together.

Category ⑤

Increment & Decrement Operators

✓

Real-Life Example:
➾ ++ (Increment) = A clicker counter at a museum entrance. Every person who walks in -- click! The count goes up by exactly 1.
➓ -- (Decrement) = The same counter but for people leaving. Each exit -- click! The count goes down by exactly 1.

Operator Name What it does Equivalent to
x++ Post-increment Increases x by 1 x = x + 1
++x Pre-increment Increases x by 1 x = x + 1
x-- Post-decrement Decreases x by 1 x = x - 1
--x Pre-decrement Decreases x by 1 x = x - 1
1
Basic increment & decrement -- no difference when used alone
Simple 
let x: number = 10;

x++;              // x = 10 + 1 = 11
console.log(x);   // 11

x--;              // x = 11 - 1 = 10
console.log(x);   // 10

// When used alone, x++ and ++x give the same result.
// The difference ONLY appears when you assign to another variable.
Category ⑤ -- Important!

Pre vs Post -- When the Order Matters

✓

Real-Life Example:
✓ Post-increment (x++) = A mailman delivers your mail FIRST, then takes the next letter from the pile. Deliver first, pick up next. → Assign old value, THEN increment.
✓ Pre-increment (++x) = The mailman picks up the next letter FIRST, then delivers it. Pick up first, then deliver. → Increment first, THEN assign.

The difference between x++ and ++x ONLY matters when you store the result in another variable:

①
Post-increment (x++) -- assign FIRST, increment AFTER
Old value saved 
let x: number = 10;
let result: number = x++;  // Step 1: result = 10 (old value)
                             // Step 2: x becomes 11

console.log(result);  // 10 ← got the OLD value!
console.log(x);       // 11 ← x was incremented AFTER assignment
②
Pre-increment (++x) -- increment FIRST, assign AFTER
New value saved 
let x: number = 10;
let result: number = ++x;  // Step 1: x becomes 11 (increment first!)
                             // Step 2: result = 11 (new value)

console.log(result);  // 11 ← got the NEW value!
console.log(x);       // 11 ← same, already incremented
Type Syntax Order of operations result gets
Post-increment result = x++ ① Assign old value → ② Increment Old value (10)
Pre-increment result = ++x ① Increment → ② Assign new value New value (11)
Post-decrement result = x-- ① Assign old value → ② Decrement Old value (10)
Pre-decrement result = --x ① Decrement → ② Assign new value New value (9)
⚠

Interview Tip: "What's the difference between x++ and ++x?" -- Post (x++): assigns THEN increments. Pre (++x): increments THEN assigns. The difference only shows when you store the result in another variable.

Category ⑥

Ternary / Conditional Operator

✓

Real-Life Example: The ternary operator is like a drive-through menu with only 2 options: "Are you 18 or older? → You get coffee. Under 18? → You get juice." One question, two possible outcomes, instant decision. No need to park, walk inside, and talk to three people (a whole if/else block).

The ternary operator is a one-line shortcut for if/else. It takes a condition and returns one of two values:

Syntax

condition ? valueIfTrue : valueIfFalse
1
Basic ternary -- which number is bigger?
Simple 
let a: number = 100, b: number = 200;

let result: number = a > b ? a : b;
//                    ^^^^   ^   ^
//                 condition ‚   """ if false: use b
//                           """ if true: use a

console.log(result);  // 200
// 100 > 200 is FALSE → so b (200) is assigned to result
2
Real-world example -- adult or minor?
Practical 
let personAge: number = 30;

let status: string = personAge >= 18 ? "Adult" : "Minor";
//                    ^^^^^^^^^^^^^     ^^^^^^^   ^^^^^^^
//                    condition         if true   if false

console.log(status);  // "Adult"
// 30 >= 18 is TRUE → "Adult" is assigned to status
i

When to use ternary: Simple yes/no decisions where you pick between two values. For complex logic with multiple conditions, stick with if/else statements (covered in a future topic).

Watch Out!

⚠ Common Beginner Mistakes

⚠
Mistake 1: Using = instead of === in conditions

A single = is assignment (putting a value into a box). A triple === is comparison (checking if two things are equal). If you accidentally use = inside an if, it ALWAYS passes -- because the assignment itself is truthy! 

let score = 50;

// ❌ WRONG  --  this ASSIGNS 100 to score, doesn't compare!
if (score = 100) {
  console.log("Perfect!");  // ALWAYS runs! score is now 100
}

// ✅ CORRECT  --  this COMPARES score to 100
if (score === 100) {
  console.log("Perfect!");  // Only runs if score is actually 100
}

Rule: Inside if(), while(), or any condition -- ALWAYS use === (three equals), NEVER = (one equal).

⚠
Mistake 2: + with strings gives concatenation, not addition!

The + operator does MATH with numbers, but GLUES TEXT with strings. If even ONE side is a string, JavaScript converts everything to strings and joins them! 

// ❌ Surprise! String + Number = String (concatenation)
console.log("5" + 3);     // "53"  (not 8!)
console.log(5 + "3");     // "53"  (not 8!)
console.log("5" + 3 + 2); // "532" (left to right, all become string)
console.log(5 + 3 + "2"); // "82"  (5+3=8 first, then "8"+"2"="82")

// ✅ CORRECT  --  convert string to number first
console.log(Number("5") + 3);   // 8
console.log(parseInt("5") + 3); // 8

Rule: If your data comes from user input (it's always a string!), convert it with Number() or parseInt() before doing math.

⚠
Mistake 3: Operator precedence surprises

Just like in school math, * and / happen BEFORE + and -. Forgetting this gives wrong results! 

// ❌ WRONG expectation
let result = 2 + 3 * 4;
console.log(result); // 14 (NOT 20!)
// Why? 3 * 4 = 12 happens first, then 2 + 12 = 14

// ✅ Use parentheses to force your intended order
let result2 = (2 + 3) * 4;
console.log(result2); // 20 ✓

Rule: When in doubt, use (parentheses) to make the order crystal clear. It also makes your code easier to read!

Mistake What you wrote What it does Fix
= in condition if (x = 5) Assigns 5 to x (always true) if (x === 5)
+ with string "5" + 3 Concatenates → "53" Number("5") + 3
Precedence 2 + 3 * 4 Gives 14, not 20 (2 + 3) * 4

Quick Reference Cheatsheet

Arithmetic

let a = 10, b = 3;
a + b   // 13   Addition
a - b   // 7    Subtraction
a * b   // 30   Multiplication
a / b   // 3.33 Division (quotient)
a % b   // 1    Remainder
a ** b  // 1000 Exponent (10³)

Assignment shortcuts

a += b   // same as: a = a + b
a -= b   // same as: a = a - b
a *= b   // same as: a = a * b
a /= b   // same as: a = a / b
a %= b   // same as: a = a % b

Comparison / Relational

a > b    // greater than         → boolean
a < b    // less than            → boolean
a >= b   // greater or equal     → boolean
a <= b   // less or equal        → boolean
a == b   // equal (value only)   → boolean
a != b   // not equal            → boolean
a === b  // strict (value+type)  → boolean

Logical

true && true    // true   --  AND: both must be true
true && false   // false
true || false   // true   --  OR: at least one true
false || false  // false
!true           // false  --  NOT: flip the value
!false          // true

Increment / Decrement

x++    // post-increment: use old value, then add 1
++x    // pre-increment:  add 1 first, then use new value
x--    // post-decrement: use old value, then subtract 1
--x    // pre-decrement:  subtract 1 first, then use new value

Ternary

// condition ? ifTrue : ifFalse
let result = age >= 18 ? "Adult" : "Minor";
